## 一、财务数字化方案

### 1.1 2026 年行业现状

| 维度 | 现状（2026） |
|------|-------------|
| 数电票覆盖率 | 规上企业 98%+，全行业贯通 |
| 国产 ERP 市场份额 | 68.4%（较 2020 年提升 22.7 个百分点） |
| 核心趋势 | 从"上云"转向"用 AI"——AI 原生财务成为主赛道 |
| 乐企平台 | 国税总局企业直连平台，替代传统税控设备，实现开票/受票/入账全链路数字化 |
| 电子会计档案 | 纸质凭证加速退出，电子凭证全流程无纸化已成主流 |
| SaaS 市场规模 | 预计突破 3,000 亿元，年复合增长率 20%+ |

### 1.2 场景识别方法

将财务数字化需求收敛为可执行的场景方案，按以下步骤推进：

1. **定义业务问题**：现状 → 痛点 → 目标 → 约束
2. **选定场景**：从场景库中匹配最相关的 3-5 个场景
3. **设计方案**：流程、规则、数据、技术、组织协同
4. **输出估算**：投入、产出、上线时间（给区间，不给伪精确数字）

### 1.3 最小上下文收集

优先收集以下信息；若信息缺失，明确写出假设后继续推进，不因信息不全而卡住：

- 行业、公司规模、组织形态（集团 / 单体 / 多子公司 / 海内外）
- 当前核心痛点（效率、合规、数据质量、资金占用、经营分析、税务风险等）
- 现有系统格局（ERP、OA、费控、CRM、资金系统、BI、档案系统等）
- 目标范围（单流程、跨流程、平台化）
- 数据与接口可用性
- 合规与安全要求
- 期望上线窗口与预算约束

### 1.4 核心场景库

| 场景 | 典型痛点 | 关键技术 | 预期产出 |
|------|----------|----------|----------|
| 报销与费控 | 手工审批慢、票据合规难 | OCR、规则引擎、工作流 | 报销周期缩短 60-80% |
| 发票与税务协同 | 进项认证滞后、税负测算不及时 | 发票 OCR、税务引擎 | 进项认证及时率 >95% |
| 应付自动化 | 三单匹配人工、付款审批慢 | 自动匹配、工作流 | 应付处理效率提升 50-70% |
| 应收与回款管理 | 账龄长、催收无体系 | 信用评估、自动催收 | 回款天数缩短 10-20 天 |
| 资金与现金预测 | 资金调度靠经验、预测不准 | 现金池、AI 预测 | 资金利用率提升 15-30% |
| 预算与经营分析 | 预算编制周期长、执行监控弱 | BI、预算引擎 | 预算编制周期缩短 50% |
| 月结与关账 | 关账周期长、跨部门协同难 | 任务引擎、自动对账 | 关账周期缩短 3-5 天 |
| 电子会计档案 | 纸质归档成本高、查找困难 | 电子签章、OCR、归档系统 | 归档成本降低 70% |

### 1.5 需求分类与交付深度

| 需求类型 | 交付物 | 适用场景 |
|----------|--------|----------|
| 机会识别 | 3-5 个高价值场景 + 排序理由 | 战略讨论、售前 |
| 单场景方案 | 完整可落地方案 | 客户已明确痛点 |
| 多场景规划 | 分阶段建设路线图 | 年度规划、转型项目 |
| BRD | 面向决策层的业务需求文档 | 立项审批 |
| PRD | 面向实施团队的产品需求文档 | 进入开发/实施 |

### 1.6 方案设计要求

每个场景至少覆盖以下内容：
- 具体业务场景与适用边界
- 当前痛点与目标指标
- 方案设计（流程、规则、数据、组织协同）
- 所需技术（工作流、规则引擎、OCR、RPA、API 集成、数据中台、BI、AI 等）
- 关键依赖与风险

### 1.7 估算规则

所有方案默认给出区间估算，避免伪精确数字。估算必须显式标注假设前提，并至少拆出：

- **预计投入**：软件许可/订阅、实施服务、集成开发、数据治理、培训与推广、运维
- **预计产出**：人效提升、处理时长缩短、错误率下降、回款加快、资金占用降低、合规风险下降、管理透明度提升
- **上线时间**：PoC、一期上线、全面推广三个层次，必要时给出里程碑

### 1.8 BRD 规则

当用户要求 BRD 时，聚焦业务价值与决策信息，至少包含：

1. 背景与现状
2. 业务目标与 KPI
3. 项目范围与边界
4. 目标流程与角色变化
5. 收益假设与 ROI 逻辑
6. 预算与资源建议
7. 实施阶段与里程碑
8. 风险、依赖、待确认事项

避免把 BRD 写成功能列表堆砌；重点放在"为什么做、做什么、值不值得做"。

### 1.9 PRD 规则

当用户要求 PRD 时，聚焦落地实现，至少包含：

1. 产品目标与成功标准
2. 用户角色与使用场景
3. 业务流程与页面/节点流程
4. 功能清单与优先级
5. 关键规则、字段、状态机
6. 集成接口与数据要求
7. 权限、安全、审计、性能要求
8. 验收标准、上线策略、迁移与培训要求

避免只写高层概念；必须给出能够进入设计/开发/实施阶段的颗粒度。

### 1.10 质量标准

- 优先给出行业可落地表达，不写"赋能、提升效率"这类空话而不解释机制
- 所有估算都给范围，并写明决定范围变化的关键因素
- 所有收益都尽量绑定可观测指标，如处理时长、FTE 节约、回款天数、月结周期、差错率
- 明确区分"标准产品能力""需要集成""需要定制开发"三类工作
- 在中国企业环境下，金额默认使用 CNY 或"人民币"
- 涉及大模型或 AI 时，补充数据安全、权限审计、模型幻觉与人工复核机制

### 1.11 交付格式

默认交付为结构化文档。文档命名建议：
- `财务数字化方案-主题-YYYYMMDD`
- `BRD-主题-YYYYMMDD`
- `PRD-主题-YYYYMMDD`

## 二、会计凭证生成

> 凭证自动生成（上传文件→输出Excel）需要 MCP Server 支持。知识指引见下方。

### 2.1 数电票时代的凭证生成变化（2026）

| 变化点 | 传统模式 | 数电票模式 |
|--------|----------|-----------|
| 发票获取 | 纸质/PDF 扫描录入 | 乐企平台自动推送 XML 结构化数据 |
| 进项认证 | 手动勾选认证 | 系统自动匹配，无需认证动作 |
| 凭证生成 | 手工录入或 OCR 识别后人工复核 | 结构化数据直接映射科目，自动生成凭证 |
| 档案归档 | 纸质装订 + 电子备份 | 电子凭证全流程无纸化，符合《会计档案管理办法》 |
| 红冲处理 | 开具红字发票信息表 → 等待审核 | 系统内直接发起红冲，实时生效 |

**乐企平台对接要点：**
- 企业通过 API 直连税务局系统，无需税控设备
- 开票数据实时回传 ERP，自动触发凭证生成
- 受票数据自动匹配采购订单/合同，三单匹配自动化
- 支持批量开票（电商/物流等高频场景）

### 2.2 常见业务会计分录

#### 销售业务
```
确认收入：
  借：应收账款 / 银行存款
  贷：主营业务收入
  贷：应交税费—应交增值税（销项税额）

结转成本：
  借：主营业务成本
  贷：库存商品
```

#### 采购业务
```
采购入库：
  借：原材料 / 库存商品
  借：应交税费—应交增值税（进项税额）
  贷：应付账款 / 银行存款
```

#### 费用报销
```
借：管理费用 / 销售费用（按费用类型）
借：应交税费—应交增值税（进项税额）（如有）
贷：银行存款 / 其他应付款
```

#### 工资计提与发放
```
计提：
  借：管理费用—工资 / 销售费用—工资 / 生产成本—工资
  贷：应付职工薪酬—工资

发放：
  借：应付职工薪酬—工资
  贷：银行存款
  贷：应交税费—应交个人所得税
  贷：其他应付款—社保/公积金（个人部分）
```

#### 折旧计提
```
借：管理费用—折旧 / 制造费用—折旧
贷：累计折旧
```

## 三、业财一体化规划

### 3.1 核心理念

业财一体化 = 业务流程与财务流程的无缝衔接，实现：
- 业务数据自动触发财务记账
- 财务数据实时反映业务状态
- 消除手工录入和重复劳动

### 3.2 关键衔接点

| 业务流程 | 财务触发点 | 自动生成 |
|----------|-----------|----------|
| 销售发货 + 客户签收 | 收入确认 | 收入凭证 |
| 采购验收入库 | 存货增加 | 入库凭证 |
| 付款审批通过 | 资金支出 | 付款凭证 |
| 费用报销审批 | 费用确认 | 费用凭证 |
| 生产完工入库 | 成本结转 | 成本凭证 |

### 3.3 系统集成方案

```
业务系统（ERP/CRM/OA）
        │
        ▼ （接口/中间件）
财务系统（金蝶/用友/SAP）
        │
        ▼
管理报表（BI/数据中台）
```

**集成方式选择**：
| 方式 | 适用场景 | 复杂度 | 实时性 |
|------|----------|--------|--------|
| API 直连 | 系统间有标准接口 | 中 | 实时 |
| 中间件/ESB | 多系统集成 | 高 | 准实时 |
| 文件导入 | 系统无接口能力 | 低 | 批量 |
| RPA | 系统老旧无法改造 | 中 | 定时 |

## 四、月结流程（Month-End Close）

### 4.1 概述

月结是财务部门每月固定的关账流程。目标：确保当月所有经济业务已完整入账，报表数据准确可靠，管理层能基于正确数据做决策。

### 4.2 月结检查清单

| 序号 | 步骤 | 说明 |
|------|------|------|
| 1 | 截止确认 | 确认当月最后一笔业务已入账，跨月业务归属正确 |
| 2 | 银行对账 | 银行日记账 vs 银行对账单核对，编制余额调节表 |
| 3 | 往来核对 | 应收/应付明细账 vs 对方确认函/对账单 |
| 4 | 计提处理 | 工资、折旧、摊销、水电费、利息等按权责发生制计提 |
| 5 | 暂估处理 | 已到货未到票的存货暂估入账 |
| 6 | 税金计算 | 增值税、附加税、所得税计提 |
| 7 | 损益结转 | 收入/费用类科目结转至本年利润 |
| 8 | 试算平衡 | 借方合计 = 贷方合计 |
| 9 | 报表生成 | 资产负债表、利润表、现金流量表 |
| 10 | 差异分析 | 本月 vs 上月 vs 预算，编写差异说明 |

### 4.3 计提调度表（Accrual Schedule）

月末计提的核心逻辑：费用已发生但尚未取得发票或尚未支付，按权责发生制确认当期费用。

**标准计提项目：**

| 计提项目 | 计算依据 | 标准分录 |
|----------|----------|----------|
| 工资薪酬 | 当月应发工资总额 | 借：管理费用/销售费用/生产成本<br>贷：应付职工薪酬 |
| 折旧 | 固定资产原值 × 月折旧率 | 借：管理费用/制造费用<br>贷：累计折旧 |
| 无形资产摊销 | 原值 ÷ 使用年限 ÷ 12 | 借：管理费用<br>贷：累计摊销 |
| 长期待摊费用 | 总额 ÷ 摊销月数 | 借：管理费用/制造费用<br>贷：长期待摊费用 |
| 借款利息 | 本金 × 年利率 ÷ 12 | 借：财务费用<br>贷：应付利息 |
| 水电费（未到票） | 上月实际或本月抄表数 | 借：管理费用/制造费用<br>贷：其他应付款 |
| 审计费 | 合同总额 ÷ 12 | 借：管理费用<br>贷：其他应付款 |

**计提表格式：**

| 项目 | 计算基础 | 本期应计提 | 前期已计提 | 本期实际计提 | 支持文件 |
|------|----------|-----------|-----------|-------------|----------|

**冲回规则：** 上月计提本月收到发票时，先红冲计提分录，再按发票金额正常入账。差异计入当期损益。

### 4.4 科目余额变动表（Roll-Forward）

审计底稿和月结包的核心工具。结构：

```
期初余额                          X
  + 本期增加                      A
  - 本期减少                      B
  ± 重分类调整                    C
  ± 汇率变动影响                  D
期末余额                          Y
```

**校验规则：** X + A - B + C + D = Y。如果不等，差额为"未解释项"——必须查明原因，不能强行轧平。

**每一行必须有来源：**
- 期初 = 上期期末（上月关账数）
- 增加/减少 = 明细账查询（科目 + 日期范围 + 摘要筛选）
- 期末 = 本期科目余额表

**常用科目的变动表：**

| 科目 | 典型增加项 | 典型减少项 |
|------|-----------|-----------|
| 固定资产 | 购入、在建工程转入 | 处置、报废 |
| 累计折旧 | 本期计提 | 处置转出 |
| 应收账款 | 赊销收入 | 收款、坏账核销 |
| 存货 | 采购入库 | 销售出库、报废 |
| 长期借款 | 新增借款 | 到期偿还、重分类至一年内 |

### 4.5 差异分析（Variance Commentary）

月结包的最后一步：解释本月数据为什么和上月/预算不一样。

**触发阈值：** 满足以下任一条件的科目需要写差异说明：
- 绝对金额变动 ≥ 重要性水平（通常为收入的 5% 或固定金额如 50 万）
- 变动比例 ≥ 20%
- 必须说明的科目（收入、人工成本、现金）

**差异分析表格式：**

| 科目 | 本月实际 | 上月实际 | 预算 | vs上月变动 | vs预算变动 | 原因说明 |
|------|----------|----------|------|-----------|-----------|----------|

**写原因的规则：**
- 说"为什么"，不是说"是什么"
- ✗ "管理费用增加了 30 万"（这是重复数字）
- ✓ "管理费用增加 30 万，主要系本月支付年度审计费 25 万及新增办公室租金 5 万"
- 如果从数据中看不出原因，写"原因待确认——需向业务部门核实"，不编造

## 五、中国企业三大报表标准科目结构

> 以下为财政部规定的一般企业财务报表格式（2019年修订版），古立特在生成 TB、做报表分析、勾稽校验时必须按此结构分类，不得混淆。

### 5.1 资产负债表（Balance Sheet）

**资产侧：**

| 分类 | 科目 |
|------|------|
| **流动资产** | 货币资金、交易性金融资产、衍生金融资产、应收票据、应收账款、应收款项融资、预付款项、其他应收款、存货、合同资产、持有待售资产、一年内到期的非流动资产、其他流动资产 |
| **非流动资产** | 债权投资、其他债权投资、长期应收款、长期股权投资、其他权益工具投资、其他非流动金融资产、投资性房地产、固定资产、在建工程、生产性生物资产、油气资产、使用权资产、无形资产、开发支出、商誉、长期待摊费用、递延所得税资产、其他非流动资产 |

**负债侧：**

| 分类 | 科目 |
|------|------|
| **流动负债** | 短期借款、交易性金融负债、衍生金融负债、应付票据、应付账款、预收款项、合同负债、应付职工薪酬、应交税费、其他应付款、持有待售负债、一年内到期的非流动负债、其他流动负债 |
| **非流动负债** | 长期借款、应付债券、租赁负债、长期应付款、预计负债、递延收益、递延所得税负债、其他非流动负债 |

**所有者权益：**

| 分类 | 科目 |
|------|------|
| **所有者权益** | 实收资本（股本）、其他权益工具、资本公积、减：库存股、其他综合收益、专项储备、盈余公积、未分配利润 |

**勾稽关系：** 资产合计 = 负债合计 + 所有者权益合计

---

### 5.2 利润表（Income Statement）

| 分类 | 科目 |
|------|------|
| **营业收入** | 营业收入 |
| **减：营业成本** | 营业成本 |
| **减：税金及附加** | 税金及附加 |
| **减：期间费用** | 销售费用、管理费用、研发费用、财务费用（其中：利息费用、利息收入） |
| **加：其他收益** | 其他收益 |
| **加：投资收益** | 投资收益（其中：对联营/合营企业的投资收益、以摊余成本计量的金融资产终止确认收益） |
| **减：信用/资产减值** | 信用减值损失、资产减值损失 |
| **加：资产处置收益** | 资产处置收益 |
| **= 营业利润** | |
| **加：营业外收入** | 营业外收入 |
| **减：营业外支出** | 营业外支出 |
| **= 利润总额** | |
| **减：所得税费用** | 所得税费用 |
| **= 净利润** | |
| **其他综合收益** | 其他综合收益的税后净额 |
| **= 综合收益总额** | |

**勾稽关系：**
- 营业利润 = 营业收入 - 营业成本 - 税金及附加 - 销售费用 - 管理费用 - 研发费用 - 财务费用 + 其他收益 + 投资收益 - 信用减值损失 - 资产减值损失 + 资产处置收益
- 利润总额 = 营业利润 + 营业外收入 - 营业外支出
- 净利润 = 利润总额 - 所得税费用

---

### 5.3 现金流量表（Cash Flow Statement）

| 分类 | 科目 |
|------|------|
| **经营活动** | 销售商品/提供劳务收到的现金、收到的税费返还、收到其他与经营活动有关的现金 |
| | 购买商品/接受劳务支付的现金、支付给职工的现金、支付的各项税费、支付其他与经营活动有关的现金 |
| | **经营活动现金流量净额** |
| **投资活动** | 收回投资收到的现金、取得投资收益收到的现金、处置固定资产/无形资产/其他长期资产收回的现金净额、处置子公司及其他营业单位收到的现金净额、收到其他与投资活动有关的现金 |
| | 购建固定资产/无形资产/其他长期资产支付的现金、投资支付的现金、取得子公司及其他营业单位支付的现金净额、支付其他与投资活动有关的现金 |
| | **投资活动现金流量净额** |
| **筹资活动** | 吸收投资收到的现金、取得借款收到的现金、收到其他与筹资活动有关的现金 |
| | 偿还债务支付的现金、分配股利/利润或偿付利息支付的现金、支付其他与筹资活动有关的现金 |
| | **筹资活动现金流量净额** |
| **汇率变动** | 汇率变动对现金及现金等价物的影响 |
| **合计** | 现金及现金等价物净增加额、期初余额、期末余额 |

**勾稽关系：**
- 现金净增加额 = 经营活动净额 + 投资活动净额 + 筹资活动净额 + 汇率变动影响
- 期末现金 = 期初现金 + 现金净增加额
- 期末现金 = 资产负债表"货币资金"（如无受限资金）

## 六、报表 Excel 输出格式规范

> 古立特在生成资产负债表、利润表、试算平衡表等 Excel 文件时，必须遵循以下格式规范。

### 6.1 资产负债表格式（左右对照式）

**中国企业资产负债表必须使用左右对照格式，不得使用上下排列格式。**

```
A列-D列（左半边：资产）    | E列-H列（右半边：负债和所有者权益）
───────────────────────────|────────────────────────────────────
第1行：表头（公司名称、报表名称、日期）
第2行：列标题
  A: 资产                   | E: 负债和所有者权益
  B: 行次                   | F: 行次
  C: 期末余额               | G: 期末余额
  D: 期初余额               | H: 期初余额
第3行起：
  左边从"流动资产"开始      | 右边从"流动负债"开始
  ...                       | ...
  非流动资产                | 非流动负债
  ...                       | ...
                            | 所有者权益
                            | ...
最后一行（必须对齐）：
  资产合计    XXX           | 负债和所有者权益合计    XXX
```

**关键规则：**
1. **左右对齐**：资产合计和负债+权益合计必须在同一行
2. **空行填充**：如果左边行数少于右边（或反之），用空行填充使合计行对齐
3. **分类标题加粗**：流动资产、非流动资产、流动负债、非流动负债、所有者权益
4. **合计行加粗+底线**

### 6.2 Excel 样式规范（Kiro 紫色系）

## 七、报表科目取数规则（从账到表的映射）

> 以下规则说明资产负债表每个报表项目如何从会计科目余额计算得出。
> 古立特在生成报表、做勾稽校验、或帮用户理解"这个数怎么来的"时，必须按此规则。

### 7.1 资产负债表——资产侧

| 报表项目 | 取数规则 |
|----------|----------|
| 货币资金 | "库存现金" + "银行存款" + "其他货币资金" |
| 交易性金融资产 | "交易性金融资产"科目余额（执行新金融工具准则） |

| 应收款项融资 | "应收款项融资"余额（以公允价值计量且变动计入其他综合收益的应收票据和应收账款） |
| 预付款项 | "预付账款"借方余额 + "应付账款"借方余额（预付性质的） - 相关坏账准备 |
| 其他应收款 | "应收利息" + "应收股利" + "其他应收款"借方余额 + "其他应付款"借方余额（应收性质的） - 相关坏账准备 |
| 存货 | "原材料" + "在途物资" + "库存商品" + "生产成本" + "制造费用" + "委托加工物资" + "周转材料" + "发出商品" 等 - "存货跌价准备" |
| 合同资产 | "合同资产"借方余额 - "合同资产减值准备"（执行新收入准则） |
| 持有待售资产 | "持有待售资产"余额 - "持有待售资产减值准备" |
| 一年内到期的非流动资产 | 将于一年内到期的长期资产净额（如一年内到期的债权投资） |
| 其他流动资产 | 不属于以上各项的流动资产（如待抵扣进项税额、预缴所得税等） |
| 债权投资 | "债权投资"余额 - "债权投资减值准备"（以摊余成本计量的长期债权） |
| 其他债权投资 | "其他债权投资"余额（以公允价值计量且变动计入其他综合收益的长期债权） |
| 长期应收款 | "长期应收款"余额 - "未实现融资收益" - 相关坏账准备 |
| 长期股权投资 | "长期股权投资"余额 - "长期股权投资减值准备" |
| 其他权益工具投资 | "其他权益工具投资"余额（指定为以公允价值计量且变动计入其他综合收益的非交易性权益投资） |
| 其他非流动金融资产 | 超过一年的以公允价值计量且变动计入当期损益的金融资产 |
| 投资性房地产 | 成本模式："投资性房地产" - "投资性房地产累计折旧/摊销" - "投资性房地产减值准备"；公允价值模式："投资性房地产"余额 |

### 7.4 关键注意事项

1. **减值准备必须扣除**：应收账款、存货、固定资产、无形资产、商誉等，报表列示的都是"净额"（原值 - 减值准备）
2. **借贷方向决定归属**：
   - "预付账款"借方余额 → 资产（预付款项）
   - "预付账款"贷方余额 → 负债（应付账款）
   - "应付账款"借方余额 → 资产（预付款项）
   - "应交税费"借方余额 → 资产（其他流动资产）
3. **一年内到期的重分类**：长期资产/负债中将于一年内到期的部分，要重分类到流动资产/流动负债
4. **新准则标记**：
   - 执行新收入准则 → 用"合同资产/合同负债"替代部分"应收账款/预收账款"
   - 执行新金融工具准则 → 用"交易性金融资产/债权投资/其他债权投资"替代旧分类
   - 执行新租赁准则 → 新增"使用权资产/租赁负债"
5. **"未实现融资收益"是抵减项**：长期应收款要扣除未实现融资收益后列示

## 八、工资个税与社保实操公式

> 类型：公式
> 来源：个人所得税法（2019修正）+ 社保征缴实务（古立特化重写）
> 置信度：准则原文（税率表）+ 实务经验（社保基数规则）
> 关联：tax_core.md - 第四章个人所得税（税率表/专项附加/汇算清缴完整规则）

[场景：编制工资表/薪资条、累计预扣个税计算、社保公积金基数确认、年终奖筹划。本节是"落地计算公式"而非税法完整规则——税法完整内容见 tax_core.md 第四章。]

### 8.1 累计预扣法（逐月计算公式）

> 类型：公式
> 来源：个人所得税法第六条 + 国家税务总局公告2018年第61号

工资薪金所得**应当**按累计预扣法逐月计算预扣税款：

```
本期应预扣预缴税额 = (累计预扣预缴应纳税所得额 × 预扣率 - 速算扣除数) - 累计减免税额 - 累计已预扣预缴税额
```

其中：
```
累计预扣预缴应纳税所得额 = 累计收入 - 累计免税收入 - 累计减除费用 - 累计专项扣除 - 累计专项附加扣除 - 累计依法确定的其他扣除

累计减除费用 = 5,000 × 当前月份数
```

**标准薪资表列结构（逐月）：**

| 列 | 说明 |
|----|------|
| 应发工资 | 基本工资 + 绩效 + 津贴 + 加班费 + 其他 |
| 社保个人部分 | 养老8% + 医疗2% + 失业0.5%（各地略有差异） |
| 公积金个人部分 | 5%-12%（单位自定比例） |
| 累计收入 | 本年1月至本月应发工资合计 |
| 累计减除费用 | 5,000 × 月份数 |
| 累计专项扣除 | 本年累计社保+公积金个人部分 |
| 累计专项附加 | 子女教育2000/月 + 继续教育400/月 + 住房贷款1000/月 + 住房租金(800-1500)/月 + 赡养老人(独生3000/非独分摊)/月 + 3岁以下婴幼儿2000/月 + 大病医疗(据实扣除年度内) |
| 累计应纳税所得额 | 累计收入 - 累计减除费用 - 累计专项扣除 - 累计专项附加 |
| 本期应预扣税额 | 按上方公式计算 |
| 实发工资 | 应发工资 - 社保个人 - 公积金个人 - 本期应预扣税额 |

### 8.2 年终奖计税决策规则

> 类型：判断表
> 来源：财税〔2018〕164号 + 财政部 税务总局公告2023年第30号（延续至2027年12月31日）
> 置信度：准则原文

全年一次性奖金**可选择**单独计税或并入综合所得：

| 方式 | 计算方法 | 适用人群 |
|------|---------|---------|
| 单独计税 | 奖金 ÷ 12 → 查月度税率表 → 奖金 × 税率 - 速算扣除数 | 年薪较高、综合所得已用满低税率空间者 |
| 并入综合所得 | 奖金加入全年收入一起算7级表 | 年薪较低（<约12万），低税率空间还有富余者 |

**简易决策规则**（实务经验，非精确临界值）：

- 年薪（不含年终奖）**＜ 约12万**：并入综合所得通常更优（低税率空间没用完）
- 年薪（不含年终奖）**≥ 约50万**：单独计税几乎必定更优
- 中间区间：**两种方式都算一遍取低的**

⚠️ 年终奖月度税率表存在"陷阱区间"（如36,001元比36,000元多交2,310元税），发放金额应避开临界点。

### 8.3 社保缴费基数封顶保底

> 类型：公式
> 来源：社会保险法 + 各地社保征缴规则
> 置信度：实务经验

```
本人月缴费基数 = MAX(当地社平下限, MIN(当地社平上限, 本人上年度月均工资))
```

- **下限**：通常为当地上年度社会平均工资的 **60%**
- **上限**：通常为当地上年度社会平均工资的 **300%**
- 每年 7 月调整（各地执行月份略有差异）

**个人缴费比例（通用参考，各地微调）：**

| 险种 | 个人比例 | 单位比例 |
|------|---------|---------|
| 养老保险 | 8% | 16% |
| 医疗保险 | 2% | 6.5%-10%（各地差异大） |
| 失业保险 | 0.5% | 0.5% |
| 工伤保险 | 0 | 0.2%-1.9%（行业差异） |
| 生育保险 | 0 | 已并入医疗 |
| 住房公积金 | 5%-12% | 5%-12%（单位自定，双方同比例） |

⚠️ 个人合计约 10.5%+公积金，实际数字以当地当年为准。

### 8.4 银行流水匹配键对账法

> 类型：流程
> 来源：实务银行对账经验
> 置信度：实务经验

[场景：银行对账单 vs 企业银行日记账逐笔匹配——用于银行余额调节表编制、费用报销核对]

**匹配键构造规则**：

以"**日期（±容差天数）+ 金额（精确到分）+ 借贷方向**"作为匹配键：

1. 金额完全一致 + 方向一致（银行支出=账面支出、银行收入=账面收入）→ 自动匹配
2. 金额一致但日期差 1-3 天 → 疑似匹配（银行处理时差），人工确认
3. 金额一致但方向相反 → 不匹配（可能是退款冲原交易，需单独核对）
4. 银行有账面无 → 银行已收/付企业未记（未达账项）
5. 账面有银行无 → 企业已记银行未到（未达账项）

⚠️ 批量代发工资/社保/税款等会合并为一笔银行流水（汇总金额），需拆分后再匹配明细。
