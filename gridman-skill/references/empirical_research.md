# 财税审实证研究方法手册

> **定位**：服务财税审领域写实证论文、做研究的人
> **不是**：计量经济学教科书摘录（那是伍德里奇、陈强、张俊瑞《会计实证研究方法》的事）
> **是**：财税场景驱动的实证手册——**按"研究问题"组织，不按"统计方法"组织**
> **目标读者**：会计/审计/税务/公司金融领域的研究者、博硕士生、写实证 MPAcc 案例的学员
> **关联模块**：
> - 写 MPAcc 教学案例 → 主参考 `mpacc_case_template.md`，本文件作方法补充
> - 写 SSCI / CSSCI 论文 → 本文件
> - 案例里需要做事件研究 → 本文件「事件研究法」一节

## 第一章：定位与边界

### 这本手册解决什么问题

**财税审领域的研究者**有几个共性痛点：

1. 通用 AI 知道"什么是 DID"，但不知道**财税圈用 DID 通常研究什么、控制变量怎么选、经典论文是哪些**
2. 计量教科书讲方法严谨，但不告诉你"这个方法在《会计研究》《审计研究》上**实际怎么用**"
3. 跨学科语言不通——做财税的看到希腊字母就头晕，做计量的看到"管理层凌驾"就迷茫

本手册的取向：

- **场景化**：每个方法配 1-2 个真实研究案例（公开发表过的论文）
- **可执行**：每个方法配 Stata 命令模板（少量 Python / R 补充）
- **避坑导向**：每个方法附"审稿人最爱质疑的点"
- **拒绝抽象**：不讲数学推导，只讲"什么时候用、怎么用、用对了没"

### 边界声明

**本手册不覆盖：**

- 时间序列计量（VAR / 协整 / GARCH）—— 财税审研究中较少用
- 高级计量（结构方程、贝叶斯、机器学习）—— 用到时单独查
- 数学推导 —— 找伍德里奇 / 陈强教材
- 软件入门 —— 找 Stata Cheatsheet / UCLA 统计计算网

**本手册重点覆盖：**

会计/审计/税务/公司金融领域**90% 的实证论文**会用到的方法：OLS 多元回归 / 面板数据 / DID / PSM / IV / Heckman / 事件研究 / Logit-Probit / 中介调节 / 稳健性检验

## 第二章：财税审实证研究的命题地图

研究的"高质量问题"比"高级方法"更重要。先看自己的问题落在哪个研究领域，常见研究范式是什么，再选方法。

### 2.1 会计研究领域

#### 2.1.1 盈余管理（Earnings Management）

**经典研究问题**：

- 公司治理因素 / 审计师特征 / 高管特征对盈余管理程度的影响
- 制度变化（IFRS 趋同、会计准则变更）的盈余管理效应
- 盈余管理的经济后果（市场反应、融资成本、被处罚概率）

**因变量度量**：操控性应计 |DA|（修正 Jones 模型估出的残差），见第六章「因变量度量库」

**主流方法**：横截面 OLS / 公司年度面板 FE / 制度冲击下的 DID

**经典文献**：
- Jones (1991) JAR 提出 Jones 模型
- Dechow, Sloan & Sweeney (1995) AR 修正 Jones 模型
- Kothari, Leone & Wasley (2005) JAE 提出业绩匹配（PMOM）
- 国内：陆建桥（1999）经济研究，最早把 Jones 模型引入中国

#### 2.1.2 会计稳健性（Accounting Conservatism）

**经典研究问题**：

- 审计师 / 治理 / 制度对稳健性的影响
- 稳健性的契约价值（债务契约、薪酬契约）

**因变量度量**：Basu (1997) 模型回归系数 / C_Score (Khan & Watts, 2009)

**主流方法**：Basu 回归（特殊形式 OLS）

**经典文献**：
- Basu (1997) JAE
- Watts (2003) AH 综述
- Khan & Watts (2009) JAE 提出 C_Score

#### 2.1.3 信息披露质量

**经典研究问题**：

- 信息披露质量的决定因素（治理、所有制、规模……）
- 信息披露质量的经济后果（融资成本、股价同步性、分析师跟踪）

**因变量度量**：深交所信息披露考核评级（A/B/C/D）/ Botosan (1997) 自建指数 / KV 指数

**主流方法**：Ordered Logit / OLS / 工具变量

#### 2.1.4 投资效率

**经典研究问题**：

- 公司治理对投资效率的影响（过度投资 vs 投资不足）
- 信息环境（披露质量、分析师跟踪）对投资效率的影响

**因变量度量**：Richardson (2006) 模型——预期投资 vs 实际投资的残差

**主流方法**：两阶段（先估预期投资，再回归残差）/ FE 面板

### 2.2 审计研究领域

#### 2.2.1 审计费用决定

**经典研究问题**：DeAngelo / Simunic 框架延展——什么因素影响审计费用？

**主流模型**：
```
ln(Audit Fee) = β₀ + β₁·Size + β₂·Inv + β₃·Rec + β₄·Loss + β₅·Big4
              + β₆·Risk + 控制变量 + 行业FE + 年度FE + ε
```

**经典文献**：Simunic (1980) JAR / Hay, Knechel & Wong (2006) AR 综述

#### 2.2.2 审计意见 / 审计师变更

**经典研究问题**：

- 影响出具非标审计意见的因素（公司、审计师、制度）
- 影响审计师变更的因素 + 变更的经济后果

**因变量**：非标意见（哑变量，1=非标 0=标准） / 审计师变更（哑变量）

**主流方法**：Logit / Probit

**核心难点**：**自选择问题**——审计师不是被随机分配的，需要 Heckman 两阶段或 PSM

#### 2.2.3 审计质量替代变量

学界没有"审计质量"的直接观测，常用替代变量：

- 操控性应计 |DA|（间接：高质量审计 → 抑制盈余管理）
- 财务重述（财务报告事后被强制更正 → 审计失败信号）
- 审计意见购买（机会主义变更审计师后获得标准意见）
- 内部控制审计意见
- 审计费用（双向解释——质量信号 vs 努力投入）

#### 2.2.4 关键审计事项（KAM）

**研究热点**（2017 年新审计报告准则后兴起）：

- KAM 披露的市场反应
- KAM 数量、内容对盈余反应系数的影响
- KAM 与审计师感知风险的关系

**主流方法**：事件研究 / 跨期 DID（以新准则实施为冲击）

### 2.3 税务研究领域

#### 2.3.1 税收激进 / 避税程度

**经典研究问题**：

- 公司治理 / 高管特征对避税程度的影响
- 避税的经济后果（公司价值、信息透明度）

**因变量度量**（多种，常并用做稳健性）：

- ETR（实际税率） = 所得税费用 / 税前利润
- Cash ETR = 实际缴纳所得税 / 税前利润
- BTD（账税差异） = 税前利润 - 应税所得（Hanlon & Heitzman, 2010 综述）
- DD_BTD（异常 BTD）：扣除应计利润后的 BTD 残差

**主流方法**：FE 面板 / DID（税制改革冲击）

**经典文献**：Hanlon & Heitzman (2010) JAE 综述 / 吴联生（2009）经济研究

#### 2.3.2 税收优惠政策评估

**经典研究问题**：

- 高新技术企业认定的政策效果
- 西部大开发 / 自贸区 / 海南自贸港的税收激励效应
- 研发加计扣除的创新激励效应

**首选方法**：DID（认定为政策冲击，未认定为对照组）/ RDD（认定门槛附近）

### 2.4 公司治理研究领域

#### 2.4.1 高管薪酬

**经典研究问题**：

- 业绩-薪酬敏感性
- 高管薪酬与公司治理 / 风险承担 / 业绩操纵的关系

**主流方法**：FE 面板 / 内生性处理（薪酬与业绩互为因果）

#### 2.4.2 股权激励效果

**研究问题**：股权激励是否提升了公司价值 / 创新产出 / 长期业绩

**首选方法**：PSM-DID（实施股权激励的公司 vs 匹配的对照公司）

#### 2.4.3 董事会特征

**研究问题**：独立董事比例 / 女性董事 / 董事会规模 / CEO 兼任董事长 → 公司绩效

**主流方法**：FE 面板 / 工具变量（董事会结构内生）

### 2.5 ESG 与可持续金融

#### 2.5.1 ESG 表现的决定因素与经济后果

**经典研究问题**：

- 公司治理 / 媒体关注 / 制度环境 → ESG 表现
- ESG 表现 → 公司价值 / 融资成本 / 股价崩盘风险 / 创新

**因变量度量**：
- MSCI ESG 评级（A 字母制）
- 华证 ESG 评级
- 商道融绿 ESG 评级
- 自建 ESG 指数（KLD 借鉴）

**主流方法**：FE 面板 / 中介机制 / 异质性分析

#### 2.5.2 绿色金融政策评估

**研究问题**：绿色信贷 / 绿色债券 / 碳排放权交易试点的政策效果

**首选方法**：DID（试点城市/行业为处理组）

### 2.6 资本市场研究

#### 2.6.1 IPO 抑价 / 增发 / 并购绩效

**研究问题**：

- IPO 抑价的影响因素
- 定向增发对长期业绩的影响
- 并购的协同效应

**主流方法**：横截面 OLS（IPO） / 事件研究（短期市场反应） / 长期 BHAR

#### 2.6.2 盈余反应系数（ERC）

**经典研究问题**：盈余披露引发的市场反应——什么因素让 ERC 更高 / 更低？

**主流方法**：事件研究 + 横截面回归 ERC = 公司特征

#### 2.6.3 股价崩盘风险

**因变量度量**：NCSKEW（负偏度系数） / DUVOL（负正波动比）

**主流方法**：FE 面板

### 2.7 一张总结图

```
研究问题 ─→ 选择研究领域 ─→ 确定因变量度量 ─→ 选方法
                                              ├─ 横截面 → OLS
                                              ├─ 多年数据 → 面板（FE / RE）
                                              ├─ 政策冲击 → DID
                                              ├─ 自选择 → PSM / Heckman
                                              ├─ 内生变量 → IV
                                              ├─ 二元因变量 → Logit / Probit
                                              ├─ 短期市场反应 → 事件研究
                                              └─ 阈值附近 → RDD
```

### 3.1 OLS 多元回归（万能起点）

#### 核心思路

最小二乘法找一组系数，让因变量预测值与实际值的平方误差最小。**财税审 80% 的实证论文回归方程都是 OLS 起家**，不管最终怎么处理内生性，第一步先跑 OLS 看基准结果。

#### 适用场景

- 因变量连续 / 近似连续（应计、ETR、薪酬对数、Tobin's Q）
- 横截面数据 + 简单面板基准
- 任何研究的「基准回归」（baseline）

#### 模型设定

标准财税论文回归：

```
Y_it = α + β·X_it + γ·Controls_it + δ·Year + θ·Industry + ε_it
```

其中：
- `Y` = 因变量（如盈余管理 |DA|）
- `X` = 关键自变量（论文核心解释变量）
- `Controls` = 公司层面控制变量（见第七章「控制变量标准菜单」）
- `Year` = 年度固定效应（控制宏观冲击）
- `Industry` = 行业固定效应（控制行业差异，常用证监会 2012 行业分类）

#### Stata 命令模板

```stata
* 1. 数据准备
xtset id year                              // 声明面板结构

* 2. 基准 OLS（含双向固定效应）
reghdfe DA Treat Size Lev ROA Growth Big4 ///
    , absorb(year industry) ///
      cluster(id)
* reghdfe 是 reg 的高效版本，处理多重固定效应快

* 旧版 Stata 用 areg 也行：
areg DA Treat Size Lev ROA Growth Big4 i.year, absorb(industry) cluster(id)

* 3. 直接 reg 的写法（小样本可用）
reg DA Treat Size Lev ROA Growth Big4 i.year i.industry, robust cluster(id)
```

#### 常见坑 + 审稿人质疑

| 坑 | 表现 | 处理 |
|---|---|---|
| 异方差 | OLS 残差方差非常数 | 必用 `robust` 或 `cluster` 选项 |
| 多重共线性 | 控制变量高度相关，系数不稳定 | VIF 检验（>10 警惕），删共线变量 |
| 序列相关 | 公司多年数据残差自相关 | 用 `cluster(id)` 调整标准误 |
| 遗漏变量 | 没控制住的因素与 X 相关 | 加固定效应 / 工具变量 / 增加控制变量 |
| 反向因果 | Y 也可能影响 X | 用滞后期 X / 工具变量 / DID |
| 自选择 | 处理组不是随机的 | PSM / Heckman |

**审稿人最爱问**：

1. "**内生性怎么处理？**" → 几乎必问，准备好 IV / DID / PSM 至少一种
2. "**为什么用这套控制变量？**" → 准备好文献依据
3. "**结果是否稳健？**" → 至少 3 种稳健性（换度量、缩样本、换方法）

#### 财税场景实例

**研究问题**：审计师是否能抑制公司盈余管理？

```stata
reghdfe abs_DA Big4 Size Lev ROA Loss Growth MTB ///
    , absorb(year industry) cluster(id)
```

- `abs_DA` = 操控性应计的绝对值（修正 Jones 模型残差）
- `Big4` = 是否四大审计（哑变量，**关键解释变量**）
- 预期 `Big4` 系数显著为负 → 四大审计抑制盈余管理

**经典文献**：
- Becker et al. (1998) CAR：四大审计与盈余管理
- 国内：王鹏 & 周黎安（2006）会计研究

### 3.2 面板数据（FE / RE）

#### 核心思路

公司多年数据有两个维度的变异：**公司间差异**（A 公司 vs B 公司）和**时间内差异**（A 公司 2020 vs A 公司 2021）。面板模型让你**控制掉不随时间变化的公司特质**（如公司文化、行业属性、产权性质），从而更干净地识别 X 的效应。

#### 适用场景

- 多年数据（财税审实证最常见的数据形式）
- 担心公司不可观测异质性（管理层风格、文化、地理位置）干扰因果识别
- 需要"组内"识别效应（within-firm variation）而非"组间"识别

#### 三种估计

| 模型 | 假设 | 适用 |
|---|---|---|
| **POOLED OLS** | 所有公司同质 | 几乎不用，作基准 |
| **固定效应（FE）** | 公司效应与 X 相关 | **财税研究首选** |
| **随机效应（RE）** | 公司效应与 X 不相关 | 假设强，少用 |

**Hausman 检验** 决定 FE vs RE：拒绝原假设 → 用 FE。**财税圈惯例几乎都直接用 FE**，因为公司效应与变量相关的可能性极高。

#### 模型设定

固定效应模型：

```
Y_it = α_i + β·X_it + γ·Controls_it + δ·Year + ε_it
```

`α_i` = 公司固定效应（吸收掉所有不随时间变化的公司特质）

注意：**用了公司 FE 后**，行业 FE 自动被吸收（因为行业是公司的属性，不随时间变化）。但**年度 FE 还要单独加**——它捕捉每年的宏观冲击。

#### Stata 命令模板

```stata
* 声明面板
xtset id year

* 1. 固定效应模型（标准写法）
xtreg Y X Controls i.year, fe cluster(id)

* 2. reghdfe（推荐，处理高维 FE 快）
reghdfe Y X Controls, absorb(id year) cluster(id)

* 3. 双重固定效应（公司 + 年度）
reghdfe Y X Controls, absorb(id year industry#year) cluster(id)
* 加 industry#year 控制行业-年度交互（同行业同年共同冲击）

* 4. Hausman 检验
xtreg Y X Controls, fe
estimates store fe
xtreg Y X Controls, re
estimates store re
hausman fe re

* 5. 是否需要时间效应：F 检验
testparm i.year
```

#### 常见坑 + 审稿人质疑

| 坑 | 表现 | 处理 |
|---|---|---|
| 公司 FE 吃掉低变异变量 | 不随时间变化的 X 系数变成 0 或不显著 | 改用 between/POLS / 找替代 X |
| 单期数据假冒面板 | 只有 1 年还用 FE | 退化为 OLS，没意义 |
| 标准误未聚类 | 残差在公司层面相关，标准误偏小 | 必须 `cluster(id)` |
| 短面板偏误 | T < 5 年 + 滞后因变量 | 用差分 GMM / 系统 GMM |

**审稿人质疑**：

1. "**为什么是 FE 不是 RE？**" → 用 Hausman 拒绝 RE，或直接说"公司不可观测异质性可能与解释变量相关"
2. "**为什么没控制行业-年度交互固定效应？**" → 越来越严格的杂志会要求加 `industry#year`

#### 财税场景实例

**研究问题**：上市公司股权激励对企业创新（专利）的影响

```stata
xtset id year
reghdfe ln_patent Equity_Incentive Size Lev ROA RD_intensity Age ///
    , absorb(id year) cluster(id)
```

- `id` 公司 FE → 控制公司创新基因等不可观测因素
- `year` FE → 控制宏观环境（疫情、政策周期）
- `Equity_Incentive` 系数显著为正 → 股权激励促进创新

**注意**：但这个回归依然不能解决"实施股权激励的公司本来就更可能创新"的自选择问题——需要 PSM-DID 才更可信。

### 3.3 双重差分（DID）

#### 核心思路

**因果推断的金钥匙**。当一项政策只影响一部分公司（处理组），不影响另一部分（对照组），通过比较「处理组前后变化」与「对照组前后变化」的**差**，识别政策效应。

「**双重**」= 处理组事后-事前差 -（对照组事后-事前差）= 政策效应

#### 适用场景

财税审领域 DID 最适合的研究场景：

| 政策冲击 | 处理组 | 对照组 |
|---|---|---|
| 高新技术企业认定 | 被认定的公司 | 未被认定的可比公司 |
| 自贸区 / 海南自贸港 | 区内企业 | 区外可比企业 |
| 营改增 | 试点行业 | 非试点行业 |
| 新会计准则颁布 | 受影响的公司 | 不受影响的公司 |
| KAM 新准则 | A+H 股先实施 | A 股 |
| 注册制改革 | 科创板/创业板 | 主板 |
| 沪港通 / 深港通 | 标的股票 | 非标的股票 |
| 绿色信贷指引 | 重污染企业 | 非重污染企业 |
| 碳排放权交易试点 | 试点省份 | 非试点省份 |

#### 模型设定

**标准 DID 方程**：

```
Y_it = α + β₁·Treat_i + β₂·Post_t + β₃·Treat_i × Post_t + γ·Controls + ε_it
```

- `Treat_i` = 是否处理组（1/0）
- `Post_t` = 是否政策实施后（1/0）
- `Treat × Post` = **DID 项，β₃ 就是政策效应**

**带固定效应版本（推荐）**：

```
Y_it = α_i + δ_t + β·DID_it + γ·Controls_it + ε_it
```

加入公司 FE（α_i）后，`Treat` 主效应被吸收；加入年度 FE（δ_t）后，`Post` 主效应被吸收。**只剩 DID 交乘项**——这是当代财税论文的标准写法。

#### Stata 命令模板

```stata
* 1. 生成 DID 变量
gen post = (year >= 2020)            // 假设 2020 年政策实施
gen did = treat * post

* 2. 双向固定效应 DID
reghdfe Y did Controls, absorb(id year) cluster(id)
* did 的系数就是政策效应

* 3. 平行趋势检验（核心！）
* 生成相对政策时点的虚拟变量
forvalues i = -3/3 {
    local j = `i' + 3
    gen pre_`j' = (year - 2020 == `i') * treat
}
drop pre_3                            // 政策前一年作基准期
reghdfe Y pre_0 pre_1 pre_2 pre_4 pre_5 pre_6 Controls ///
    , absorb(id year) cluster(id)
coefplot, drop(_cons) yline(0) vertical

* 政策前的系数应不显著为零（平行趋势成立）
* 政策后的系数应显著（政策起效）

* 4. 安慰剂检验（虚假政策时点）
* 把政策时点提前 2-3 年，应该跑不出显著结果
gen post_fake = (year >= 2018)
gen did_fake = treat * post_fake
reghdfe Y did_fake Controls, absorb(id year) cluster(id)
* did_fake 应不显著

* 5. 安慰剂检验 2（随机抽处理组）
* 随机赋予 treat，重复 1000 次，看真实 did 系数是否在分布尾部
permute treat _b[did], reps(1000) seed(1234): reghdfe Y did Controls, absorb(id year) cluster(id)
```

#### 五大稳健性检查（缺一审稿人不放）

1. **平行趋势检验** —— 政策前处理组和对照组趋势一致（最关键）
2. **安慰剂检验** —— 假政策时点 / 假处理组，应不显著
3. **PSM-DID 联合** —— 先匹配再 DID，控制处理组和对照组在政策前的可观测差异
4. **排除混淆政策** —— 确认同期没有其他政策影响样本
5. **多种因变量度量** —— 换 1-2 种 Y 的度量方式

#### 常见坑 + 审稿人质疑

| 坑 | 表现 | 处理 |
|---|---|---|
| **平行趋势不满足** | 政策前处理组 / 对照组本就在分化 | 改用合成控制 / 找新对照组 |
| 处理时点交错（staggered DID） | 不同公司不同时间被处理 | 用 Goodman-Bacon (2021) / Callaway-Sant'Anna |
| SUTVA 违背 | 处理组影响对照组（溢出效应） | 排除溢出可能或使用空间 DID |
| 选择性进入处理组 | 公司主动进入而非随机 | PSM-DID |

**最近的 DID 文献革命**（2021 年起，**重要**）：

- **传统两期 DID**（一次性政策时点）依然用 `reghdfe` 加 `did` 交乘项
- **多期/交错 DID**（不同公司不同时间被处理）— 传统 TWFE 估计存在偏误，现在主流方法：
  - Goodman-Bacon (2021) 分解
  - Callaway & Sant'Anna (2021) 估计量
  - Sun & Abraham (2021) 估计量
  - de Chaisemartin & D'Haultfœuille (2020)
  - Stata 命令：`csdid`, `eventstudyinteract`, `did_imputation`

**审稿人最爱问**：

1. "**平行趋势满足吗？**" → **必须**画图 + 系数表
2. "**为什么这个时点是好的政策冲击？**" → 论证政策外生性
3. "**对照组怎么选的？**" → 选择标准要可辩护
4. "**有没有同期混淆政策？**" → 列表说明

#### 财税场景实例

**研究问题**：营改增对企业税负的影响

```stata
* 处理组：2012 年起被纳入营改增的服务业（如交通运输、电信）
* 对照组：未被纳入的行业
* 政策时点：2012 / 2013 / 2014（分批实施）

gen treat = (industry == "交通运输" | industry == "电信" | ...)
gen post = (year >= 2012)
gen did = treat * post

reghdfe ETR did Size Lev ROA Loss Growth ///
    , absorb(id year) cluster(id)

* 平行趋势检验、安慰剂检验……
```

**经典文献**：
- 王跃堂、王亮亮、贡彩萍（2010）经济研究：所得税改革
- 林志帆、刘诗源（2022）经济研究：研发加计扣除

### 3.4 倾向得分匹配（PSM）

#### 核心思路

DID 解决「政策冲击 + 时间变化」的因果识别，PSM 解决「**处理组和对照组本来就不一样**」的选择性偏误。先用一组协变量估计每家公司「被处理的概率（倾向得分）」，再把处理组每家公司**匹配**到一家或多家倾向得分相似的对照组公司，把这群"看上去差不多的"作为新样本再回归。

#### 适用场景

- 处理与被处理不是随机分配（**财税审场景几乎都是这样**）
- 需要构造可比对照组：股权激励研究 / 审计师变更研究 / IPO 选择 / 海外 ipo / "走出去"战略
- **PSM-DID 联合使用**——这是当前财税顶刊的标配（先 PSM 匹配，再用匹配样本做 DID）

#### 模型设定

**两步走**：

```
第一步：估计倾向得分
P(Treat = 1 | X) = Φ(γ₀ + γ₁·X₁ + γ₂·X₂ + ...)
                 ↑ Logit 或 Probit

第二步：匹配 + 估计平均处理效应（ATT）
ATT = E[Y(1) - Y(0) | Treat = 1]
    = (1/N_T) · Σ [Y_i^T - Σ w_ij · Y_j^C]
                       匹配权重
```

#### Stata 命令模板

```stata
* 0. 准备：定义协变量（用于估倾向得分）
global xvars "Size Lev ROA Growth Age Big4 SOE BoardSize Indep"

* 1. 估计倾向得分
logit treat $xvars i.year i.industry
predict pscore, pr

* 2. 检查共同支撑域（common support）
* 看处理组和对照组的 pscore 分布是否重叠
twoway (kdensity pscore if treat==1) (kdensity pscore if treat==0), ///
    legend(label(1 "处理组") label(2 "对照组"))

* 3. 多种匹配方法（写论文时多种并用做稳健性）

* 3.1 一对一最近邻匹配
psmatch2 treat $xvars, n(1) common ate

* 3.2 一对四最近邻匹配（更稳健）
psmatch2 treat $xvars, n(4) common ate

* 3.3 卡尺匹配（设定容差）
psmatch2 treat $xvars, n(1) caliper(0.01) common ate

* 3.4 核匹配（用所有对照组加权）
psmatch2 treat $xvars, kernel common ate

* 3.5 半径匹配
psmatch2 treat $xvars, radius caliper(0.05) common ate

* 4. 平衡性检验（核心）
* 匹配后处理组与对照组在协变量上应无显著差异
pstest $xvars, both graph

* 5. PSM-DID（推荐：先匹配再 DID）
* 5.1 提取匹配权重
psmatch2 treat $xvars, n(4) common
* 自动生成 _weight 变量

* 5.2 用匹配样本做 DID
reghdfe Y did Controls [pweight=_weight], absorb(id year) cluster(id)

* 6. PSM 替代命令（teffects 系列）
teffects psmatch (Y) (treat $xvars), atet nn(4)
```

#### 平衡性检验：必看的两组数据

匹配前 vs 匹配后，所有协变量在处理组和对照组应该「**T 检验不显著、标准化偏差 |%bias| < 10%**」：

```
变量    匹配前(差异%) 匹配后(差异%)  匹配后T检验p值
Size      35.2*         3.1           0.32
Lev       28.4*         5.7           0.18
ROA       22.1*         2.4           0.65
...                                   都不显著 ✓
```

不通过 → 重新设定协变量 / 换匹配方法 / 缩小卡尺

#### 常见坑 + 审稿人质疑

| 坑 | 处理 |
|---|---|
| 共同支撑域窄（极少处理组找得到对照） | 删掉 pscore 极端值；或换匹配变量 |
| 匹配变量选择不当 | 选 X 必须是**政策前**就观察到的，不能用 Y 或政策后的变量 |
| 隐藏偏误（unobservables） | PSM 只能控制可观测因素，**不能解决不可观测异质性** |
| 单一匹配方法不稳健 | 至少报告 3 种匹配方法的结果 |

**审稿人质疑**：

1. "**为什么这些协变量？**" → 文献依据 + 排除政策后变量
2. "**平衡性检验过了吗？**" → 必须报 |%bias| < 10
3. "**PSM 不能解决不可观测因素影响，怎么办？**" → 加 PSM-DID + Rosenbaum 敏感性检验

#### 财税场景实例

**研究问题**：股权激励是否提高公司创新（PSM-DID）

```stata
* 处理组：实施股权激励的公司
* 对照组：没实施的公司

* 第一步：用政策实施前一年的特征估倾向得分
logit treat Size Lev ROA RD_Intensity Age SOE BoardSize Indep ///
    if year == 2014                  // 政策前一年

predict pscore, pr

* 第二步：1:4 匹配
psmatch2 treat Size Lev ROA RD_Intensity Age SOE BoardSize Indep ///
    if year == 2014, n(4) common

* 第三步：用匹配样本做 DID
reghdfe ln_patent did Controls [pweight=_weight] ///
    , absorb(id year) cluster(id)
```

**经典文献**：
- 田轩 & 孟清扬 (2018) 南开管理评论：股权激励与创新
- 谢德仁等 (2018) 管理世界

### 3.5 工具变量（IV）

#### 核心思路

OLS 因 X 内生（与残差相关）导致系数有偏。**找一个变量 Z**，满足：

1. **相关性**：Z 与 X 相关（强工具）
2. **外生性**：Z 通过 X 才能影响 Y，与 ε 无关（排除性约束）

用 Z 来"洗"X 的内生部分，再回归。这就是「**两阶段最小二乘**（2SLS）」。

#### 适用场景（**财税审论文最难的部分**）

- 反向因果（盈余管理 ↔ 公司治理质量）
- 遗漏变量（管理层能力影响一切）
- 测量误差（治理变量代理不准）
- 任何审稿人会问"内生性"的研究

#### 模型设定

**两阶段方程**：

```
第一阶段：X = π₀ + π₁·Z + π₂·Controls + u    （Z 预测 X）
第二阶段：Y = β₀ + β₁·X̂ + β₂·Controls + ε    （用 X̂ 而不是 X）
```

X̂ 是第一阶段 X 的拟合值。`β₁` 就是 IV 估计量。

#### Stata 命令模板

```stata
* 1. 标准 2SLS
ivregress 2sls Y (X = Z) Controls i.year i.industry, robust cluster(id)

* 2. 多个工具变量
ivregress 2sls Y (X = Z1 Z2) Controls, robust cluster(id)

* 3. 检验三件套（缺一不可，必报告）
* 3.1 弱工具变量检验（一阶段 F 统计量）
estat firststage
* F > 10 → 强工具
* F < 10 → 弱工具，IV 估计可能比 OLS 更糟（Stock-Yogo）

* 3.2 不可识别检验（Kleibergen-Paap LM）
* 拒绝原假设（p<0.05）= 工具变量与内生变量相关

* 3.3 过度识别检验（Hansen J / Sargan）
* 当工具数 > 内生变量数时
estat overid
* 不拒绝原假设（p>0.10）= 工具外生

* 3.4 内生性检验（Hausman / Durbin-Wu-Hausman）
estat endogenous
* 拒绝 → X 确实内生，IV 比 OLS 更优
* 不拒绝 → X 可能外生，OLS 已够用

* 4. 三件套一起的好用命令：xtivreg2 / ivreg2
ivreg2 Y (X = Z) Controls i.year i.industry, robust cluster(id) first endog(X)
```

#### 财税审常用工具变量类型

| 类型 | 案例 |
|---|---|
| **历史 / 制度变量** | 用 1840 年是否被开埠作为现代市场化程度的工具 |
| **政策外生冲击** | 用沪港通名单作为投资者关注的工具 |
| **同行业-同地区均值** | 用同行业其他公司的 X 均值作为本公司 X 的工具（Bellemare 排除性争议） |
| **滞后期** | 用 X_(t-2) 作为 X_t 的工具（弱工具风险） |
| **自然 / 地理变量** | 用降雨量作为农业相关变量的工具 |
| **第三方评级** | 用穆迪评级方法变化作为评级的工具 |

#### 常见坑 + 审稿人质疑

| 坑 | 表现 | 处理 |
|---|---|---|
| **弱工具变量** | 一阶段 F < 10 | 找新工具 / LIML 估计（弱识别稳健） |
| **排除性约束被质疑** | "Z 难道只能通过 X 影响 Y？" | 文献+经济直觉论证 / 多个工具的 J 检验 |
| **工具不可识别** | KP rk LM 不拒绝原假设 | 工具失效，重选 |
| **同行业均值工具** | Bellemare-Masaki-Pepinsky (2017) 批评 | 慎用，加敏感性分析 |

**审稿人质疑（IV 论文必经折磨）**：

1. "**为什么 Z 满足排除性约束？**" → 这是 IV 文章被毙的最常见原因
2. "**一阶段 F 统计量多少？**" → 必须 > 10，建议 > 20
3. "**用 LIML 估计结果一致吗？**" → 弱工具下 LIML 比 2SLS 偏误小
4. "**reduced form（Z 直接对 Y 回归）显著吗？**" → 应该显著

#### 财税场景实例

**研究问题**：分析师跟踪是否提高股价信息含量（防止反向因果用 IV）

```stata
* 内生变量：分析师跟踪人数 X
* 工具变量 Z：到最近的券商总部距离（地理外生，影响分析师跟踪成本，排除性较强）

ivreg2 Stock_Sync (Analysts = Distance_Brokerage) Size Lev ROA Growth ///
    i.year i.industry, robust cluster(id) first endog(Analysts)

* 必报告：
* - First stage F (>10)
* - KP rk LM p-value (<0.05 拒绝)
* - Sargan/Hansen J p-value (>0.10 不拒绝过度识别)
* - Endogeneity test (拒绝 → IV 优于 OLS)
```

**经典文献**：
- Yu (2008) JFE 用券商分析师裁员作为分析师跟踪的工具变量
- 国内：醋卫华、李培功（2012）金融研究

### 3.6 Heckman 两阶段（自选择问题）

#### 核心思路

样本本身**不是随机抽取的**——只有"选择某种行为"的公司才能观察到 Y。这导致 OLS 系数有偏。Heckman 用两阶段方法，第一阶段估"被选择的概率"，第二阶段加入逆米尔斯比率（Inverse Mills Ratio, IMR）作为额外的控制变量，校正自选择偏误。

#### 适用场景

财税审典型自选择问题：

- 审计师变更研究（只有变更的公司才能观测变更后表现）
- 海外上市决策（只有 IPO 的公司才能算 IPO 抑价）
- 并购决策（只有发起并购的公司才能算并购回报）
- 高管股权激励（只有授予的公司才能观测激励效果）

#### 模型设定

**两阶段方程**：

```
第一阶段（选择方程）：Probit
    Pr(Selected_i = 1 | Z) = Φ(γ₀ + γ·Z_i)
    → 计算 IMR：λ_i = φ(γ·Z) / Φ(γ·Z)

第二阶段（结果方程）：
    Y_i = β·X_i + θ·λ_i + ε_i
    （仅对被选样本：Selected = 1）
```

`λ_i`（IMR）就是用第一阶段估出的选择概率算出来的，加进第二阶段后吸收掉自选择偏误。

#### 关键约束：「排除性变量」

第一阶段的 Z 中，**至少要有 1 个变量不出现在第二阶段的 X 中**。否则 Heckman 模型识别不了（与 OLS 等价）。

#### Stata 命令模板

```stata
* 1. heckman 命令（标准）
heckman Y X Controls, select(Selected = Z X Controls)
* select() 是第一阶段，Z 是排除性变量（影响选择不直接影响 Y）

* 2. 两步法（更透明）
* 第一阶段：Probit
probit Selected Z X Controls
predict xb, xb
gen IMR = normalden(xb) / normal(xb)

* 第二阶段：含 IMR 的 OLS（仅选择样本）
reg Y X Controls IMR if Selected == 1, robust

* 3. 报告 IMR 系数（θ）
* θ 显著 → 自选择确实存在，Heckman 校正有效
* θ 不显著 → 自选择不严重，OLS 可能已够
```

#### Heckman vs PSM 区别（写论文时常被问）

| | Heckman | PSM |
|---|---|---|
| 解决的问题 | 不可观测因素引发的自选择 | 可观测协变量分布不平衡 |
| 假设 | 选择方程残差与结果方程残差正态联合 | 处理组的潜在结果分布与对照组类似 |
| 必备条件 | 排除性变量（外生） | 共同支撑域 |
| 适用方向 | 校正系数偏误 | 构造可比对照组 |
| 当前流行度 | 下降（排除性争议大） | 上升（特别是 PSM-DID） |

#### 常见坑 + 审稿人质疑

| 坑 | 处理 |
|---|---|
| 排除性变量找不到 | Heckman 失效；考虑 PSM 或 IV |
| 排除性变量弱 | 同弱工具问题，估计不稳定 |
| 残差非正态 | Heckman 假设违背；半参数 Heckman / PSM |
| IMR 不显著但仍报告 Heckman 结果 | 实际自选择不严重，宜回到 OLS |

**审稿人质疑**：
1. "**排除性变量是什么？为什么它满足排除性？**"
2. "**IMR 显著吗？**"——如不显著说明自选择不严重，应回 OLS
3. "**两步法和 ML 法结果一致吗？**" → 都跑一遍

#### 财税场景实例

**研究问题**：自愿性披露非财务信息对融资成本的影响

```stata
* 自选择：公司可能选择性披露——只有"想披露"的公司才披露
* 排除性变量 Z：是否被纳入媒体压力指数（影响披露动机，不直接影响融资成本）

```

### 3.7 事件研究法（Event Study）

#### 核心思路

某事件（公告、政策、变更）发生后**短期内**，市场如何反应？通过比较「事件窗口期实际收益率」与「正常情况下的预期收益率」之差（**异常收益率 AR**）累计而成（**累计异常收益率 CAR**），衡量事件的市场效应。

#### 适用场景（财税审超常用）

- 审计师变更 / 出具非标意见的市场反应
- KAM 披露的市场反应
- 财务重述公告 / 监管处罚公告的市场反应
- 分拆上市 / 并购 / 定增公告的市场反应
- 高管变更 / 离职 / 被立案的市场反应
- 会计准则变更的市场反应

#### 模型设定

**步骤一**：定义事件日 T₀，估计期 [T₀-130, T₀-11]，事件窗口期（多个窗口对比）

| 窗口 | 长度 | 用途 |
|---|---|---|
| [-1, 1] | 短窗口 | 即时反应 |
| [-3, 3] | 中窗口 | 含信息泄露和后续消化 |
| [-10, 10] | 长窗口 | 长期反应 + 信息渗透 |

**步骤二**：估计正常收益率（市场模型）

```
R_it = α_i + β_i · R_mt + ε_it    （估计期数据）
```

`R_it` = 公司 i 在 t 日实际收益率
`R_mt` = 市场指数在 t 日收益率（沪深 300 / 中证 800）

**步骤三**：计算异常收益率与累计异常收益率

```
AR_it = R_it - (α̂_i + β̂_i · R_mt)
CAR_i[T1, T2] = Σ_{t=T1}^{T2} AR_it
```

**步骤四**：检验

```
H₀: 平均 CAR = 0（事件无影响）
检验统计量：t = mean(CAR) / [std(CAR) / √N]
```

**长期事件研究**用 BHAR（Buy-and-Hold Abnormal Return）：

```
BHAR_i[T1,T2] = ∏(1+R_it) - ∏(1+R_mt)
              t=T1..T2     t=T1..T2
```

#### Stata 命令模板

```stata
* 直接用 eventstudy2 / estudy 命令（用户写包）
ssc install eventstudy2

* 1. 准备数据：events 表（公司代码 + 事件日）
* events.dta:  id  event_date

* 2. 跑事件研究
eventstudy2 _id _date using events, ///
    estimator(market_model) ///
    estimation_window(120) ///
    event_window(-10 10) ///
    market_index(market_ret) ///
    lower_gap(11)            // 估计期与事件窗口隔 10 天

* 3. 输出 AR / CAR / t 统计量

* ──── 手动写法（更透明）────
* a. 估计期回归市场模型
foreach company of local sample_ids {
    quietly reg R_it market_ret if id == `company' & est_window == 1
    predict AR_temp if id == `company' & event_window == 1, residuals
    replace AR = AR_temp if id == `company'
}

* b. 计算窗口期 CAR
gen CAR = .
forvalues w = -1/1 {
    bysort id: egen CAR_temp = total(AR) if event_day >= -1 & event_day <= 1
    replace CAR = CAR_temp
}

* c. 检验平均 CAR ≠ 0
ttest CAR == 0
```

#### 横截面回归：CAR = f(公司特征)

事件研究后的标准动作是 **横截面回归 CAR**，看哪些因素让市场反应更强 / 更弱：

```
CAR_i = α + β·X_i + γ·Controls_i + ε_i
```

`X` 可以是公司治理变量、信息质量代理、媒体关注程度等。

```stata
reg CAR_neg2_2 X1 X2 Size Lev ROA, robust
```

#### 常见坑 + 审稿人质疑

| 坑 | 处理 |
|---|---|
| 估计期与事件期重叠 | 估计期与事件窗口至少隔 10 个交易日 |
| 多事件污染 | 排除窗口期内有其他公告的样本 |
| 横截面相关（同行业同期事件） | 用 Patell / BMP / Kolari-Pynnonen 检验 |
| 长期 BHAR 偏度 | 用对照组匹配（控制规模、BM）+ 自助法 |
| 事件日定义不准 | 用公告时间（盘前/盘中/盘后）+ T+1 调整 |

**审稿人质疑**：
1. "**事件日是怎么确定的？**"
2. "**为什么这个窗口长度？**"
3. "**控制了样本期内其他重大公告吗？**"
4. "**多重事件如何处理？**"

#### 财税场景实例（已在 MPAcc 模板中提及，此处补完）

**研究问题**：注册会计师被监管处罚对公司股价的影响

```
事件日：处罚决定公告日
处理组：被处罚的注会签字客户公司
窗口：[-1, 1] [-3, 3] [-10, 10]
预期：CAR 为负且显著
进一步：横截面回归 CAR = f(治理质量、信息透明度)
```

**经典文献**：
- Brown & Warner (1985) JFE：事件研究方法奠基
- 国内：陈关亭 (2014) 会计研究

### 3.8 Logit / Probit（二元选择）

#### 核心思路

因变量是 0/1 二元变量时，OLS 不合适（线性概率模型预测可能 < 0 或 > 1，且残差异方差）。Logit 假设残差 logistic 分布，Probit 假设标准正态分布。**财税审论文里 Logit 用得更多**（系数解释更直接：odds ratio）。

#### 适用场景

- 是否被出具非标审计意见（1=非标，0=标准）
- 是否变更审计师（1=变更，0=不变更）
- 是否发生财务重述
- 是否被证监会立案 / 处罚
- 是否实施股权激励
- 是否发起并购 / 增发 / 分拆
- 是否避税激进（基于 ETR 阈值）

#### 模型设定

```
Logit:    Pr(Y=1|X) = 1 / (1 + exp(-(β₀ + β·X)))
Probit:   Pr(Y=1|X) = Φ(β₀ + β·X)
```

**注意**：Logit/Probit 的**系数本身不能直接解读**，要看：
- **Odds Ratio**（Logit 才有，OR = exp(β)）
- **边际效应**（marginal effects）：dY/dX 在某点的值

#### Stata 命令模板

```stata
* 1. Logit / Probit
logit  Y X Controls i.year i.industry, cluster(id)
probit Y X Controls i.year i.industry, cluster(id)

* 2. 报告 Odds Ratio
logit  Y X Controls, or

* 3. 边际效应（最重要的解读方式）
margins, dydx(*) atmeans         // 在均值处的边际效应
margins, dydx(*)                  // 平均边际效应（AME，更推荐）
marginsplot

* 4. 拟合优度
estat classification             // 分类正确率
lroc                             // ROC 曲线
estat gof                        // Hosmer-Lemeshow 检验

* 5. 固定效应 Logit（短面板要小心，"incidental parameters" 问题）
xtlogit Y X Controls, fe         // 条件 Logit
xtlogit Y X Controls, re

* 6. Multinomial Logit（多类别因变量）
mlogit Y X Controls, baseoutcome(0)
```

#### 常见坑 + 审稿人质疑

| 坑 | 处理 |
|---|---|
| 直接解读 logit 系数 | 报告边际效应，不报告原始系数 |
| 罕见事件偏误 | 当 Y=1 占比 < 5%，用 Firth Logit / 罕见事件 Logit |
| 预测变量分离（separation） | 数据中某变量完美预测 Y，logit 不收敛 |
| 完美预测会让某些观测被自动删除 | xtlogit fe 时大量样本被剔除 |

**审稿人质疑**：
1. "**Logit 还是 Probit？**" → 通常结果差异很小，惯例选 Logit
2. "**报告了边际效应吗？**" → 必须
3. "**为什么不用线性概率模型（LPM）？**" → 大样本时 LPM + cluster 也常用，因为系数好解释

### 3.9 中介效应与调节效应

#### 中介效应（Mediation）

**问题**：X 通过什么**机制**影响 Y？是否通过中介变量 M？

**Baron-Kenny 三步法**（虽然有质疑，依然是财税审论文最常报的方法）：

```
Step 1:  Y = α + c·X + ε              要求：c 显著
Step 2:  M = β + a·X + ε              要求：a 显著
Step 3:  Y = α' + c'·X + b·M + ε      要求：b 显著
```

**判断**：

- 若 Step 3 中 c' 仍显著但小于 c → **部分中介**
- 若 Step 3 中 c' 不显著 → **完全中介**
- 中介效应大小 = a × b

**必做的稳健性**：**Bootstrap 检验** 中介效应（推荐 5000 次重复）

```stata
* Stata
ssc install med4way

* 标准三步法
reg Y X Controls
reg M X Controls
reg Y X M Controls

* Bootstrap 中介效应（更严谨）
bootstrap r(ind_eff) = (_b[X]*_b[M]/_b[X]), reps(5000): ///
    sgmediation Y, mv(M) iv(X) cv(Controls)
```

#### 调节效应（Moderation）

**问题**：X 对 Y 的影响在不同条件下是否不同？调节变量 W 改变了什么？

**模型**：

```
Y = α + β₁·X + β₂·W + β₃·X×W + γ·Controls + ε
```

`β₃` 显著 → 调节效应存在
`β₃` 符号 → 强化（同号）/ 削弱（异号）

```stata
* 1. 调节效应
gen XW = X * W                            // 交乘项
reghdfe Y X W XW Controls, absorb(year industry) cluster(id)

* 推荐：变量先中心化，避免共线性
egen X_c = std(X)
egen W_c = std(W)
gen XW_c = X_c * W_c
reghdfe Y X_c W_c XW_c Controls, absorb(year industry) cluster(id)

* 2. 简单斜率分析（写论文常用）
* 在 W 的均值 / ±1SD 处看 X 对 Y 的边际效应
margins, dydx(X) at(W = (-1 0 1))
marginsplot
```

#### 常见坑 + 审稿人质疑

| 坑 | 处理 |
|---|---|
| 中介变量内生 | M 也可能被 Y 影响——选择**滞后期** M / 用 IV |
| 中介路径过多 | 可能有更深的中介链——文献支持 |
| 多重共线性（交乘） | 自变量先中心化 |
| 完全中介结论过强 | 报告 Bootstrap 置信区间，避免硬下结论 |

**审稿人质疑**：
1. "**中介机制的理论依据是什么？**" → 必须有理论
2. "**Bootstrap 检验做了吗？**"
3. "**调节效应的简单斜率图呢？**"

#### 财税场景实例

**研究问题**：高管激励 → ESG 表现（中介：研发投入）

```
Step 1:  ESG = α + c·Equity_Inc + ε         （总效应）
Step 2:  RD  = β + a·Equity_Inc + ε         （X→M）
Step 3:  ESG = α' + c'·Equity_Inc + b·RD + ε（直接 + 中介）

机制：股权激励 → 增加研发投入 → 提升 ESG 表现
```

**研究问题（调节）**：审计师独立性 → 盈余管理（调节：客户重要性）

```
|DA| = α + β₁·Indep + β₂·Importance + β₃·Indep×Importance + ε
β₃ 显著 → 客户越重要，独立性的抑制作用越弱（捕获效应）
```

### 3.10 稳健性检验「四件套」

财税审论文如果只跑一次主回归就提交，**99% 被拒**。审稿人最爱说的话：「**结果稳健吗？**」

#### 四件套

**1. 替换关键变量度量**

- 因变量：|DA| → 用 DD 模型 |DA| / 业绩匹配 |DA|
- 自变量：审计师变更 → 用强制变更 / 自愿变更分别测
- 调节：连续变量 → 哑变量

**2. 改变样本范围**

- 排除特殊行业（金融、地产、ST、退市股）
- 排除极端值（缩尾 / 截尾，常用 1% / 99%）
- 分行业 / 分时期分别回归
- 使用平衡面板 vs 非平衡面板

**3. 改变模型设定**

- 加更多控制变量（拟合更彻底）
- 不同固定效应组合（公司 / 行业-年度 / 省份-年度）
- 不同标准误聚类方式（公司 / 行业 / 公司+年度）
- 改用 Logit / Probit 替代 OLS（如果 Y 可二值化）

**4. 处理内生性**

- IV 估计
- PSM 匹配
- DID（如有政策冲击）
- Heckman（如有自选择）
- 系统 GMM（动态面板）
- Lewbel (2012) 方法（无外部 IV）

#### 写论文时的稳健性章节模板

```
5. 稳健性检验

5.1 替换关键变量度量
（讲明换了什么，结果如何，附表）

5.2 缩小样本范围
（去除金融行业 / 缩尾处理 / 平衡面板）

5.3 改变模型设定
（不同 FE / 不同聚类）

5.4 内生性处理
5.4.1 工具变量法
5.4.2 PSM 匹配
5.4.3 排除替代解释
```

每条至少配 1 张回归表。**主结论符号、显著性都不变 → 稳健**。

#### Stata 命令快速参考

```stata
* 缩尾处理（1%-99%）
winsor2 Y X Controls, replace cuts(1 99)

* 排除特殊样本
drop if industry == "金融"
drop if ST == 1

* 不同聚类方式
reghdfe Y X Controls, absorb(id year) cluster(id)
reghdfe Y X Controls, absorb(id year) cluster(industry)
reghdfe Y X Controls, absorb(id year) cluster(id year)    // 双向聚类

* 替换 FE
reghdfe Y X Controls, absorb(id year) cluster(id)
reghdfe Y X Controls, absorb(id year industry#year) cluster(id)
reghdfe Y X Controls, absorb(id year province#year) cluster(id)
```

## 第六章：因变量度量库（财税独有）

写实证论文时，「**因变量怎么算出来的**」是审稿人最较真的地方。每个度量都有对应的经典文献依据，乱用会被拒。

### 6.1 盈余管理度量

#### 6.1.1 修正 Jones 模型（Dechow et al., 1995）

**最常用的盈余管理度量。三步法**：

**第一步**：分行业-年度估计参数

```
TA_it / A_(i,t-1) = α₁·(1/A_(i,t-1)) + α₂·(ΔREV_it/A_(i,t-1))
                                     + α₃·(PPE_it/A_(i,t-1)) + ε
```

- `TA_it` = 总应计 = 净利润 - 经营现金流量净额
- `A_(i,t-1)` = 上年末总资产
- `ΔREV` = 营业收入变动
- `PPE` = 固定资产原值

**第二步**：用估出的 α₁ α₂ α₃ 计算"正常应计"（NDA）

```
NDA_it = α̂₁·(1/A_(i,t-1)) + α̂₂·((ΔREV_it - ΔAR_it)/A_(i,t-1))
                          + α̂₃·(PPE_it/A_(i,t-1))
```

注意：第二步用 `ΔREV - ΔAR`（应收账款变动），不是直接 `ΔREV`——这就是「修正」之处，剔除掉信用销售操纵。

**第三步**：操控性应计 = 总应计 - 正常应计

```
DA_it = TA_it/A_(i,t-1) - NDA_it
```

- `|DA|` 越大 → 盈余管理程度越大
- DA > 0 → 调高利润；DA < 0 → 调低利润

```stata
* Stata 实现修正 Jones（按行业-年度分组估计）
egen industry_year = group(industry year)

statsby _b _se r2=e(r2), by(industry_year): ///
    reg TA_A_lag inv_A_lag dRev_A_lag PPE_A_lag

* 合并系数后计算 DA
```

#### 6.1.2 业绩匹配 Jones 模型（Kothari et al., 2005）

在修正 Jones 基础上加一项 `ROA_(i,t-1)`，进一步控制业绩对应计的影响：

```
TA/A_(t-1) = α₁·(1/A_(t-1)) + α₂·(ΔREV-ΔAR)/A_(t-1)
           + α₃·PPE/A_(t-1) + α₄·ROA_(t-1) + ε
```

#### 6.1.3 DD 应计模型（Dechow & Dichev, 2002）

更接近"应计质量"，常用作稳健性：

```
ΔWC_t = β₀ + β₁·CFO_(t-1) + β₂·CFO_t + β₃·CFO_(t+1) + ε
```

`ΔWC` = 营运资本变动；`σ(ε)` 越大，应计质量越差（盈余管理空间越大）。

#### 6.1.4 真实盈余管理（Roychowdhury, 2006）

不是通过会计估计，而是通过**真实业务安排**操纵盈余：

- 异常生产成本 R_PROD（过度生产降单位成本）
- 异常酌量费用 R_DISX（削减研发广告培训）
- 异常经营现金流 R_CFO

```
PROD_t/A_(t-1) = α₁·(1/A_(t-1)) + α₂·SALES_t/A_(t-1)
              + α₃·ΔSALES_t/A_(t-1) + α₄·ΔSALES_(t-1)/A_(t-1) + ε

R_PROD = 残差（绝对值越大 → 真实盈余管理程度越大）
```

#### 6.1.5 盈余管理度量的稳健性

写论文时**至少用 2-3 种度量**做对照：

```
|DA|_Jones（主回归）
|DA|_PMOM (业绩匹配)（稳健性 1）
|DA|_DD（稳健性 2）
真实盈余管理（稳健性 3）
```

### 6.2 会计稳健性度量

#### 6.2.1 Basu (1997) 模型

**核心思想**：稳健会计要求"坏消息（损失）比好消息（收益）更及时确认"。

```
EPS_it/P_(i,t-1) = β₀ + β₁·DR_it + β₂·R_it + β₃·DR_it·R_it + ε
                                              ↑
                               β₃ 显著为正 = 公司具有稳健性
```

- `R_it` = 当年股票收益率（市场调整）
- `DR_it` = 1 if R_it < 0 else 0（坏消息哑变量）
- `EPS/P` = 盈余与股价比（控制规模）

**β₃ 越大，稳健性越强**。

#### 6.2.2 Khan & Watts (2009) C_Score

把 Basu 模型每年每家公司单独估计，得到公司层面的稳健性度量：

```
C_Score = β₃ = λ₁ + λ₂·SIZE + λ₃·MTB + λ₄·LEV
```

实际操作：分年度跑 Basu 回归 + 公司特征交互，从交互项中提取 C_Score。

#### 6.2.3 Givoly-Hayn 应计稳健性

```
NACC = (净利润 - 经营现金流 + 折旧摊销) / 期初总资产
NACC_avg = 过去 3 年的 NACC 移动平均
```

NACC_avg 越**负** → 稳健性越强。

### 6.3 投资效率（Richardson, 2006）

**两阶段模型**：

**第一阶段**：估计预期投资水平

```
INV_it = β₀ + β₁·Q_(i,t-1) + β₂·Cash_(i,t-1) + β₃·Age_it
       + β₄·Size_(i,t-1) + β₅·Lev_(i,t-1) + β₆·Return_(i,t-1)
       + β₇·INV_(i,t-1) + Year + Industry + ε
```

**第二阶段**：残差就是非效率投资

```
NEI_it = ε_it（残差）

NEI > 0 → 过度投资（OverInv）
NEI < 0 → 投资不足（UnderInv）
|NEI|  → 投资非效率程度
```

```stata
reg INV Q_lag Cash_lag Age Size_lag Lev_lag Return_lag INV_lag i.year i.industry
predict resid, residuals
gen NEI = resid
gen OverInv = resid if resid > 0
gen UnderInv = -resid if resid < 0
```

### 6.4 避税程度

#### 6.4.1 实际税率（ETR）

```
ETR = 所得税费用 / 税前利润

Cash ETR = 实际缴纳所得税 / 税前利润   （更纯净）
长期 ETR = ∑(实际所得税) / ∑(税前利润)，5-10 年
```

**ETR 越低 → 避税程度越大**

筛选规则：剔除 ETR < 0 或 > 1 的样本（异常）。

#### 6.4.2 账税差异（BTD）

```
BTD = 税前会计利润 - 应税所得
应税所得 = 当期所得税费用 / 法定税率
```

**BTD 越大 → 账面利润比税务申报利润高得越多 → 可能避税**

#### 6.4.3 异常账税差异（DD_BTD，Desai-Dharmapala, 2006）

剔除应计部分后的 BTD：

```
BTD_it = β·TA_it + μ_i + ε_it
DD_BTD = μ̂_i + ε̂_it（残差 + 公司平均效应）
```

DD_BTD 反映**有意识的避税安排**，比单纯 BTD 更精准。

### 6.5 信息披露质量

#### 6.5.1 深交所信息披露评级

A / B / C / D 四级（A 最好），转换为有序变量做 Ordered Logit 或转换为哑变量做 Logit。

#### 6.5.2 KV 指数（Kim & Verrecchia, 1994）

```
KV = |return| / |volume|（在公告日前后）
```

KV 越高 → 信息不对称程度越严重 → 披露质量越差。

#### 6.5.3 自建披露指数

参照 Botosan (1997) 方法，对年报某些类别（背景信息、汇总历史结果、关键非财务统计、预测信息、管理层讨论与分析）打分加总。

### 6.6 股价崩盘风险（Hutton et al., 2009）

#### 6.6.1 NCSKEW（负偏度系数）

```
NCSKEW = -[n(n-1)^(3/2) · ΣW³] / [(n-1)(n-2) · (ΣW²)^(3/2)]
```

W = 周特定收益率（剔除大盘和滞后期影响）

#### 6.6.2 DUVOL（负正波动比）

```
DUVOL = log[ ((n_u-1)·Σ_{down}W²) / ((n_d-1)·Σ_{up}W²) ]
```

n_u / n_d = 高于/低于年均收益率的周数

**两个度量值越大 → 崩盘风险越高**。论文里通常两个都报告。

### 6.7 公司治理度量

#### 6.7.1 综合治理指数

巴鲁克学院的 Gov-Score / 国内的 GovIndex：把若干治理指标（独董比例、CEO 兼任、董事会规模、股权制衡度、机构持股、内部交易披露……）打分汇总。

#### 6.7.2 单一治理变量

不做综合指数时，常用的单一变量：

| 维度 | 变量 |
|---|---|
| 董事会 | 独立董事比例、董事会规模、董事长 CEO 兼任 |
| 股权 | 第一大股东持股、股权制衡度（z指数）、机构投资者持股 |
| 高管 | 持股比例、薪酬激励、股权激励 |
| 监事 | 监事会规模、监事独立性 |

### 6.8 ESG 度量

| 度量 | 来源 | 形式 |
|---|---|---|
| MSCI ESG | MSCI | A-G 字母制 |
| 华证 ESG | 华证指数 | A-D 字母制（中国上市公司全覆盖） |

## 第七章：控制变量「标准菜单」

财税审论文里，**几乎每篇都会用到这些控制变量**。新人写论文不知道选什么 → 直接照菜单挑。

### 7.1 公司层面（必备）

| 变量 | 定义 | 缩写 |
|---|---|---|
| 公司规模 | ln(总资产) | Size |
| 资本结构 | 总负债 / 总资产 | Lev |
| 盈利能力 | 净利润 / 总资产 | ROA |
| 成长性 | 营业收入增长率 / Tobin's Q / MTB | Growth / Q / MTB |
| 经营现金流 | 经营现金流量 / 总资产 | OCF |
| 上市年限 | 上市以来年数 | Age |
| 是否亏损 | 净利润 < 0 哑变量 | Loss |
| 固定资产密度 | 固定资产净额 / 总资产 | PPE |
| 无形资产密度 | 无形资产 / 总资产 | Intangible |

**生成命令**（Stata）：

```stata
gen Size = ln(total_assets)
gen Lev = total_liab / total_assets
gen ROA = net_income / total_assets
gen Growth = (revenue - L.revenue) / L.revenue
gen MTB = market_value / book_value
gen OCF = operating_cf / total_assets
gen Age = year - listing_year
gen Loss = (net_income < 0)
gen PPE = ppe_net / total_assets
gen Tobin_Q = (market_cap + total_liab) / total_assets
```

### 7.3 审计层面（审计研究专用）

| 变量 | 定义 |
|---|---|

## 第八章：实证论文写作公式

财税审实证论文的标准结构（**几乎所有顶刊都按这套**）：

### 8.1 引言（Introduction）

**5 段式公式**：

```
第 1 段：问题提出 + 现实背景
第 2 段：现有文献的局限 / 缺口
第 3 段：本文的研究设计简要描述
第 4 段：主要发现 + 数据简介
第 5 段：本文贡献（理论 + 实务 + 方法）
```

**字数**：1500-3000 字。**贡献写 3 条**，不要写 5 条（显得没重点）。

### 8.2 文献综述与假设推导

**结构**：

```
1. 关键变量 X 相关文献综述（一段）
2. 关键变量 Y 相关文献综述（一段）
3. X 与 Y 关系的现有研究 + 缺口（一段）
4. 假设 1 推导（理论A → X → Y）→ H1: ...
5. 假设 2 推导（如有调节/中介，理论B → X×W → Y）→ H2: ...
```

**关键技巧**：每个假设要有**理论支撑 + 反向直觉的提示**（"虽然有的文献认为相反..."），显示思考的全面。

### 8.3 研究设计（Research Design）

**必含 4 个子节**：

#### 8.3.1 样本选择

```
样本期间：2010-2022 年 A 股上市公司
剔除标准：
  - 金融保险行业
  - ST / *ST / 退市
  - 上市当年 / 上市前一年
  - 关键变量缺失
最终样本：N 家公司，M 个公司-年度观测值
```

#### 8.3.2 变量定义

**用表格列出**所有变量（缩写、定义、计算方法、数据来源），1-2 页，**经典做法**。

#### 8.3.3 模型设定

**写出主回归方程**：

```
Y_it = β₀ + β₁·X_it + β₂·Controls_it + Year_FE + Industry_FE + ε_it     (1)

其中：
- Y_it 表示...
- X_it 表示...
- Controls 包括...
- 我们关心 β₁ 的符号和显著性
- 标准误在公司层面聚类
```

#### 8.3.4 描述性统计 + 相关性矩阵（写论文必报）

```
表 X：描述性统计
        N    Mean   Std.   Min   p25   Median   p75   Max
Y      ...
X      ...
...

表 X：相关性矩阵（上三角 Pearson，下三角 Spearman）
```

### 8.4 实证结果（Empirical Results）

**5 部分**：

```
4.1 主回归结果           （表 X：基准回归）
4.2 内生性处理           （IV / PSM / DID）
4.3 机制检验             （中介 / 调节）
4.4 稳健性检验           （4 种以上）
4.5 异质性分析           （分组：产权 / 规模 / 行业）
```

**每张回归表的标配**：

- 列 (1)：纯 OLS（不加控制变量）
- 列 (2)：加控制变量
- 列 (3)：加固定效应
- 列 (4)：完整模型 = 控制变量 + FE + 聚类
- 列 (5)：稳健性 / 子样本

### 8.5 结论与启示

```
1. 研究发现总结（与假设对照）
2. 理论贡献
3. 实务启示（监管 / 公司 / 投资者）
4. 局限与未来展望
```

**字数**：500-1000 字，不要拖。

## 第九章：审稿人「常见质疑」与应对话术

财税审实证论文经常被这几个问题反复打回：

### 9.1 "内生性怎么处理？"

**这是最常见的质疑**。

**应对策略**：

1. 论证主要担心的是**反向因果 / 遗漏变量 / 自选择**中的哪一个
2. 针对性处理：
   - 反向因果 → 滞后期 X / 工具变量 / 政策外生冲击
   - 遗漏变量 → 增加控制 / 公司 FE / 工具变量
   - 自选择 → PSM / Heckman
3. **多种方法结果一致** → 可信

**话术模板**：

> 本文的结果可能受到内生性的干扰，主要包括：（1）反向因果——……；（2）遗漏变量——……；（3）自选择——……。为缓解上述问题，本文采取以下方法：

### 9.2 "控制变量为什么是这些？"

**应对**：

- 引用**至少 3 篇相同领域的经典论文** + "本文借鉴...的做法"
- **基于理论选择**——不是凑数

**话术模板**：

> 借鉴 XX (2019)、XX (2020) 等的做法，本文控制了以下变量……

### 9.3 "稳健性够吗？"

**应对**：保证**至少 4 类稳健性**

- 替换变量度量
- 缩小样本
- 改变模型设定
- 处理内生性

### 9.4 "结果有什么实务意义？"

**应对**：

- 经济意义计算：「X 增加 1 个标准差，Y 平均**变化 X.X%**」
- 与现有文献对比：「我们的结果意味着……」

```stata
* 计算经济意义
sum X
local std_X = r(sd)
local coef = _b[X]
display "X 增加 1SD = " `std_X' " ; Y 平均变化 = " `coef' * `std_X'
```

### 9.5 "样本期太老了"

**应对**：补充最近 1-2 年数据；或论证样本期的特殊性（如政策窗口）。

### 9.6 "为什么没考虑 XX 替代解释？"

**应对**：列出**所有可能的替代解释**，逐一排除：

```
表 X：排除替代解释
替代解释 1：XX → 通过 [方法] 排除
替代解释 2：YY → 通过 [方法] 排除
```

| 错误 | 正确做法 |
|---|---|
| 直接跑一次主回归就提交 | 必加 4 类稳健性（变量 / 样本 / 模型 / 内生） |
| 用因变量当工具变量 | 工具变量必须是 **政策前 / 外生** 的 |
| 系数显著 = 业务意义 | 计算经济意义（标准差换算） |
| 内生性问题"一笔带过" | 至少用 2 种方法处理（IV + PSM 联合） |
| 边写边查文献 | 写文献综述前先列**核心 30 篇**，确保覆盖 |
| 中文论文里完全没引外文文献 | 引用国际顶刊文献，体现宽度 |
| 滥用主流方法（DID / IV / PSM） | 选最适合研究问题的方法，不要为了"高级"而用 |
| 控制变量是 "我看着像就加" | 文献支持 + 政策前观察的 |
| 样本期 < 5 年还用面板 FE | 短面板用 GMM 或 RE |
| 报告 Logit 系数原始值 | 必须报告**边际效应** |
| 平行趋势不画图 | DID 必须画系数图 |
| Heckman 排除性变量编不出 | 不要硬上 Heckman，换 PSM |
| 文献只用 5 年内的 | 经典文献（1980-2010）也要引 |
