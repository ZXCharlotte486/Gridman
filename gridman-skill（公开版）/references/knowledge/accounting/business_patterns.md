# 常见多步业务模式库

> 定位：面对真实序时账时，识别一笔分录处于哪个业务链条的哪一步，判断是否漏步、科目是否用错、时点是否恰当
> 来源：真实企业账务数据提炼（某医药集团+某科技企业，已脱敏）+ CPA 2026《会计》准则口径校正
> 覆盖：费用报销、采购入库、销售出库、薪酬社保、税金、资产、资金划转
> 性质：实操+判断

本文件是「业务链条识别中枢」，回答"这笔分录是什么业务、第几步、对不对"。
准则计量细节（为什么这么记、确认条件、计量公式）不在本文件展开，统一关联到 accounting_core.md / accounting_advanced.md / tax_core.md 主存放点。
本文件只保留三样独有内容：①链条时序 ②识别特征 ③判断钩子（卡点在哪、看什么决定走哪条分支）。

> 章节索引（L2）：一、费用报销类 ｜ 二、采购业务类 ｜ 三、销售业务类 ｜ 四、薪酬社保类 ｜ 五、税金业务类 ｜ 六、资产类 ｜ 七、资金划转类 ｜ 使用规则 ｜ 与 MCP 工具的协同（看真账时）

与 voucher_scan 工具的关系：voucher_scan 用代码机械扫描七大类风险（差旅占比/供应商拆分付款/成本费用失衡/大额整数/周末凭证/红字冲销/现金大额），只负责"标记可疑项"；本文件负责"把标记项放回业务链条判断真伪"。机器筛查，模型研判。涉及风险识别特征处，口径与 voucher_scan 对齐。

## 一、费用报销类

### 1.1 先借款后报销模式

> 类型：示例
> 来源：真实账务提炼 + CAS 基本准则（权责发生制）
> 关联：accounting_core.md - 第一章（权责发生制：费用按发生归属期确认，不按付款时点）

```
第1步：员工借款
  借：其他应收款——员工/备用金
    贷：银行存款/库存现金

第2步：报销冲销（报销金额 = 借款金额）
  借：管理费用/销售费用/研发费用（按费用性质）
    贷：其他应收款——员工/备用金

第2步变体A：报销金额 < 借款金额（退回余款）
  借：管理费用/销售费用（实际报销额）
  借：银行存款/库存现金（退回差额）
    贷：其他应收款——员工/备用金

第2步变体B：报销金额 > 借款金额（补付差额）
  借：管理费用/销售费用（实际报销额）
    贷：其他应收款——员工/备用金（原借款额）
    贷：银行存款/库存现金（补付差额）
```

**识别特征**：
- 摘要含"借款"/"备用金"/"预支"
- 借方为其他应收款-个人
- 后续出现同一人的费用报销冲销

**判断钩子**：
- 看到"其他应收款——员工"挂账，先判断这是链条第1步（借款），不是最终费用。费用尚未确认，不能据此认定当期成本。
- 报销冲销时必须比对报销额与借款额，决定走主分支还是变体A/B。三者借贷结构不同，记错会导致其他应收款挂账不平。
- 期末若"其他应收款——员工"长期挂账未冲销 → 链条停在第1步未完成。⚠️ 可能是借款未及时报销（应催报），也可能是变相占用资金。

⚠️ 风险点：
- 长期挂账的个人借款，税务上可能被认定为"个人借款年度终了未归还且未用于生产经营"→ 视同股东分红征 20% 个税（仅针对股东/投资者，详见 tax_core.md - 个人所得税）。
- 大额整数借款、同一人频繁借款 → 触发 voucher_scan 现金大额/大额整数标记，需研判是否为资金占用。

### 1.2 直接报销模式

> 类型：示例
> 来源：真实账务提炼

```
借：管理费用/销售费用/研发费用
  贷：银行存款/库存现金/其他应付款——个人报销
```

**识别特征**：
- 摘要含"报销"
- 无前置借款步骤
- 部分企业通过"其他应付款-个人报销"过渡（先挂账后付款）

**判断钩子**：
- 贷方直接是银行/现金 → 一步到位，无后续链条。
- 贷方是"其他应付款——个人报销" → 这是先挂账（1.3 模式的第1步），后续必有付款，别误判为已结清。

### 1.3 企业先挂账后付款模式（某医药集团等）

> 类型：示例
> 来源：真实账务提炼（某医药集团记账惯例）
> 关联：accounting_core.md - 第一章（权责发生制：费用在发生时确认，与付款时点分离）

```
第1步：费用发生挂账
  借：销售费用/管理费用/研发费用
    贷：其他应付款——个人报销

第2步：实际付款
  借：其他应付款——个人报销
    贷：银行存款
```

**判断钩子**：
- 费用在第1步（挂账）就已确认计入损益，第2步只是结算，不再影响利润。判断"当期费用"看第1步，不看付款。
- 看到"其他应付款——个人报销"贷方挂账，是链条第1步；借方冲减是第2步。期末余额=未付的已报销款。

⚠️ 风险点：跨期挂账（费用计入本期、迟迟不付）属正常权责发生制；但若大量费用集中在期末挂账、次期红冲 → 可能调节利润，触发 voucher_scan 红字冲销标记。

## 二、采购业务类

### 2.1 标准采购（发票与入库同步）

> 类型：示例
> 来源：真实账务提炼 + CAS 1 存货
> 关联：accounting_core.md - 第二章 存货（初始计量：成本=采购成本+加工成本+其他成本）

```
借：原材料/库存商品
借：应交税费——应交增值税（进项税额）
  贷：应付账款——供应商
```

**判断钩子**：发票与货物同步到达是最简情形，一步完成。若两者不同步 → 转入 2.2 暂估或 2.3 单独录进项模式。判断走哪条，看"票"和"货"哪个先到。

### 2.2 暂估入库模式（先到货后到票）

> 类型：示例
> 来源：真实账务提炼 + CAS 1 存货
> 关联：accounting_core.md - 第二章 存货；增值税进项确认详见 tax_core.md - 增值税（进项税额抵扣凭证）

```
第1步：货到票未到，暂估入库
  借：原材料/库存商品
    贷：应付账款——暂估应付货款

第2步：次月初红冲暂估
  借：原材料/库存商品（红字）
    贷：应付账款——暂估应付货款（红字）

第3步：发票到达，正式入账
  借：原材料/库存商品
  借：应交税费——应交增值税（进项税额）
    贷：应付账款——供应商
```

**识别特征**：
- 摘要含"暂估"
- 贷方为"应付账款——暂估应付货款"
- 月初出现红字冲销同一科目

**判断钩子**：
- 看到孤立的红字冲销（第2步），别当异常——先判断上月有无对应暂估入库。是暂估链条的常规动作。
- 暂估时**不确认进项税**（票未到，无抵扣凭证），进项税在第3步发票到达才记。判断进项是否漏记，看链条走到第几步。
- 期末若有暂估余额未转正（停在第1步）→ 货已入、票未到，属正常；但跨年度长期挂暂估 → ⚠️ 关注是否存在无票采购、是否影响成本真实性。

⚠️ 风险点：暂估入库的存货若已领用/销售，其成本已进损益，但进项税未抵扣 → 形成税会时间性差异；长期无票 → 该笔成本企业所得税前扣除存疑（详见 tax_core.md - 企业所得税扣除凭证）。

### 2.3 进项税单独录入模式（某医药集团特有）

> 类型：示例
> 来源：真实账务提炼（某医药集团记账惯例，非通用做法）

```
第1步：取得发票时只录进项税
  借：应交税费——应交增值税（进项税额_经营性）
    贷：应付账款——货款

第2步：实际入库时
  借：库存商品/原材料
    贷：应付账款——暂估应付货款
```

**判断钩子**：这是特定企业惯例，进项税与存货拆成两笔、不同步确认。看到只有进项税、无存货的分录，别误判为漏记存货——是该企业链条第1步。

⚠️ 风险点：进项税和存货不同步确认，属完整性风险点（存货可能虚增或漏记）。审计时需把两步配对核对。此模式为个别企业惯例，遇到其他企业不要套用——以其 entities/ 上下文为准。

### 2.4 预付款采购模式

> 类型：示例
> 来源：真实账务提炼 + CAS 1 存货
> 关联：accounting_core.md - 第二章 存货

```
第1步：预付货款
  借：预付账款——供应商
    贷：银行存款

第2步：收到货物和发票
  借：原材料/库存商品
  借：应交税费——应交增值税（进项税额）
    贷：预付账款——供应商

第2步变体：货款大于预付（补付差额）
  借：原材料/库存商品
  借：应交税费——应交增值税（进项税额）
    贷：预付账款——供应商（预付部分）
    贷：应付账款——供应商（差额部分）
```

**判断钩子**：
- "预付账款"借方挂账是第1步，货未到、不确认存货也不确认进项。判断存货是否到位，看第2步有没有发生。
- ⚠️ 同一供应商分多次预付 → 正常采购节奏；但 voucher_scan 可能标记为"供应商多笔付款/拆分付款"。研判时区分"分期预付（正常）"与"拆分付款规避审批（异常）"——看是否对应真实到货、是否每笔都在审批权限下故意切小。

## 三、销售业务类

### 3.1 标准销售（出库即确认收入）

> 类型：示例
> 来源：真实账务提炼 + CAS 14 收入
> 关联：accounting_core.md - 第十七章 收入确认五步法（控制权转移时点）

```
第1步：确认收入
  借：应收账款——客户
    贷：主营业务收入
    贷：应交税费——应交增值税（销项税额）

第2步：结转成本
  借：主营业务成本
    贷：库存商品
```

**判断钩子**：收入与成本应配比——看到确认收入（第1步）必应有结转成本（第2步）。只确认收入不结转成本 → ⚠️ 漏结转，虚增利润。判断的核心前提是"控制权已转移"，若控制权未转移则不能走本模式，转 3.2 发出商品。

### 3.2 发出商品模式（电商/某医药集团）

> 类型：示例
> 来源：真实账务提炼 + CAS 14 收入
> 关联：accounting_core.md - 第十七章 收入确认五步法（在某一时点确认收入 = 控制权转移时点）

```
第1步：出库发货（控制权未转移）
  借：发出商品
    贷：库存商品——产成品

第2步：确认收入（控制权转移）
  借：应收账款/银行存款
    贷：主营业务收入
    贷：应交税费——应交增值税（销项税额）

第3步：结转成本
  借：主营业务成本
    贷：发出商品
```

**识别特征**：
- 摘要含"XSCKD"（销售出库单）
- 借方为"发出商品"而非直接确认收入
- 电商平台常用（发货≠确认收入）

**判断钩子**（本类核心，最易判错）：
- 关键卡点是**控制权何时转移**。发货（第1步）只是物流位移，控制权未必转移——客户尚未签收/验收、附退货权且退货率不可靠估计时，不确认收入。
- 看到"发出商品"借方挂账，判断为链条第1步，**此时不确认收入**。若有人在第1步就确认了收入 → ⚠️ 提前确认收入（典型收入操纵手法）。
- 增值税纳税义务时点 ≠ 会计收入确认时点：发出商品可能已触发增值税纳税义务（如直接收款方式已收款），但会计收入因控制权未转移而不确认 → 产生税会差异。判断销项税是否该计提，看增值税纳税义务发生时间（详见 tax_core.md - 增值税纳税义务发生时间）。
- 期末"发出商品"余额 = 已发货未确认收入的成本。金额异常大 → ⚠️ 关注是否存在压货、是否该确认而未确认。

### 3.3 预收款销售模式

> 类型：示例
> 来源：真实账务提炼 + CAS 14 收入
> 关联：accounting_core.md - 第十七章 收入确认五步法 + 合同负债

```
第1步：收到预收款
  借：银行存款
    贷：合同负债/预收账款

第2步：发货确认收入
  借：合同负债/预收账款
    贷：主营业务收入
    贷：应交税费——应交增值税（销项税额）

第3步：结转成本
  借：主营业务成本
    贷：库存商品
```

**判断钩子**：
- "合同负债/预收账款"贷方挂账是第1步，**收款不等于确认收入**，履约义务尚未完成。判断当期收入，不看收款看履约（第2步）。
- 执行新收入准则的企业用"合同负债"，老科目为"预收账款"。看到哪个科目可反推企业是否已切换新准则。
- ⚠️ 期末合同负债余额大 → 大量已收款未履约，是未来收入蓄水池，也是履约义务，关注能否按期交付。

### 3.4 分期收款销售（含重大融资成分）

> 类型：示例
> 来源：真实账务提炼 + CAS 14 收入
> 关联：accounting_core.md - 第十七章 收入确认五步法（交易价格中的重大融资成分）

```
第1步：发货确认收入（按现值）
  借：长期应收款（总额）
    贷：主营业务收入（现值）
    贷：未实现融资收益（差额）
    贷：应交税费——应交增值税（销项税额）

第2步：每期收款+确认利息收入
  借：银行存款
    贷：长期应收款
  借：未实现融资收益
    贷：财务费用（冲减）
```

**判断钩子**（链条最长、最易记错）：
- 卡点在"是否含重大融资成分"。分期收款超过一年（通常）→ 收入按**现值**确认，不按合同总额；总额与现值的差额是"未实现融资收益"，分期摊入财务费用（冲减）。
- 看到"长期应收款"+"未实现融资收益"同时出现 → 这是含融资成分的分期销售第1步。若有人按合同总额全额确认收入 → ⚠️ 虚增当期收入（未拆分融资成分）。
- 销项税在第1步按全额（或按纳税义务时点）一次计提，与收入现值确认不同步 → 税会差异。
- 每期（第2步）有两笔：收款冲长期应收款 + 摊销未实现融资收益冲财务费用。漏了第二笔 → 利息收入漏记。

## 四、薪酬社保类

### 4.1 工资计提与发放

> 类型：示例
> 来源：真实账务提炼 + CAS 9 职工薪酬
> 关联：accounting_core.md - 第九章 职工薪酬（短期薪酬：在职工提供服务的会计期间确认）

```
第1步：月末计提工资
  借：管理费用/销售费用/生产成本/研发费用——工资
    贷：应付职工薪酬——工资

第2步：发放工资（代扣个税和社保个人部分）
  借：应付职工薪酬——工资
    贷：银行存款（实发）
    贷：应交税费——应交个人所得税（代扣）
    贷：其他应付款——社保（个人部分）
    贷：其他应付款——公积金（个人部分）
```

**判断钩子**：
- 工资费用在**计提**（第1步、职工提供服务的当月）确认，不在发放时。判断当期人工成本看第1步。次月发放（第2步）只是结算。
- 计提金额（应发）≠ 发放时银行付款（实发）。差额是代扣的个税+社保公积金个人部分。看到实发小于计提别当差错，是代扣。
- 借方科目按部门归属：生产工人→生产成本，车间管理→制造费用，行政→管理费用，销售→销售费用，研发→研发费用。归错科目会扭曲成本与费用结构。

⚠️ 风险点：代扣个税（第2步贷方）必须在次月申报缴纳，挂账不缴 → 税务风险。工资表与个税申报、社保缴费基数应一致，三者不符是稽查重点（详见 tax_risk_indicators.md）。

### 4.2 社保公积金

> 类型：示例
> 来源：真实账务提炼 + CAS 9 职工薪酬
> 关联：accounting_core.md - 第九章 职工薪酬

```
第1步：计提单位部分
  借：管理费用/销售费用/生产成本——社保/公积金
    贷：应付职工薪酬——社保/公积金

第2步：缴纳（单位+个人）
  借：应付职工薪酬——社保/公积金（单位部分）
  借：其他应付款——社保/公积金（个人部分）
    贷：银行存款
```

**判断钩子**：
- 区分两部分：单位部分是企业费用（第1步计提进损益）；个人部分是代扣（来自工资发放时挂的"其他应付款"），不是企业费用。
- 缴纳（第2步）时单位部分冲"应付职工薪酬"、个人部分冲"其他应付款"，两个来源、一笔付出。借方只冲一个科目会不平。

## 五、税金业务类

### 5.1 增值税月末结转

> 类型：示例
> 来源：真实账务提炼 + 增值税法规
> 关联：tax_core.md - 增值税（应纳税额计算、月末结转）

```
月末：进项 < 销项（需缴税）
  借：应交税费——应交增值税（转出未交增值税）
    贷：应交税费——未交增值税

次月缴纳：
  借：应交税费——未交增值税
    贷：银行存款
```

**判断钩子**：
- 月末把"应交增值税"借贷差额转入"未交增值税"，是把当月应缴额固化。判断本月是否需缴税，看销项与进项孰大。
- 进项>销项（留抵）时不做转出，留抵下期。看到没有转出分录，可能是留抵状态，不是漏记。

### 5.2 附加税计提与缴纳

> 类型：示例
> 来源：真实账务提炼 + 税法
> 关联：tax_core.md - 城建税及附加

```
第1步：计提
  借：税金及附加
    贷：应交税费——城建税/教育费附加/地方教育附加

第2步：缴纳
  借：应交税费——城建税/教育费附加/地方教育附加
    贷：银行存款
```

**判断钩子**：附加税以实际缴纳的增值税+消费税为计税依据，跟着流转税走。本月无增值税应缴（留抵）则附加税也无。计提基数错误会连带附加税错。

### 5.3 企业所得税

> 类型：示例
> 来源：真实账务提炼 + 所得税会计
> 关联：accounting_advanced.md - 所得税会计（递延所得税）；tax_core.md - 企业所得税

```
第1步：季度预缴计提
  借：所得税费用
    贷：应交税费——应交企业所得税

第2步：缴纳
  借：应交税费——应交企业所得税
    贷：银行存款

年末汇算清缴：补缴或退税
```

**判断钩子**：
- 这里是当期所得税（应交税费）。若涉及税会暂时性差异，还需确认递延所得税（借/贷"递延所得税资产/负债"对"所得税费用"）——本模式未含，详见 accounting_advanced.md 所得税会计。
- 季度按会计利润预缴是近似，年度汇算清缴做纳税调整后多退少补。看到年末大额补缴/退税，对应汇算清缴调整。

## 六、资产类

### 6.1 固定资产购置

> 类型：示例
> 来源：真实账务提炼 + CAS 4 固定资产
> 关联：accounting_core.md - 第三章 固定资产（初始计量=达到预定可使用状态前的合理必要支出；折旧）

```
第1步：购入
  借：固定资产
  借：应交税费——应交增值税（进项税额）
    贷：银行存款/应付账款

第2步：按月计提折旧
  借：管理费用/制造费用/销售费用——折旧
    贷：累计折旧
```

**判断钩子**：
- 初始成本含达到预定可使用状态前的合理必要支出（运费、安装调试等），不只是买价。看到运费/安装费单独进费用而非资本化 → ⚠️ 可能少计资产成本。
- 折旧从达到预定可使用状态的次月起计提，借方科目按资产用途归属（生产用→制造费用，行政用→管理费用）。归错影响成本费用结构。
- 区分资本性支出与费用性支出：大额支出直接进费用 → ⚠️ 关注是否应资本化（详见 cas_practical_judgments.md - 资本化判断）。

### 6.2 在建工程转固

> 类型：示例
> 来源：真实账务提炼 + CAS 4 固定资产
> 关联：accounting_core.md - 第三章 固定资产；借款费用资本化详见 accounting_core.md - 第十一章 借款费用

```
第1步：工程支出
  借：在建工程
    贷：银行存款/应付账款/原材料/应付职工薪酬

第2步：达到预定可使用状态
  借：固定资产
    贷：在建工程
```

**判断钩子**：
- 卡点是"达到预定可使用状态"的时点——它决定三件事：转固时点、折旧起点、借款费用停止资本化时点。
- "在建工程"长期挂账不转固 → ⚠️ 可能延迟转固以推迟折旧、虚增利润，或推迟停止资本化以多计资产。判断是否该转，看工程是否实质完工可用，不看竣工决算是否办完。
- 专门借款利息在资本化期间计入在建工程成本，达到预定可使用状态后停止（转费用）。

### 6.3 长期待摊费用摊销

> 类型：示例
> 来源：真实账务提炼（某医药集团研发费用分摊案例）
> 关联：accounting_core.md - 第一章（费用按受益期间分摊）

```
第1步：发生支出
  借：长期待摊费用
    贷：银行存款

第2步：按期摊销
  借：管理费用/销售费用/研发费用——长期待摊费用摊销
    贷：长期待摊费用——摊销
```

**识别特征**（某医药集团研发费用分摊案例）：
- 摘要含"分摊"/"摊销"
- 借方为"研发费用_长期待摊费用摊销"
- 贷方为"长期待摊费用_摊销_其他"

**判断钩子**：一次性支出（第1步）先挂"长期待摊费用"，再按受益期分摊（第2步）。看到"摊销"分录是链条第2步，对应前期某笔支出。判断当期费用看分摊额，不看原始支出。摊销期不合理（过长/过短）→ 影响各期损益。

## 七、资金划转类

### 7.1 内部账户划转

> 类型：示例
> 来源：真实账务提炼

```
借：银行存款——A账户
  贷：银行存款——B账户
```

**判断钩子**：同一主体不同账户间划转，不影响损益、不影响资产总额，只是货币资金内部位移。看到这类分录别当收支。⚠️ 但频繁、大额、整数的账户划转 → voucher_scan 可能标记，需研判是否涉及资金占用、体外循环（详见 financial_fraud.md - 资金穿透）。

### 7.2 定期存款

> 类型：示例
> 来源：真实账务提炼
> 关联：accounting_core.md - 第十三章 金融工具（货币资金/其他货币资金列报）

```
存入：
  借：其他货币资金——定期存款
    贷：银行存款

到期收回：
  借：银行存款
    贷：其他货币资金——定期存款
    贷：财务费用——利息收入（红字）或 贷：利息收入
```

**判断钩子**：活期转定期，仍是货币资金内部结构调整，不是投资支出。到期收回时确认利息收入。看到"其他货币资金——定期存款"借方是存入，贷方是收回。

## 使用规则

1. 判断单笔分录业务性质时，先匹配本文件的业务模式，再判断处于链条第几步。
2. 判断"当期损益/成本"时，认准链条中"确认损益的那一步"——多数业务是计提/确认步，不是收付款步。这是本文件最核心的判断价值。
3. 判断科目用得对不对、有没有漏步时，对照链条完整时序：少了某一步 → 可能漏记或链条未完成。
4. 准则依据、确认条件、计量细节不在本文件展开——按各模式的「关联」指引跳转到 accounting_core.md / accounting_advanced.md / tax_core.md 主存放点。本文件不复制准则内容，避免与主存放重复。
5. 企业有 entities/ 上下文文件时，以企业实际记账惯例为准（本文件为通用模式，2.3、1.3 等标注"特有/某企业"的为个别惯例，不可套用到其他企业）。
6. 本文件覆盖最常见业务类型，特殊行业（房地产、金融、建筑、跨境电商等）有额外模式，分别详见对应模块。

## 与 MCP 工具的协同（看真账时）

- voucher_scan（凭证巡检）扫出可疑项后，用本文件把"标记项"放回业务链条研判真伪：
  - "供应商多笔付款"→ 对照 2.4 预付款采购，区分"分期预付（正常）"与"拆分付款规避审批（异常）"
  - "红字冲销"→ 对照 2.2 暂估月初红冲、1.3 挂账冲减，区分"常规链条动作（正常）"与"跨期调利润（异常）"
  - "大额整数/现金大额"→ 对照 1.1 个人借款，区分"正常借款"与"资金占用"
  - "账户划转"→ 对照 7.1，区分"内部归集（正常）"与"体外循环（异常）"
- cutoff_test（截止测试）筛出截止日前后交易后，用 3.1/3.2/3.3 判断收入确认时点是否恰当（控制权是否真在期内转移）。
- audit_adjustment（调整分录）/reclassification（重分类）：本文件帮助判断原始分录错在链条哪一步，据此设计调整分录。

**核心分工：工具机械筛查（扫描、计算、出文件），本文件支撑模型研判（业务链条识别、真伪判断）。对应内审数智化指引"机器筛查、人工研判"原则。**
