> 定位：当正常审计程序因资料缺失/配合不足/条件受限而无法执行时，审计师的决策路径

## 使用说明

> 类型：流程
> 来源：实务方法论
> 置信度：实务经验

**什么是 IF 线：** 当标准实质性程序（`audit_procedures.md` 中的科目循环步骤表）因某个环节卡住而无法正常执行时，审计师需要做的分支决策——"如果 A 拿不到，走 B；B 也不行，走 C；C 还不行，评价对意见的影响"。

**IF 线不是什么：**
- 不是替代检查单（检查单仍然是底稿结构的骨架）
- 不是允许降低审计质量的借口（走替代路径仍然要获取"充分适当"的证据）
- 不是机械执行的公式（分支终点的"够不够"仍然需要职业判断）

**阈值说明：** 本文中出现的百分比阈值（如"覆盖率 80%"）均为实务经验值，非准则硬性规定。具体项目中应结合重要性水平和风险评估确定。

**与 `audit_procedures.md` 的关系：**
- `audit_procedures.md` = "正常路径怎么做"（程序表）
- 本文件 = "正常路径走不通时怎么退"（决策树）
- 两者配合使用：先看程序表确定该做什么，卡住了来这里找退路

> 详见：audit_procedures.md - 第4阶段：风险应对-实质性程序

## 一、存货循环 IF 线

### 阻塞场景 1：收发存明细无法取得

> 类型：判断表
> 来源：CSA 1311 + 实务经验
> 置信度：实务经验

**前提：** 被审计单位无法提供完整的存货收发存汇总表或出入库明细账。

```
IF  期末存货明细表可取得 且 金额合计 = 总账余额
    → 期末快照可用，以此编制审定表
    → 收发过程用替代方法验证（见下方"倒挤法"）
    → 底稿标注"无收发存，仅有期末明细"

ELSE IF  期末明细表可取得，但与总账有差异
    IF  差异 < 实际执行重要性
        → 记录差异，评价原因，继续审计
        → 差异列入未更正错报汇总
    ELSE
        → 停。基础数据不可靠
        → 要求被审计单位解释差异并补充
        → 仍无法解释 → 范围受限事项

ELSE  连期末明细表也没有
    → 🔴 存货科目整体无法审计
    → 仅能依赖监盘结果（如果能盘）
    → 如果连盘点都无法覆盖 → 该科目范围受限
    → 评价对审计意见的影响（金额占比 + 利润影响）
```

**倒挤法（收发存缺失时的替代验证）：**

```
期初存货（上年审定）+ 本期入库（采购发票/入库单汇总）- 期末存货（实盘或明细）= 应转成本
↕ 对比
账面主营业务成本
↕ 差异评价
```

| 差异情况 | 判断 | 后续动作 |
|---------|------|---------|
| 差异 < 明显微小错报阈值 | 可接受 | 底稿记录 |
| 明显微小 < 差异 < 实际执行重要性 | 需解释 | 要求被审计单位说明差异来源 |
| 差异 ≥ 实际执行重要性 | 重大 | 审计调整或受限事项 |

⚠️ 例外：倒挤法的前提是能获取"本期入库合计"。如果入库数据也不完整，倒挤法不可用，退到分析性程序（毛利率/周转率合理性验证），但分析性程序不能独立支撑认定。

### 阻塞场景 2：监盘无法覆盖全部存放地点

> 类型：判断表
> 来源：CSA 1311 第7-14条
> 置信度：实务经验

**前提：** 存货分散在多个地点，部分地点（外地子公司/第三方仓库/客户处）无法在盘点日到场。

```
IF  可监盘部分金额 ÷ 存货总额 ≥ 80%（经验值）
    → 覆盖率充分，正常监盘
    → 未覆盖部分走替代程序
    → 底稿量化未覆盖金额

ELSE IF  覆盖率 60%-80%
    → 监盘可覆盖部分 + 未覆盖部分强化替代程序
    → 替代程序需至少两条路径交叉验证
    → 评价替代程序能否弥补 → 能则可接受 → 不能则受限

ELSE  覆盖率 < 60%
    → 🔴 重大范围受限可能
    → 必须执行全部可用替代程序
    → 仍不足 → 保留意见或无法表示意见
```

**替代程序路径（按证据力排序）：**

| 优先级 | 替代程序 | 证据力 | 适用条件 |
|--------|---------|--------|---------|
| 1 | 期后销售凭证（证明期末存在且正常流转） | 强 | 期后已出库/开票 |
| 2 | 向持有方函证数量和状况 | 强 | 存货在子公司/第三方/客户处 |
| 3 | 持有方自行盘点 + 影像证据 | 中 | 能配合但审计师无法到场 |
| 4 | 检查入库凭证（证明曾经入库） | 中 | 有入库单/采购发票 |
| 5 | 分析性程序（周转率/毛利率合理性） | 弱（辅助） | 任何情况都可做 |

⚠️ 单独使用第 5 级（分析性程序）不能替代监盘。必须至少有一条第 1-4 级证据支撑。

### 阻塞场景 3：发出商品无法验证存在

> 类型：判断表
> 来源：CSA 1311 第14条 + CAS 14 收入确认
> 置信度：实务经验

**前提：** 期末存货中包含"发出商品"（已发货未确认收入），需验证其真实存在。

```
IF  已发函证 且 收到回函确认
    → 存在 + 权利义务认定已覆盖 ✅
    → 进一步检查：期后是否正常确认收入（截止认定）

ELSE IF  未函证，但有期后开出的发票
    → 存在认定：期后开票 = 客户确实收到货 ✅
    → 截止认定：
        IF 发货在资产负债表日前 且 开票在资产负债表日后
            → 期末挂发出商品正确 ✅
        ELSE IF 发货也在资产负债表日后
            → 🔴 期末不应存在该笔发出商品
            → 审计调整：冲回发出商品

ELSE IF  未函证、无期后发票，但有签收单/物流回单
    → 存在认定：有签收证据 = 客户收到了 ✅（证据力中等）
    → 截止认定同上

ELSE  既没函证、也没期后发票、也没签收单
    → 🔴 无法证明发出商品真实存在
    → 路径1：要求补发函证
    → 路径2：获取客户对账单
    → 路径3：均无法取得 → 存在性认定失败
    → 考虑审计调整（全额计提/冲回）或审计意见影响
```

### 阻塞场景 4：出库单缺失导致截止测试无法执行

> 类型：判断表
> 来源：CSA 1311 + 实务经验
> 置信度：实务经验

**前提：** 无出库单或出库单不完整，截止测试失去直接基础。

```
IF  能获取全年或大部分月份的出库单
    → 正常执行截止测试（期末最后几张出库单 ↔ 收入确认时点）
    → 缺失月份单独注明

ELSE IF  完全没有出库单，但有销售发票 + 物流记录
    → 用"发票日期 + 物流发货日期"替代出库单做截止测试
    → 核对：发票日期是否在发货日期之后（正常）
    → 标记：发票日期在资产负债表日前 但 发货在日后 → 提前确认风险

ELSE  出库单、物流记录均缺失
    → 🔴 截止认定证据严重不足
    → 替代路径：
        1. 倒挤法验证成本结转总额合理性
        2. 毛利率月度趋势分析（异常月份 = 截止风险高发月份）
        3. 期后退货/红字发票检查（期后退货可能反映期末提前确认）
    → 替代程序只能验证总额合理性，无法逐笔验证截止
    → 底稿写明受限程度
```

## 二、应收账款循环 IF 线

### 阻塞场景 1：函证不回

> 类型：判断表
> 来源：CSA 1312 第15-19条
> 置信度：准则原文

**前提：** 已发出询证函，但对方未回函。

```
IF  未回函金额合计 < 实际执行重要性
    → 金额不重大，替代测试可选做
    → 底稿记录未回函清单即可

ELSE  未回函金额 ≥ 实际执行重要性
    → 必须执行替代测试：
        1. 检查期后回款（银行流水中找到对应入账）
        2. 检查销售合同 + 发货单 + 签收单（证明交易真实发生）
        3. 核对销售发票 + 出库单
    → 评价替代测试结果：
        IF 期后回款能覆盖 ≥ 80% 未回函金额
            → 可接受 ✅
        ELSE IF 覆盖 50%-80%
            → 未覆盖部分检查原始单据（合同/发货/签收）
            → 单据齐全 → 可接受
            → 单据缺失 → 该部分存在性认定不足
        ELSE  覆盖 < 50% 且 原始单据也缺失
            → 🔴 存在性证据不足
            → 考虑是否为坏账 → 评价减值充分性
            → 或为虚构应收 → 舞弊风险信号
```

---

### 阻塞场景 2：回函不符

> 类型：判断表
> 来源：CSA 1312 第20-23条
> 置信度：准则原文

**前提：** 收到回函，但对方确认金额与账面不一致。

```
IF  差异可解释（在途款项/未达账项/截止差异）
    → 逐笔核实差异原因
    → 编制调节表（类似银行余额调节表思路）
    → 调节后一致 → 可接受 ✅

ELSE IF  差异不可解释 且 差异金额 < 实际执行重要性
    → 列入未更正错报
    → 扩大检查范围（追加抽样）
    → 评价是否有系统性偏差

ELSE  差异不可解释 且 差异金额 ≥ 实际执行重要性
    → 🔴 可能存在：
        a. 虚增收入（账上有/对方说没有）
        b. 未记录退货/折让
        c. 记错客户（串户）
    → 追加程序：
        1. 检查对应收入的原始凭证（合同/发货/签收全链条）
        2. 检查期后是否有红字发票/退货
        3. 检查是否存在同名/近似名客户（串户可能）
    → 仍无法解释 → 审计调整
```

### 阻塞场景 3：管理层拒绝发函

> 类型：判断表
> 来源：CSA 1312 第8-9条
> 置信度：准则原文

**前提：** 管理层以"影响客户关系"等理由拒绝对某些客户发函。

```
IF  管理层拒绝理由合理（如客户明确表示不配合函证）
    → 执行替代测试（期后回款 + 原始单据全套检查）
    → 替代程序的质量和范围必须更充分（弥补未函证的弱点）
    → 底稿记录管理层拒绝理由 + 替代程序结论

ELSE IF  管理层拒绝理由不合理 或 范围过大（拒绝大部分客户发函）
    → 🔴 范围受限 + 舞弊风险信号
    → 考虑：为什么管理层不让发函？
        → 虚增收入（客户不存在/交易不存在）
        → 关联方伪装为客户
    → 向项目合伙人报告
    → 评价对审计意见的影响
    → 如果金额重大 → 保留意见或无法表示意见
```

⚠️ CSA 1312 第 9 条：管理层不允许寄发询证函时，注册会计师应当**质疑拒绝的理由**，并评价对相关认定的影响。

### 阻塞场景 4：账龄存疑/坏账准备不充分

> 类型：判断表
> 来源：CAS 22 + CSA 1540
> 置信度：实务经验

**前提：** 账龄分析显示大额长期挂账，但管理层不愿意计提充分的坏账准备。

```
IF  有充分期后回款证据
    → 虽然账龄长，但确实能收回 → 不计提合理 ✅
    → 底稿附期后回款截图/流水

ELSE IF  无期后回款，但有法院判决/担保物/还款计划
    → 可收回性有一定支撑 → 评价担保物价值/执行可能性
    → 计提比例可低于同账龄其他客户，但需单独说明理由

ELSE  无回款、无担保、无还款计划、客户经营异常
    → 🔴 减值证据充分，管理层不计提 = 报表错报
    → 独立估计应计提金额
    → 差额 ≥ 实际执行重要性 → 审计调整
    → 管理层拒绝调整 → 列为未更正错报 → 评价对意见的影响
```

## 三、收入循环 IF 线

### 阻塞场景 1：截止测试缺乏支持文件

> 类型：判断表
> 来源：CSA 1315 + 实务经验
> 置信度：实务经验

**前提：** 收入截止测试需要核对"收入确认时点 vs 发货/签收时点"，但发货单/签收单缺失。

```
IF  有出库单 + 物流单 + 客户签收单（全套）
    → 正常执行截止测试 ✅

ELSE IF  有出库单 + 物流单，无客户签收单
    → 物流签收记录可替代客户签收单（证据力略低）
    → 核对物流送达日期 vs 收入确认日期
    → 物流在资产负债表日前送达 + 收入在日前确认 → 截止正确

ELSE IF  只有销售发票，无发货/物流/签收
    → 🔴 截止测试的直接证据缺失
    → 替代路径：
        1. 与销售合同条款比对（约定的风险转移时点是什么）
        2. 检查期后退货/红字发票（有退货 → 可能提前确认了）
        3. 检查 ERP 发货状态记录（如果有系统）
        4. 毛利率月度分析（Q4 突增 → 截止风险信号）
    → 替代程序只能提供间接证据
    → 如果截止日前后金额 ≥ 重要性 → 底稿写明受限

ELSE  无发票、无发货单、无任何单据
    → 🔴🔴 收入真实性（不只是截止）存疑
    → 转入舞弊风险评估程序
```

### 阻塞场景 2：毛利率异常但管理层解释"合理"

> 类型：判断表
> 来源：CSA 1520 + 实务经验
> 置信度：实务经验

**前提：** 分析性程序发现毛利率异常波动（月度间/产品间/较上年），管理层给出解释。

```
IF  管理层解释可验证 且 验证结果支持
    → 接受解释 ✅
    → 底稿记录：异常现象 + 管理层解释 + 验证程序 + 结论
    验证方法举例：
    - "原材料涨价" → 核对采购单价变动
    - "产品结构变化" → 核对高毛利/低毛利产品占比变动
    - "一次性折扣/退货" → 核对对应凭证

ELSE IF  管理层解释不可验证（"市场竞争加剧""客户要求降价"但无合同/报价单支撑）
    → 解释不充分 ≠ 错报，但 = 需要追加程序
    → 追加：
        1. 抽取异常月份的大额销售逐笔检查
        2. 与同行业毛利率对比
        3. 检查是否存在关联方交易（关联方非公允定价）
    → 追加程序未发现问题 → 可接受（但底稿写明"解释未完全验证"）
    → 追加程序发现问题 → 转审计调整或受限

ELSE  管理层拒绝解释 或 解释自相矛盾
    → 🔴 舞弊风险信号（收入真实性/成本完整性）
    → 升级为特别风险
    → 向合伙人报告
```

### 阻塞场景 3：收入确认时点争议（五步法第五步）

> 类型：判断表
> 来源：CAS 14 第四条、第十一条
> 置信度：准则原文

**前提：** 对"控制权何时转移"存在分歧（时点 vs 时段、发货 vs 签收 vs 安装验收）。

```
IF  合同明确约定验收条款 且 验收有实质意义
    → 收入确认时点 = 客户验收完成日
    → 验收前 → 发出商品 / 合同资产

ELSE IF  合同有验收条款，但验收只是形式性的（客户几乎不可能拒收）
    → 收入确认时点 = 发货/交付日（实质控制权已转移）
    → 需要判断依据：历史拒收率、退货率、验收延迟原因

ELSE IF  合同无验收条款，FOB 发货点/目的地
    → FOB 发货点 → 装运时确认
    → FOB 目的地 → 送达时确认
    → 核对物流时间确定实际到达日

ELSE IF  一次销售分多批发货
    → 需判断：是一个履约义务还是多个？
        IF 每批货物客户可独立使用
            → 多个履约义务，分批确认收入
        ELSE 各批次相互依赖（必须全部到齐才能使用）
            → 一个履约义务，全部交付后确认

ELSE  无合同 / 口头协议 / 先货后款
    → 合同仍然存在（CAS 14 第五条：口头/行为默示合同有效）
    → 控制权转移时点 = 客户实际取得货物并可使用时
    → 通常 = 发货 + 客户签收/默认收货时
```

## 四、固定资产/在建工程 IF 线

### 阻塞场景 1：转固时点争议

> 类型：判断表
> 来源：CAS 4 第九条
> 置信度：准则原文

**前提：** 在建工程已实际使用，但管理层以"未办理竣工决算"为由不转固。

```
IF  已达到预定可使用状态（可客观判断）
    → 应当转固，不论是否已办理竣工决算
    → CAS 4 第九条：达到预定可使用状态时按暂估价转固
    → 竣工决算完成后调整暂估差异（不追溯折旧）

    "预定可使用状态"判断标准：
    - 实体建造已完成（主体结构、设备安装到位）
    - 已开始生产/使用 或 已具备生产/使用条件
    - 后续支出金额占预算比例极低（<5% 经验值）

ELSE IF  尚未达到预定可使用状态（仍在调试/试运行/整改）
    → 继续挂在建工程 ✅
    → 但关注：试运行产出的产品/收入是否已冲减在建工程成本

审计调整：
    IF 应转固但未转固的金额 ≥ 实际执行重要性
        → 计算少提折旧金额
        → 审计调整：借 管理费用/制造费用（折旧）  贷 累计折旧
        → 同时：借 固定资产  贷 在建工程
    ELSE
        → 列入未更正错报 + 管理建议书
```

⚠️ "长期挂在建工程不转固"是常见的利润操纵手段（少提折旧 → 虚增利润）。发现这个信号时保持职业怀疑。

### 阻塞场景 2：固定资产台账不完整/实物抽查有差异

> 类型：判断表
> 来源：CSA 1301 + 实务经验
> 置信度：实务经验

**前提：** 固定资产台账不完整，或实物抽查发现账实不符。

```
IF  台账不完整但 ERP 数据可用
    → 以 ERP 台账为主
    → 做 IPE 程序验证 ERP 数据准确性
    → 抽查验证：ERP 记录 ↔ 实物 ↔ 产权证/发票

ELSE IF  台账不完整且无 ERP
    → 🟡 证据获取困难
    → 替代路径：
        1. 从本期增加入手：逐笔检查采购合同/发票/验收单
        2. 从折旧重算入手：如果能拿到资产清单（哪怕不完整），重算折旧
        3. 实地抽查关键资产（金额大的逐笔查看）
    → 底稿写明台账缺失程度

实物抽查差异处理：
IF  账上有/实物无（可能已处置未销账）
    → 检查是否有报废/处置审批
    → 有 → 补做处置分录
    → 无 → 可能为虚增资产 → 审计调整（减少固定资产）

ELSE IF  实物有/账上无（可能未入账）
    → 检查是否为租赁资产/借用/代管
    → 确属自有但未入账 → 完整性问题 → 补入账
```

### 阻塞场景 3：减值迹象存在但管理层不做减值测试

> 类型：判断表
> 来源：CAS 8 第四、五条
> 置信度：准则原文

**前提：** 存在减值迹象（设备闲置/技术淘汰/长期亏损），但管理层认为"以后还能用"。

```
IF  减值迹象明确（已停产/已报废/市场价格持续下跌且有公开数据）
    → 应当做减值测试，管理层不做 = 潜在错报
    → 审计师独立估计可收回金额：
        a. 公允价值减去处置费用（有市场报价时）
        b. 预计未来现金流量现值（无市场报价时）
    → 估计减值金额 ≥ 实际执行重要性 → 审计调整
    → 管理层拒绝 → 未更正错报 → 评价意见影响

ELSE IF  减值迹象存在但不明确（行业下行/产能利用率下降/但仍在使用）
    → 要求管理层书面说明不做减值测试的理由
    → 评价理由合理性：
        - 是否有未来订单/使用计划支持
        - 行业下行是否影响到该资产的使用价值
    → 理由合理 → 可接受，底稿记录判断过程
    → 理由不合理 → 同上（独立估计+调整）

ELSE  无减值迹象
    → 不需要做减值测试 ✅
    → 但底稿应记录"已评估，未发现减值迹象"
```

## 五、长期股权投资 IF 线

### 阻塞场景 1：被投资方财务数据无法获取或未经审计

> 类型：判断表
> 来源：CSA 1301 + CAS 2 第十三条
> 置信度：实务经验

**前提：** 权益法核算需要被投资方净利润，但被投资方未审/未提供报表/报表不可靠。

```
IF  被投资方有经审计的报表
    → 正常用审计后净利润做权益法调整 ✅

ELSE IF  被投资方有未审报表，但无重大异常信号
    → 可暂用未审数据
    → 但需评价未审报表的可靠性：
        - 编制基础是否与本企业一致
        - 是否有其他信息交叉验证（如管理报表/纳税申报）
    → 底稿记录"使用未审数据"+ 理由 + 敏感性分析（如果被投资方利润±20%对本企业利润影响）

ELSE IF  被投资方完全不提供报表
    → 🔴 无法执行权益法核算
    → 替代路径：
        1. 检查被投资方银行对账单/纳税申报表（间接估计经营状况）
        2. 检查被投资方工商年报公示数据
        3. 评估是否应改为成本法（丧失重大影响的迹象？）
    → 如果完全无法获取任何数据 → 长投科目范围受限
    → 评价金额占比 → 影响审计意见
```

### 阻塞场景 2：减值判断缺乏外部支持

> 类型：判断表
> 来源：CAS 8 + CAS 2
> 置信度：实务经验

```
IF  被投资方持续盈利 且 净资产 > 投资账面价值
    → 通常无减值 ✅

ELSE IF  被投资方亏损，但投资账面价值 < 被投资方净资产 × 持股比例
    → 暂无减值（投资未超过其在被投资方中的权益份额）
    → 但需关注持续经营能力

ELSE IF  投资账面价值 > 被投资方净资产 × 持股比例（含商誉）
    → 存在减值迹象
    → 需要做可收回金额测试
    → 管理层不做 → 审计师独立估计或认定为潜在错报

ELSE IF  被投资方已停业/注销/资不抵债
    → 投资应全额减值
    → 管理层不计提 → 审计调整
```

## 六、研发支出 IF 线

### 阻塞场景 1：资本化时点判断争议

> 类型：判断表
> 来源：CAS 6 第九条
> 置信度：准则原文

**前提：** 管理层将研发支出资本化，但审计师对"是否满足五条件"有争议。

**五条件（必须同时满足）：**
1. 技术上可行
2. 有意图完成并使用或出售
3. 能够使用或出售
4. 能够产生经济利益（市场存在/自用有用）
5. 有足够的资源支持完成

```
IF  五条件全部有证据支撑
    → 资本化合理 ✅
    → 需验证的证据类型：
        - 技术可行：技术评审报告/测试通过记录/专利申请
        - 意图：立项审批文件/预算拨款/管理层声明
        - 能使用/出售：市场调研/客户意向/内部使用计划
        - 经济利益：可行性研究报告/NPV 测算
        - 资源充足：资金/人员/设备到位证明

ELSE IF  部分条件证据薄弱（如"技术可行"仅有管理层口头说明）
    → 提示：CAS 6 要求的是"企业能够证明"，不是"管理层认为"
    → 要求补充证据
    → 仍无法补充 → 该时点起的资本化不应确认 → 审计调整
    → 调整金额 = 不应资本化的支出转费用 + 已摊销部分调回

ELSE IF  工时记录不完整（无法区分研究/开发阶段人工）
    → 🟡 分配基础不可靠
    → 替代验证：
        1. 项目立项文件（是否明确划分阶段时间节点）
        2. 里程碑记录/测试报告日期（确定研究→开发转换点）
        3. 人员考勤 × 项目归属（重新估计工时分配）
    → 如果完全无法区分 → 全额费用化（稳健原则）
```

## 七、通用 IF 线：审计意见影响评价

> 类型：判断表
> 来源：CSA 1501/1502/1503
> 置信度：准则原文

**当任一科目走到"受限"终点时，需要统一评价对审计意见的影响：**

```
评价两个维度：
  1. 重大性：受限金额 vs 整体重要性
  2. 广泛性：是否影响多个科目/认定/报表使用者的决策

┌────────────────┬──────────────────┬──────────────────┐
│                │ 不广泛            │ 广泛              │
├────────────────┼──────────────────┼──────────────────┤
│ 错报（已识别） │ 保留意见          │ 否定意见          │
├────────────────┼──────────────────┼──────────────────┤
│ 范围受限        │ 保留意见          │ 无法表示意见      │
└────────────────┴──────────────────┴──────────────────┘
```

**"广泛性"判断标准（三选一即为广泛）：**
- 影响涉及财务报表的多个项目
- 虽只影响单一项目，但该项目构成/可能构成财务报表的主要部分
- 涉及披露时，对使用者理解财务报表至关重要

**底稿受限事项标准写法模板：**

```
审计受限事项说明：

事项：[具体描述无法执行的程序及原因]

影响金额：[具体金额] 万元，占 [科目/报表] 的 [百分比]%

已执行的替代程序：
  (1) [程序1及结果]
  (2) [程序2及结果]
  (3) ...

结论：
  □ 替代程序已获取充分适当证据，不影响审计意见
  □ 替代程序未能完全消除受限影响
    → 受限金额重大但不广泛 → 保留意见事项
    → 受限金额重大且广泛 → 无法表示意见事项
```

## 七（续）、受限程度动态累积评估

> 类型：流程
> 来源：CSA 705/706 + 实务经验
> 置信度：实务经验

**问题：** 第七章的 2×2 矩阵是终点判断工具——但审计师在项目过程中面对的是"一个信号一个信号地叠加"，很难一步到位判断"我现在到底受限到什么程度了"。

**解决：** 用一张**受限台账**动态追踪，每次新增一项"拿不到/做不了"时更新评估。

### 受限台账格式

```
┌──────────────────────────────────────────────────────────────────────────┐
│ 审计受限事项累积台账                                                       │
├────┬────────┬──────────┬──────────┬───────┬──────────┬─────────────────┤
│ 序号│ 科目    │ 受限事项  │ 受限金额  │ 占总资产│ 替代程序状态│ 残余受限金额  │
├────┼────────┼──────────┼──────────┼───────┼──────────┼─────────────────┤
│ 1  │ 存货    │ 监盘覆盖不足│ 800万    │ 5.3%  │ 已执行，覆盖60%│ 320万        │
│ 2  │ 应收账款│ 函证不回   │ 1200万   │ 8.0%  │ 期后回款80%  │ 240万        │
│ 3  │ ...    │ ...       │ ...      │ ...   │ ...        │ ...           │
├────┴────────┴──────────┼──────────┼───────┼──────────┼─────────────────┤
│                        │ 合计     │ 受限合计│          │ 残余受限合计    │
└────────────────────────┴──────────┴───────┴──────────┴─────────────────┘
```

### 动态评估逻辑

每新增一项受限事项时，按以下步骤更新判断：

```
步骤1：记录新增受限项（科目 + 事项 + 金额）

步骤2：执行替代程序后，计算"残余受限金额"
        残余受限金额 = 受限金额 × (1 - 替代程序覆盖率)

步骤3：累加所有科目的残余受限金额

步骤4：与重要性水平比较
        IF 残余受限合计 < 明显微小错报阈值
            → 🟢 当前状态：无影响
        ELSE IF 残余受限合计 < 实际执行重要性
            → 🟡 当前状态：需关注，继续追踪
        ELSE IF 残余受限合计 ≥ 实际执行重要性 但 < 整体重要性
            → 🟠 当前状态：接近保留意见边界
            → 立即报告项目合伙人
        ELSE IF 残余受限合计 ≥ 整体重要性
            → 🔴 当前状态：保留意见或更严重
            → 进入第七章 2×2 矩阵做最终判断

步骤5：广泛性评估（每次都做）
        受限涉及的科目数 ≥ 3 个 → 广泛性 = 是
        受限涉及单一科目但占报表主体 ≥ 25% → 广泛性 = 是
        → 广泛 + 重大 → 无法表示意见方向
```

### 阈值信号灯

| 信号 | 残余受限合计 vs 重要性 | 建议动作 |
|------|----------------------|---------|
| 🟢 无影响 | < 明显微小错报 | 正常推进，底稿记录即可 |
| 🟡 关注 | ≥ 明显微小 且 < 实际执行重要性 | 持续追踪，优先清理可解决的项 |
| 🟠 警戒 | ≥ 实际执行重要性 且 < 整体重要性 | 报告合伙人，评估是否能再补程序 |
| 🔴 触发 | ≥ 整体重要性 | 进入意见类型判断（第七章矩阵） |

### 与项目记忆的关系

当用户在使用古立特走 IF 线时，每次新增一个受限事项，古立特自动在 mind 的项目文件中追加受限台账记录。下次对话时基于已有台账继续累积，不需要用户重复描述之前的受限项。

⚠️ 这个台账是**过程工具**，不是底稿本身。最终底稿仍然按第七章的标准格式出具。台账的价值在于：让审计师在项目过程中**实时感知自己离意见边界有多远**。

## 八、货币资金 IF 线

### 阻塞场景 1：银行函证未收到回函

> 类型：判断表
> 来源：CSA 1312 + 实务经验
> 置信度：实务经验

**前提：** 已向开户行发出函证，但未收到回函。

```
IF  未回函银行账户余额合计 < 实际执行重要性
    → 金额不重大，替代程序可精简
    → 核对银行对账单 + 余额调节表即可

ELSE  未回函银行账户余额 ≥ 实际执行重要性
    → 必须执行替代程序：
        1. 获取银行对账单原件（注意辨别真伪：银行柜台打印/网银导出/盖章）
        2. 编制银行余额调节表，逐笔核实未达账项
        3. 检查期后银行流水（验证期末余额的后续变动合理）
        4. 核对银行开户信息（是否与人行备案一致）
    → 评价替代程序结果：
        IF 对账单可验证真实性 + 调节表差异可解释
            → 可接受 ✅
        ELSE IF 对账单来源存疑（仅客户提供复印件/无银行盖章）
            → 🟡 证据可靠性降级
            → 尝试：审计师亲自到银行柜台打印 / 使用网银当场查询
            → 仍无法验证 → 该账户范围受限
        ELSE IF 调节表未达账项长期未清/金额异常大
            → 🔴 可能存在资金挪用/虚构存款
            → 升级为舞弊风险信号
            → 追加：检查未达账项原始凭证 + 期后是否入账
```

⚠️ 银行函证是货币资金审计的必要程序（CSA 1312）。所有银行账户（含零余额/已注销）都应函证。未回函不能仅靠对账单替代——必须验证对账单来源的可靠性。

### 阻塞场景 2：银行余额调节表差异无法解释

> 类型：判断表
> 来源：CSA 1301 + 实务经验
> 置信度：实务经验

**前提：** 银行余额调节表编制后，调节后余额仍不一致，或未达账项性质异常。

```
IF  调节后余额一致 且 未达账项性质正常（时间性差异）
    → 正常 ✅
    → 逐笔检查大额未达账项的期后入账情况

ELSE IF  调节后余额一致，但存在长期未达账项（>3个月）
    → 🟡 异常信号
    → 长期企付银未付 → 可能是虚构支出（款项未真正流出）
    → 长期银收企未收 → 可能是隐瞒收入
    → 追加检查：
        1. 追溯到原始付款/收款凭证
        2. 检查对方单位是否真实存在
        3. 期后该笔是否最终到账/清理

ELSE IF  调节后余额不一致（即使加上未达账项也对不上）
    → 🔴 基础数据错误
    → 路径1：检查是否有银行账户未纳入核算（隐瞒账户）
    → 路径2：检查企业银行日记账是否有遗漏/重复入账
    → 路径3：要求被审计单位重新核对并解释差异
    → 仍无法解释 → 记录差异金额 → 评价是否构成错报
```

### 阻塞场景 3：存在受限/冻结资金但管理层未披露

> 类型：判断表
> 来源：CAS 31 + CSA 1312
> 置信度：实务经验

```
IF  函证回函显示存在质押/冻结/保证金 但 报表未披露
    → 🔴 列报认定存在错报
    → 要求管理层在附注中补充披露
    → 如涉及保证金/定期存款 → 同时检查是否应重分类为"其他非流动资产"

ELSE IF  银行函证中显示"资金归集/现金池"安排
    → 需评价：实际控制权是否在被审计单位
    → 如果资金被母公司/关联方实际占用 → 可能构成关联方资金占用
    → 检查资金归集协议条款 + 实际划转记录

ELSE IF  存在大额定期存款 但 企业同时高息借款（"存贷双高"）
    → 🔴 舞弊风险信号（虚构货币资金）
    → 追加程序：
        1. 定期存款到期日与借款到期日比对
        2. 定期存款是否被质押（函证确认）
        3. 利息收入是否与存款规模匹配
        4. 审计师亲自到银行核实余额
```

## 九、应付账款/预付账款 IF 线

### 阻塞场景 1：应付账款完整性——寻找未入账负债

> 类型：判断表
> 来源：CSA 1301 + 实务经验
> 置信度：实务经验

**前提：** 应付账款的核心风险是**低估（完整性）**而非高估。需要主动寻找是否有应付未付但未入账的负债。

```
IF  能获取期后付款明细（期后银行流水/付款清单）
    → 逐笔检查期后付款对应的发票/收货日期
    → IF 收货在资产负债表日前 但 未入账
        → 应补记应付账款（暂估）
        → 金额 ≥ 实际执行重要性 → 审计调整
    → IF 收货在资产负债表日后
        → 不需要调整 ✅

ELSE IF  无法获取期后付款明细
    → 替代路径：
        1. 检查期末有无大额入库单但无对应应付（暂估应到未到）
        2. 获取供应商对账单 → 与账面比较 → 差异即为可能的未入账负债
        3. 检查资产负债表日后收到的发票（日期在日前的 = 应暂估）
    → 全部做不到 → 完整性认定证据不足 → 底稿注明

ELSE IF  管理层不做暂估（"没收到发票就不入账"）
    → 🟡 这是权责发生制 vs 收付实现制的问题
    → 如果金额 < 实际执行重要性 → 记为管理建议事项
    → 如果金额 ≥ 实际执行重要性 → 审计调整（补暂估）
```

---

### 阻塞场景 2：供应商函证不回/管理层不配合函证

> 类型：判断表
> 来源：CSA 1312 + 实务经验
> 置信度：实务经验

**前提：** 应付账款函证的目的与应收不同——应付函证主要测试**完整性**（查漏），而非存在性。

```
IF  管理层配合发函 且 回函确认一致
    → 完整性认定有支撑 ✅

ELSE IF  回函金额 > 账面金额
    → 可能存在未入账负债
    → 检查差异原因：
        a. 在途（已发货未收到）→ 检查是否应暂估
        b. 对方多记（争议款项）→ 核实合同条款
    → 差异无法解释 → 可能需要补记应付

ELSE IF  回函金额 < 账面金额
    → 可能存在：提前入账/重复入账/虚构负债
    → 检查差异对应的入库单/发票 → 是否真实

ELSE IF  未收到回函
    → 应付账款函证不回的替代程序：
        1. 获取供应商对账单（非函证形式的对账）
        2. 检查对应采购合同 + 入库单 + 发票三单一致
        3. 检查期后付款（证明负债真实存在且后续正常清偿）
    → 应付函证不回的风险通常低于应收不回（应付低估是风险方向，不回函不影响存在性）

ELSE IF  管理层拒绝函证
    → 应付科目管理层拒绝函证的风险信号弱于应收（因为应付的风险方向是低估）
    → 但仍需评价理由
    → 替代：供应商对账单 + 三单核对 + 期后付款
```

### 阻塞场景 3：预付账款长期挂账/无法确认资产性质

> 类型：判断表
> 来源：CAS 30 + 实务经验
> 置信度：实务经验

```
IF  预付账款长期挂账（>1年）且 无对应合同/采购在执行
    → 🟡 可能已不具备资产性质
    → 需判断：
        a. 对方是否仍在经营（工商查询）
        b. 是否有追回预付款的计划/能力
        c. 是否实质为关联方借款（换了个名字）
    → 对方已注销/失联 → 应转为损失或计提减值
    → 实质为借款 → 重分类至其他应收款 + 披露关联方

ELSE IF  预付账款金额与合同约定不符
    → 超付部分 → 检查是否有补充协议
    → 少付部分 → 正常

ELSE IF  预付账款对方为关联方
    → 检查商业理由
    → 是否存在变相资金占用
    → 定价是否公允
```


> 类型：判断表
> 来源：CSA 1301 + 实务经验
> 置信度：实务经验

**前提：** 抽凭时发现大额费用无合同/发票/审批单等原始凭证支撑。

```
IF  缺少的是发票（有合同有审批有付款）

    → 🟡 内控缺陷，但不一定是错报

ELSE IF  合同/发票/审批全部缺失
    → 🔴 无法验证交易真实发生
    → 金额 ≥ 实际执行重要性 → 考虑审计调整（冲回费用）
    → 金额 < 实际执行重要性 → 列入未更正错报 + 扩大抽凭范围
    → 如果多笔费用都无凭证 → 舞弊风险信号（虚列费用/套取资金）
```

> 置信度：实务经验

```
IF  期末最后几天大量记录费用（Q4 费用占全年比例异常高）
    → 检查对应发票/服务日期：

ELSE IF  期初头几天记录了大量上期应计费用
    → 说明上期可能少计费用（完整性问题）
    → 如果是本期审计 → 检查本期期末是否存在同样问题
    → 如果是对上期的追溯 → 评价对期初余额的影响
```

## 十二、所得税 IF 线

### 阻塞场景 1：纳税调整项无法获取完整信息

> 类型：判断表
> 来源：企业所得税法 + 实务经验
> 置信度：实务经验

**前提：** 审计师需复核所得税纳税调整，但被审计单位未提供完整的纳税调整明细或汇算清缴申报表。

```
IF  能获取上年汇算清缴申报表 + 本年利润表明细
    → 基于知识库（tax_core）自行编制纳税调整初稿
    → 核心调整项：
        - 业务招待费（发生额×60% 与 收入×0.5% 孰低）
        - 广告宣传费（收入×15% 或 30%）
        - 职工福利费/工会经费/教育经费（工资×14%/2%/8%）
        - 折旧差异（会计 vs 税法年限）
        - 坏账准备（税法不允许扣除计提额）
        - 跌价准备（同上）
        - 滞纳金/罚款（不可扣除）
    → 与管理层确认后出审定数

ELSE IF  连利润表明细都不够详细（费用只有大类合计）
    → 要求补充费用明细（至少到二级科目）
    → 仍无法获取 → 基于大类金额做保守估计
    → 底稿注明"部分调整项因信息不足可能遗漏"

ELSE IF  管理层声称"没有需要调整的"但审计师发现存在
    → 独立计算应纳税所得额
    → 差异 = 审计师计算的所得税费用 - 管理层确认的
    → 差异 ≥ 实际执行重要性 → 审计调整
```

### 阻塞场景 2：递延所得税资产确认争议

> 类型：判断表
> 来源：CAS 18 第十三条
> 置信度：准则原文

**前提：** 被审计单位确认了大额递延所得税资产（DTA），但未来可利用的应纳税所得额存在不确定性。

```
        - 行业前景

    → DTA/DTL 应按预计转回期间的税率计量

```

## 十三、流转税（增值税/附加税）IF 线

### 阻塞场景 1：增值税进销项不匹配/税负率异常

> 类型：判断表
> 来源：增值税法 + 实务经验
> 置信度：实务经验

```
IF  税负率在行业正常范围内（±1个标准差）
    → 正常 ✅
    → 底稿记录税负率及行业对比即可

ELSE IF  税负率明显偏低（远低于行业均值）
    → 可能原因：
        a. 大量采购但未销售（囤货期）→ 检查存货增加是否合理
        b. 出口退税占比大 → 正常
        c. 多抵进项（虚开风险）→ 检查进项发票真实性
        d. 迟延确认销项（收入截止问题）→ 结合收入截止测试
    → 如果原因 c/d → 🔴 升级为税务风险/舞弊风险

ELSE IF  税负率明显偏高
    → 可能原因：
        a. 进项不足（供应商不开票/小规模纳税人采购多）
        b. 视同销售未抵扣进项
    → 通常不构成审计风险（高税负 ≠ 报表错报）
    → 管理建议书提示税务优化空间

ELSE IF  应交增值税期末借方余额（留抵税额）异常大
    → 检查是否存在虚开进项（进项远超经营需要）
    → 如果留抵真实 → 确认重分类至"其他流动资产"
```

---

### 阻塞场景 2：应交税费明细与申报表不一致

> 类型：判断表
> 来源：实务经验
> 置信度：实务经验

```
IF  差异可解释（时间性差异：账面按月计提，申报按季/按次）
    → 检查时间差是否在期后消除
    → 消除 → 可接受 ✅

ELSE IF  差异无法解释 且 账面 > 申报（多计提）
    → 可能是多计税金 → 需要冲回
    → 也可能是有未申报的税款（欠税）→ 风险判断

ELSE IF  差异无法解释 且 账面 < 申报（少计提/漏记）
    → 完整性问题 → 补记应交税费
    → 金额 ≥ 实际执行重要性 → 审计调整
```

## 十四、债务循环（短期/长期借款）IF 线

### 阻塞场景 1：借款函证不回/回函不符

> 类型：判断表
> 来源：CSA 1312 + 实务经验
> 置信度：实务经验

**前提：** 向银行发出的函证（含借款信息）未回函或回函内容与账面不一致。

```
IF  借款函证已回函且一致
    → 存在 + 完整性 + 权利义务 已覆盖 ✅

ELSE IF  未收到回函
    → 借款函证不回比存款函证不回更罕见（银行通常会回复）
    → 替代程序：
        1. 检查借款合同原件
        2. 核对借款到账银行流水
        3. 检查利息支付记录（与合同利率/本金一致）
        4. 检查期后还款记录
    → 如果借款合同 + 银行流水 + 利息都能对上 → 可接受
    → 如果连合同都没有 → 🔴 借款真实性存疑

ELSE IF  回函金额 ≠ 账面
    → 回函 > 账面 → 可能存在未入账借款（完整性问题）
        → 检查遗漏的借款合同/资金到账记录
        → 补记借款 + 利息
    → 回函 < 账面 → 可能存在虚构借款 或 已还款未销账
        → 检查还款记录
        → 如果已还但未销账 → 审计调整
        → 如果无还款凭证 → 🔴 虚构借款风险

ELSE IF  回函显示有担保/质押/限制条款 但 报表未披露
    → 补充披露
    → 检查是否违反借款合同条款（可能影响持续经营/流动性判断）
```

---

### 阻塞场景 2：利息费用与借款本金/利率不匹配

> 类型：判断表
> 来源：CAS 17 + 实务经验
> 置信度：实务经验

```
IF  重算利息 = 账面利息费用（容差 < 明显微小错报）
    → 利息费用确认准确 ✅

ELSE IF  重算利息 > 账面（少计利息）
    → 可能是期末利息未计提
    → 审计调整：借 财务费用  贷 应付利息
    → 金额 ≥ 实际执行重要性 → 调整

ELSE IF  重算利息 < 账面（多计利息）
    → 可能原因：
        a. 计提了不存在的借款利息 → 结合借款真实性检查
        b. 资本化利息误计入费用 → 应重分类至在建工程/研发支出
        c. 利率适用错误
    → 差异 ≥ 实际执行重要性 → 审计调整
```

## 十五、权益循环 IF 线

### 阻塞场景 1：出资不实/抽逃出资

> 类型：判断表
> 来源：公司法 + 实务经验
> 置信度：实务经验

**前提：** 检查实收资本/股本时，发现出资可能不实。

```
IF  货币出资已到账 且 验资报告/银行进账单齐全
    → 正常 ✅
    → 但关注：出资到账后是否被立即转出（抽逃出资信号）
        IF 出资后短期内（<30天）以借款/往来/采购等名义大额转出
            → 🔴 抽逃出资嫌疑
            → 检查转出对方是否为关联方/股东本人
            → 如确认抽逃 → 实收资本不实 → 审计调整或披露

ELSE IF  非货币出资（实物/知识产权/土地）
    → 检查评估报告是否存在高估
    → 检查资产是否已实际交付/过户
    → 评估值明显高于市场价 → 出资不实 → 考虑减值

ELSE IF  认缴制下尚未实缴
    → 检查章程约定的出资期限
    → 未到期 → 不需要处理（附注披露）
    → 已到期未缴 → 检查股东是否有能力缴纳 → 可能构成其他应收款
```

---

### 阻塞场景 2：未分配利润/盈余公积计算错误

> 类型：判断表
> 来源：公司法 + CAS 30 + 实务经验
> 置信度：实务经验

```
IF  期末未分配利润 = 期初 + 本期净利润 - 提取盈余公积 - 分配股利 ± 追溯调整
    → 正确 ✅

ELSE IF  勾稽关系不成立
    → 逐项排查：
        a. 本期净利润是否与利润表一致
        b. 盈余公积是否按10%正确提取（法定盈余公积达注册资本50%后可不提）
        c. 分配股利是否有股东会决议支持
        d. 是否有未通过利润表的直接调整（如前期差错更正追溯调整）
    → 找到差异来源 → 调整或说明

ELSE IF  存在大额利润分配但无股东会决议
    → 🔴 程序性问题（公司法要求）
    → 分配是否经过合法程序？
    → 无决议 → 管理建议书 + 检查是否构成抽逃资金
```

---

### 阻塞场景 3：其他综合收益变动无法追溯

> 类型：判断表
> 来源：CAS 30 + 实务经验
> 置信度：实务经验

```
IF  其他综合收益变动可追溯至具体科目（其他债权投资公允价值变动/外币报表折算差额等）
    → 正常 ✅
    → 核对变动金额与对应资产/负债变动一致

ELSE IF  变动来源不明/无法追溯
    → 要求管理层提供OCI变动明细台账
    → 无台账 → 从源科目反向推导：
        1. 其他债权投资：期末公允价值 - 摊余成本 = OCI累计余额
        2. 权益法 OCI：被投资方 OCI × 持股比例
        3. 外币折算：检查合并报表折算过程
    → 推导结果与账面一致 → 可接受
    → 不一致 → 审计调整
```
