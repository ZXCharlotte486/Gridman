# 审计程序与底稿通用逻辑

> 定位：事务所底稿编制的通用核心逻辑，不绑定特定模板
> 来源：真实事务所底稿模板提炼 + 天职国际指引
> 覆盖：六阶段、实质性程序、报表填列、重分类、截止测试、函证（含 CSA 1312 核心规则）、供应商风险分级、未披露关联方信号
> 性质：实操

> 章节索引（L2）：审计全流程六阶段总览 ｜ 第1阶段：初步业务活动 ｜ 第2阶段：风险评估 ｜ 第3阶段：风险应对-一般性程序与控制测试 ｜ 第4阶段：风险应对-实质性程序 ｜ 第5阶段：完成阶段 ｜ 第6阶段：特定项目程序 ｜ 集团审计与尽调通用规则 ｜ 报表调整体系（试算平衡表） ｜ 银行函证标准格式（财会〔2022〕39号） ｜ 函证准则核心规则（CSA 1312） ｜ 供应商风险分级与供应商审计（采购循环补充） ｜ 科目表标准体系 ｜ 古立特输出规范

---

## 审计全流程六阶段总览

```
1.初步业务活动 → 2.风险评估 → 3.风险应对-一般性程序与控制测试 → 4.风险应对-实质性程序 → 5.完成阶段 → 6.特定项目程序
```

| 阶段 | 核心任务 | 古立特能力 |
|------|----------|-----------|
| 1.初步业务活动 | 承接/保持决策、独立性、业务约定 | 知识判断（checklist） |
| 2.风险评估 | 了解环境、了解内控、识别重大错报风险 | 数据分析 + 知识判断 |
| 3.风险应对-一般性程序与控制测试 | 穿行测试、控制有效性测试 | 知识指引 |
| 4.风险应对-实质性程序 | 各科目审计（审定表/明细表/测试表） | **核心计算能力** |
| 5.完成阶段 | 汇总错报、形成结论、出具报告 | 汇总计算 + 知识判断 |
| 6.特定项目程序 | 关联方、持续经营、舞弊等专项 | 知识判断 |

## 第2阶段：风险评估

> 风险评估是审计的"导航系统"——决定后续所有程序的方向和力度。
> 古立特在此阶段的角色：数据分析驱动风险识别，辅助审计师做出判断。

### 2.1 了解被审计单位及其环境

**古立特能做：** 收集公开信息、财务数据分析、行业对标
**人来做：** 访谈管理层、现场观察、判断诚信

#### 2.1.1 环境分析框架

| 维度 | 了解内容 | 古立特数据来源 | 人工补充 |
|------|----------|---------------|----------|
| 行业状况 | 行业特征、竞争格局、监管变化、技术趋势 | 年报下载(`report_download`)、行业数据 | 行业专家访谈 |
| 法律环境 | 适用准则、法规、税收政策 | 知识库(tax_core/accounting_core) | 律师函证 |
| 经营性质 | 业务模式、收入来源、供应链、客户集中度 | 财务数据分析(`stock_financial`) | 管理层访谈 |
| 治理结构 | 股权结构、董事会、审计委员会 | 公告下载、企查查 | 章程审阅 |
| 会计政策 | 重要政策选择、变更、估计 | 附注分析 | 前期底稿 |
| 目标战略 | 经营目标、战略规划、KPI | 年报MD&A | 管理层沟通 |
| 财务业绩 | 关键指标、趋势、异常 | `financial_ratios` + `analytical_procedures` | — |

#### 2.1.2 检查相关信息

古立特自动收集：
- 企业年报/半年报（`report_download`）
- 历史财务数据（`stock_financial`）
- 行业对标数据
- 监管处罚记录（联网查询）

#### 2.1.3 初步分析程序（古立特核心能力）

**输入：** 上年审定 TB + 本期未审 TB

**古立特执行：**
1. `analytical_procedures` → 各科目波动分析（金额、结构比、横比%）
2. `financial_ratios` → 关键比率变动（流动比率、周转率、负债率、毛利率等）
3. 自动标记异常：波动>10%的科目、从无到有的科目、比率恶化的指标

**输出结构：**
```json
{
  "fluctuation_summary": {
    "异常科目数": 12,
    "重要科目异常": ["应收账款+45%", "存货周转天数60→120天"],
    "新增科目": ["使用权资产", "租赁负债"],
    "消失科目": []
  },
  "ratio_alerts": [
    {"指标": "流动比率", "上年": 1.8, "本期": 1.1, "判断": "流动性恶化"},
    {"指标": "应收账款周转天数", "上年": 45, "本期": 89, "判断": "回款显著放缓"}
  ],
  "risk_signals": [
    "应收账款增长远超收入增长 → 收入真实性风险",
    "存货周转大幅放缓 → 跌价风险",
    "经营现金流为负但利润为正 → 盈余质量风险"
  ]
}
```

#### 2.1.4 重要性计算（古立特核心能力）

**古立特执行：** `materiality_calculator`

**输入：** 税前利润 / 总资产 / 营业收入（三选一或自动选择）

**输出：**
- 整体重要性水平
- 实际执行重要性（通常为整体的 50%-75%）
- 明显微小错报阈值（通常为整体的 5%）
- 职业判断说明（可直接复制到底稿）

#### 2.1.5 了解收入模式

古立特基于财务数据分析：
- 收入构成（主营/其他，产品/服务）
- 收入季节性分布（月度波动）
- 前五大客户集中度
- 收入与现金流匹配度
- 收入舞弊迹象信号（收入增长异常、期末突增、关联方占比高）

#### 2.1.6 识别关联方

古立特辅助：
- 从股权结构识别法定关联方
- 从交易数据识别异常交易对手（金额大、频率高、条件异常）
- 与企查查/天眼查数据交叉验证

### 2.3 识别和评估重大错报风险

**古立特能做：** 基于数据分析结果，自动生成风险评估初稿
**人来做：** 最终定级、确认特别风险

#### 2.3.1 风险评估矩阵

```
对每个重要科目/认定：
  固有风险等级 = 高/中/低（基于科目性质+波动分析）
  控制风险等级 = 高/中/低（基于内控了解结果）
  重大错报风险 = 固有风险 × 控制风险
  → 确定进一步审计程序的性质、时间和范围
```

**古立特自动判断固有风险的规则：**

| 信号 | 固有风险判断 |
|------|-------------|
| 科目波动 > 30% 且金额超过重要性 | 高 |
| 涉及重大会计估计（商誉/公允价值/ECL） | 高 |
| 收入类科目（推定舞弊风险） | 高 |
| 科目波动 10%-30% | 中 |
| 科目稳定、金额小于重要性 | 低 |
| 从无到有的新科目 | 至少中 |

#### 2.3.2 认定层次

**资产负债表认定：**
- 存在（E）：资产/负债在期末确实存在
- 权利和义务（R）：资产归企业所有，负债是企业义务
- 完整性（C）：所有应记录的都已记录
- 计价和分摊（V）：金额正确、减值充分
- 列报（P）：分类和披露恰当

**利润表认定：**
- 发生（O）：交易确实发生
- 完整性（C）：所有交易都已记录
- 准确性（A）：金额正确
- 截止（CO）：记录在正确期间
- 分类（CL）：记入正确科目

#### 2.3.3 特别风险识别

以下情形应评估为特别风险：
- 收入确认（推定存在舞弊风险）
- 管理层凌驾于控制之上（推定存在）
- 重大非常规交易（关联方、年末大额）
- 重大会计估计（高度不确定性）
- 重大关联方交易

**管理层凌驾测试——日记账异常分录筛选（古立特能做）：**

从序时账中筛选以下特征的分录，作为重点检查对象：

| 筛选规则 | 异常信号 | 古立特工具 |
|----------|----------|-----------|
| 期末最后几天录入 | 粉饰报表 | `cutoff_test`（window_days=3） |
| 非工作时间录入（周末/深夜） | 绕过正常审批 | `data_filter`（按日期筛选） |
| 整数金额（万/十万/百万） | 人为编造 | `cutoff_test`（自动标记整数） |
| 借贷科目异常组合（收入↔费用直接对冲） | 虚增收入/隐藏费用 | `data_filter` |
| 摘要含"调整"/"冲回"/"更正" | 非常规调整 | `data_filter`（contains） |
| 金额恰好低于审批权限 | 规避审批 | `data_filter`（阈值筛选） |
| 编制人为管理层/非常规人员 | 凌驾控制 | `data_filter` |

**执行方式：** 用户提供序时账 → 古立特按上述规则批量筛选 → 输出可疑分录清单 → 审计师逐笔检查原始凭证。

#### 2.3.4 舞弊风险因素（三角分析）

| 因素 | 信号 |
|------|------|
| 动机/压力 | 业绩对赌、融资需求、管理层薪酬与业绩挂钩、行业下行 |
| 机会 | 内控薄弱、管理层凌驾、复杂交易结构、关联方众多 |
| 态度/借口 | 管理层诚信存疑、频繁更换审计师、激进会计政策 |

**古立特辅助：** 从财务数据识别舞弊信号
- 收入增长远超行业 → 虚增收入风险
- 经营现金流持续为负但利润为正 → 盈余质量风险
- 应收账款/收入比率持续上升 → 提前确认收入
- 期末大额异常交易 → 粉饰报表
- 关联方交易占比异常 → 利益输送

#### 2.4.3 古立特风险评估工作流

```
用户输入：上年审定TB + 本期未审TB + 企业基本信息
                    ↓
Step 1: materiality_calculator → 三档重要性
Step 2: analytical_procedures → 波动分析 + 异常标记
Step 3: financial_ratios → 关键比率对比（本期vs上期vs行业）
Step 4: 知识层判断 → 基于波动+比率自动识别风险信号
Step 5: 生成风险评估初稿
                    ↓
输出：
  ├── 重要性水平计算表（含职业判断说明）
  ├── 初步分析程序表（各科目波动+异常标记）
  ├── 财务比率对比表（含行业对标）
  ├── 风险信号清单（自动识别）
  ├── 建议的风险等级（按科目/认定）
  └── 建议的审计重点领域
```

## 第3阶段：风险应对-一般性程序与控制测试

### 3.1 一般性程序

| 程序 | 内容 |
|------|------|
| 总体应对 | 项目组讨论、保持职业怀疑、分派经验丰富人员 |

### 3.2 控制测试

**古立特输出：** 控制测试方案

| 要素 | 内容 |
|------|------|
| 测试对象 | 拟信赖的关键控制 |
| 测试方法 | 询问+观察+检查+重新执行 |

### 4.3 往来款重分类

> 往来科目实质性程序完成后、编制报表调整之前的必要步骤。
> 准则依据：CAS 30 第十三条——资产和负债不得相互抵销；报表项目应按明细余额方向分别列报。

#### 重分类规则表

| 原始科目 | 明细余额方向 | 重分类至报表项目 | 备注 |
|---------|-------------|----------------|------|
| 应收账款 | 借方 | **应收账款** | 按客户汇总后的借方余额 |
| 应收账款 | 贷方 | **预收款项**（或合同负债） | 新收入准则下有履约义务→合同负债 |
| 预收账款 | 贷方 | **预收款项** | 正常 |
| 预收账款 | 借方 | **应收账款** | 实质是应收 |
| 应付账款 | 贷方 | **应付账款** | 按供应商汇总后的贷方余额 |
| 应付账款 | 借方 | **预付账款** | 实质是预付 |
| 预付账款 | 借方 | **预付账款** | 正常 |
| 预付账款 | 贷方 | **应付账款** | 实质是应付 |
| 其他应收款 | 借方 | **其他应收款** | 按对方单位汇总 |
| 其他应收款 | 贷方 | **其他应付款** | 实质是欠对方 |
| 其他应付款 | 贷方 | **其他应付款** | 按对方单位汇总 |
| 其他应付款 | 借方 | **其他应收款** | 实质是对方欠本企业 |
| 合同负债 | 贷方 | **合同负债** | 合同负债科目余额直接列示，不做方向重分类；但其他科目贷方余额满足履约义务条件时可重分类至此项目 |
| 应交税费 | 贷方 | **应交税费** | 正常 |
| 应交税费 | 借方 | **其他流动资产** | 留抵税额等 |

**核心原则**：按同一往来单位的明细余额方向分别汇总，借方→资产，贷方→负债。禁止净额抵消。

#### 操作步骤

**Step 1：数据准备**

从财务系统导出往来科目明细账，必要字段：
- 对方单位名称（客户/供应商编码）
- 科目代码 + 科目名称
- 余额方向（借/贷）
- 余额金额

**Step 2：按对方单位+科目汇总**

将所有借方余额明细汇总为"借方总额"，所有贷方余额明细汇总为"贷方总额"。

⚠️ 不要对同一单位同一科目做净额抵消后再重分类——正确做法是将贷方余额全部转出。

**Step 3：确定重分类方向**

对照规则表，借方总额→资产类项目，贷方总额→负债类项目。

**Step 4：编制重分类调整分录**

| 情形 | 分录 |
|------|------|
| 应收账款贷方余额→预收款项 | 借：预收账款 / 贷：应收账款 |
| 应付账款借方余额→预付账款 | 借：预付账款 / 贷：应付账款 |
| 其他应收款贷方余额→其他应付款 | 借：其他应付款 / 贷：其他应收款 |
| 预收账款借方余额→应收账款 | 借：应收账款 / 贷：预收账款 |
| 预付账款贷方余额→应付账款 | 借：预付账款 / 贷：应付账款 |
| 其他应付款借方余额→其他应收款 | 借：其他应收款 / 贷：其他应付款 |

**Step 5：报表项目填列公式**

| 报表项目 | 计算公式 |
|---------|---------|
| 应收账款 | 应收账款借方余额合计 + 预收账款借方余额合计 - 坏账准备 |
| 预收款项 | 应收账款贷方余额合计 + 预收账款贷方余额合计 |
| 合同负债 | 合同负债贷方余额 + 其他科目贷方余额中满足履约义务条件的部分（需价税分离） |
| 应付账款 | 应付账款贷方余额合计 + 预付账款贷方余额合计 |
| 预付账款 | 应付账款借方余额合计 + 预付账款借方余额合计 - 坏账准备 |
| 其他应收款 | "应收利息" + "应收股利" + 其他应收款借方余额 + 其他应付款借方余额 - 坏账准备 |
| 其他应付款 | "应付利息" + "应付股利" + 其他应收款贷方余额 + 其他应付款贷方余额 |

#### 合同负债 vs 预收款项（新收入准则）

- **合同负债**：已收/应收对价，尚未履行履约义务，不含增值税
- **预收款项**：不构成履约义务的预收款（预收租金、押金等）
- 应收账款贷方余额重分类：有履约义务→合同负债（需价税分离）；无履约义务→预收款项
- 价税分离示例：应收贷方 113 元（含13%税）→ 合同负债 100 + 应交税费-待转销项税额 13

#### 古立特工具支持

直接调用 `reclassification` 工具：输入往来明细（含对方名称+科目+借方/贷方余额）→ 自动输出重分类明细表 + 调整分录 + 报表项目汇总。覆盖应收/应付/预收/预付/其他应收/其他应付/合同负债等科目。

#### 常见错误

| 错误 | 正确做法 |
|------|---------|
| 同一单位应收和预收直接抵消 | 分别列示，不得抵消 |
| 只重分类应收，忽略应付和其他往来 | 所有往来科目都需重分类 |
| 重分类后未同步调整账套 | 编制重分类凭证入账，或建立重分类底稿 |
| 合同负债和预收账款不加区分 | 按履约义务区分（新收入准则） |
| 对净额做重分类 | 应对借方/贷方余额分别汇总后各自重分类 |

### 5.7 审计报告（人做）

古立特不出具审计报告，但能提供判断依据：

| 意见类型 | 条件 | 古立特辅助 |
|----------|------|-----------|
| 无保留意见 | 报表在所有重大方面公允反映 | 错报<重要性 + 勾稽通过 |
| 保留意见 | 错报重大但不广泛，或范围受限但不广泛 | 标记超重要性的科目 |
| 否定意见 | 错报重大且广泛 | 标记多科目超重要性 |
| 无法表示意见 | 范围受限且广泛 | — |

#### 审计意见实务判断规则（实务经验）

**强调事项段 vs 保留意见的区别：**
- **保留意见** = 报表存在重大错报（事实问题）
- **强调事项段** = 存在需要使用者关注的重大事项，但不影响意见（提醒性质）
- 两者可以同时存在（保留意见 + 强调事项段）

**强调事项段适用情形：**
- 持续经营重大不确定性（管理层仍按持续经营编制，且已充分披露）
- 重大未决诉讼/担保（已在附注披露）
- 重大期后事项（已在附注披露）

**持续经营与审计意见的关系：**

| 情形 | 意见类型 |
|------|----------|
| 存在重大不确定性，管理层已采取缓解措施且充分披露 | 无保留 + 强调事项段 |
| 存在重大不确定性，管理层未充分披露 | 保留意见 |
| 持续经营假设不适当，但管理层仍按持续经营编制 | 否定意见 |
| 无法判断持续经营能力（证据不足） | 无法表示意见 |

**期后事项对审计意见的影响：**
- 期后发现报告期内差错 → 审计调整 → 可能影响意见
- 期后新发生事项 → 非调整事项 → 附注披露 → 不影响意见
- 期后事项导致持续经营存疑 → 可能需要强调事项段

### 5.9 管理建议书编制

> 管理建议书（Management Letter）是审计的增值产出——审计报告给股东看，管理建议书给管理层看。
> 准则依据：注册会计师对审计过程中注意到的内部控制重大缺陷，应当告知被审计单位管理当局，必要时可出具管理建议书。

#### 编制时机

管理建议书不是项目结束后才写，而是贯穿全流程：

| 阶段 | 动作 |
|------|------|
| 计划阶段 | 项目经理要求各组以书面形式记录发现的问题 |
| 执行阶段 | 重要问题书面记录，及时与企业沟通了解详情 |
| 完成阶段 | 分类汇总 → 与总监确定框架 → 分配撰写 → 项目经理汇总 |

#### 四大板块框架

| 板块 | 覆盖内容 |
|------|---------|
| 一、公司治理结构 | 独立性、关联交易、三会执行、股权激励、同业竞争、信息披露制度 |
| 二、公司经营管理 | 经营指标分析（毛利率/周转率/同行对比）、销售/采购/生产/存货/资产/合同管理问题 |
| 三、财务核算管理 | 会计处理不正确、做账习惯差、研发核算、收入确认时点、成本核算、税务问题 |
| 四、上年问题整改 | 上年提出的问题改了没有？为什么没改？ |

#### 单条建议写作结构（三段式）

每条建议必须包含：

| 段落 | 内容 | 要点 |
|------|------|------|
| 现状 | 描述问题事实 | 客观陈述，引用具体业务场景或数据 |
| 影响 | 分析问题后果 | 用数据量化风险或损失 |
| 建议 | 提出改进措施 | 必须可操作、切实可行 |

排列顺序：按对公司经营和报表披露的影响程度排列，不堆砌。

#### 高频问题清单

**经营管理类：**
- 客户/供应商集中度风险
- 存货库龄管理（呆滞/长库龄/减值）
- 发出商品长期未结转
- 工程项目未招投标
- 对账执行不及时
- 长账龄应收款回款风险
- 子公司投后管理薄弱
- ERP 功能不完善

**财务核算类：**
- 未按权责发生制做账（按发票入账）
- 关联交易统计不完整
- 母子公司拆借款利息未计提
- 做账习惯差（摘要/二级科目/借贷方不规范）
- 研发人工归集问题（工时记录/人员认定）
- 收入确认时点问题
- BOM 未及时维护

**税务类：**
- 增值税进项加计扣除提醒
- 滞纳金事项
- 大量红字发票风险
- 集团内部转移利润风险
- 高新申报表填列风险
- 个税手续费返还未申报

#### 汇报对象与关注点

| 对象 | 关心什么 |
|------|---------|
| 实控人/董事长 | 舞弊风险、成本数据、降本增效、合规风险、行业对标 |
| CFO | 经营分析支持、明细数据、月度分析能力 |
| 投资人 | 业绩趋势、在手订单、持续盈利能力、大股东动向、核心人员稳定性 |

#### 写作规范

- 结论先行，自上而下
- 用词庄重平实，避免冗长
- 多用数据和图表，量化影响
- 问题要有深度（不在多而在精）
- 不局限于财务——扩展到运营、销售、HR、IT
- 提交形式建议 PPT（非 Word），提升正式感
- 沟通后持续跟进整改，形成闭环

#### 质量红线

- 不得与已出具的审计报告/底稿结论相冲突
- 项目组内部先统一意见再与客户沟通
- 合伙人和总监必须深度参与（事前确定框架）
- 提交前由项目负责人复核

#### 常见质量缺陷（自检清单）

> 类型：判断表
> 来源：实务事务所管理建议书复核经验
> 置信度：实务经验

| 缺陷 | 后果 |
|------|------|
| 建议数量过少（<5条） | 客户感知不到增值，认为审计"走过场" |
| 偏财务核算/内控，经营管理类建议少 | 实控人/CFO 觉得"说的都是我知道的" |
| 没有主体框架，直列问题清单 | 缺乏层次感，阅读体验差 |
| 合伙人/总监参与度低 | 框架和方向没把关，问题深度不够 |
| 提交时间偏晚（报告出了才给） | 错失沟通窗口，管理层已不关注 |
| 建议不可操作（"加强管理""完善制度"） | 形同虚设，不如不写 |

#### 沟通要求（完成后闭环）

> 类型：流程
> 来源：实务事务所管理建议书沟通经验
> 置信度：实务经验

- 沟通形式：以 **PPT 当面汇报**为主（提升正式感和客户体验感），电话会议为辅
- 参会对象：合伙人/总监/项目经理均应参与；客户侧**至少包含实控人（民企）或董事长（国企）**——较高层级管理层参与，建议才能推进落地
- 沟通闭环：不定期跟踪管理建议书中问题的整改情况，形成"发现→建议→整改→复核"闭环，便于下一期审计工作顺利开展
- 不便留痕的具体问题可模糊化处理，现场口述表达

#### 经营管理层面高频问题补充（详细清单）

> 类型：判断表
> 来源：实务事务所管理建议书案例库
> 置信度：实务经验

以下为经营管理板块常见问题的细化清单，供构建管理建议书提纲时逐项对照：

| 领域 | 常见问题 |
|------|---------|
| 经营方向 | 是否聚焦主业；资产无序扩张风险 |
| 销售管理 | 客户集中度风险；发出商品长期未结转；物流运费台账缺失；新增客户尽调不细致；项目管理不到位；退货未及时冲减收入成本 |
| 采购管理 | 工程项目未招投标（价格公允性）；业务流程管控需完善；应扩大集中采购范围降本 |
| 生产/存货 | 存货库龄管理（呆滞/过时/积压）；盘点流程瑕疵；产能利用管理；成本管控（能耗分析）；项目超预算；减值风险未充分评估 |
| 资产管理 | 未及时入账/转固时点偏晚；长期资产减值可能；固定资产标识卡不完善；部分资产未办产权证；在建工程台账不完善 |
| 募投项目 | 转固对未来报表的影响金额；延期结项；资金超额度 |
| 子公司管理 | 收购子公司未派遣财务人员；母子公司人员共用/费用归属不清；投后管理薄弱；应推进"瘦身强体" |
| ERP/信息系统 | ERP 功能不完善（暂估未通过 ERP 处理）；数据交互薄弱；信息化覆盖率不足 |
| 其他风险 | 融资性贸易嫌疑；子公司营业执照被吊销仍经营；关联方拆借协议未及时签订；合同台账未覆盖全集团 |

### 5.10 审计问题分类与报告化

> 适用场景：收支审计、合规审计、专项检查——审计人员已发现问题，需要整理为规范化审计报告。
> 古立特角色：将口语化/零散的问题清单 → 公文式重述 + 分类 + 法规匹配 + 建议。

#### 问题分类体系（12 类）

| 编码 | 类别 | 涵盖内容 |
|------|------|---------|
| ZD | 重大政策落实 | 违规收费、中介服务清理不到位、政策执行打折扣 |
| YS | 预算决算 | 预算编制不实、无预算支出、结转结余较多、决算不准确 |
| ZJ | 资金管理 | 专项资金违规、改变用途、现金结算、公务卡不到位、非税未缴库 |
| ZC | 资产管理 | 配置超标、未入账、闲置、处置不规范、账实不符、产权不完整 |
| CG | 政府采购 | 未招标、化整为零、单一来源违规、定点采购不规范 |
| XM | 项目管理 | 立项/概算/验收/决算/转固/产权办理不规范 |
| FY | 费用支出 | 接待/差旅/会议/培训/津贴补贴/福利等超标或不合规 |
| SJ | 审计监督 | 整改不到位、同类问题重复、对下属单位监督不到位 |
| KJ | 会计基础 | 科目错误、凭证不规范、收入核算不完整、折旧未计提 |
| NK | 内部控制 | 制度脱节、审批倒置、重大事项未集体决策、合同台账缺失 |
| SW | 税务票据 | 未代扣代缴、发票类型错误、票据与业务不符 |
| QT | 其他 | 小金库、事企交叉、违规收受礼品礼金 |

#### 分类优先级规则

问题交叉时，按直接审计认定对象归主类：
- 资产入账/转固/盘点/处置 → ZC
- 预算编制和执行约束 → YS
- 现金/公务卡/账户/缴库 → ZJ
- 接待/差旅/培训/津贴等支出标准 → FY
- 往来款长期未清理/费用挂账 → NK
- 报销凭证不完整但核心是费用违规 → 归对应费用类，事实中补充 KJ 关联

#### 单条问题标准结构

```
【问题编号】X-XX
【问题类型】[编码] 类别名称
【事实描述】经审计核查，[时间/凭证号]，[单位][行为]，涉及金额X,XXX.XX元。
【定性问题】上述事项不符合《法规全称》第X条的规定。
【涉及金额】X,XXX.XX元
```

#### 写作规范

- 使用正式、克制、可核验的审计报告语言
- 事实描述只写证据能支撑的内容，不补造时间/金额/文号
- 金额统一 `X,XXX.XX元`；不明确时写"金额需进一步核实"
- 法规引用优先国家层面；不确定时标注"法规依据需人工核验"，不编造
- 同类问题可合并，但保留每条关键事实和金额
- 明细表已有日期/凭证号/金额时，审计重述列只写问题事实，不重复表格信息

#### 信息不足时的处理

以下情形先追问再写（2-3 个关键问题）：
- 只有结论没有事实（"采购不规范"）
- 缺少主体/时间/金额/凭证依据
- 法规适用取决于地区/年度/单位性质但未提供
- 涉及高风险事项但事实链条不完整

#### 费用支出类常见二级类型

| 二级类型 | 典型情形 |
|---------|---------|
| 公务接待不规范 | 未附接待函/清单/点餐单，未填人数 |
| 差旅费不规范 | 满车报销、事由不充分、以考察名义旅游 |
| 会议培训不规范 | 未附通知/签到表/讲师资质，非定点场所 |
| 福利慰问不规范 | 慰问品/通讯费/个人费用依据不足 |
| 津贴补贴不规范 | 值班与出差冲突、重复发放、超范围 |
| 列支非本单位费用 | 项目成本中列支非本项目/非本单位费用 |

#### 政府采购类常见二级类型

| 二级类型 | 典型情形 |
|---------|---------|
| 未履行询价比价 | 未附询价手续/资料 |

### 6.1 关联方（古立特辅助）

| 步骤 | 程序 | 古立特工具 | 谁做 |
|------|------|-----------|------|
| 1 | 从股权结构识别法定关联方 | 知识判断 + 企查查数据 | 🤖+👤 |
| 2 | 从交易数据筛选异常交易对手 | `data_filter`（高频/大额/异常条件） | 🤖 |
| 3 | 关联交易汇总（按对手方/交易类型） | `data_group_summary` | 🤖 |
| 4 | 关联交易定价公允性分析 | `analytical_procedures`（与非关联方对比） | 🤖+👤 |
| 5 | 检查未披露的关联方关系 | — | 👤 |
| 6 | 关联方披露完整性核对 | — | 👤 |

**古立特自动识别异常交易对手的规则：**
- 同一对手方交易金额占比 > 20%
- 交易条件异常（无息借款、明显低价/高价）
- 期末突增的大额往来
- 与企业注册地址/法人相同的对手方
- 供应商参保人数为 0（空壳公司信号）
- 规律性转账（固定日期、整进整出）

#### 关联方认定判断规则（实务经验）

**核心原则：实质重于形式，不能只看持股比例。**

| 情形 | 判断 | 理由 |
|------|------|------|
| 持股 20% 但无董事席位 | 可能不构成重大影响（需具体分析） | 无参与决策的证据 |
| 持股 10% 但派驻董事+业务依赖 | ✅ 关联方 | 实质重大影响 |
| 持股 2% 无其他关系 | ❌ 不是关联方 | 无控制/影响 |
| 兄妹公司（同受母公司控制） | ✅ 互为关联方 | 同受控制 |
| 母公司高管控制的私人公司 | ✅ 关联方 | 关键管理人员关联方 |
| 联营企业 | ✅ 关联方 | 重大影响 |
| 供应商持股 5% 无其他关系 | ❌ 通常不是 | 无控制/影响 |
| 母公司代垫款项给子公司 | ✅ 关联交易 | 资金往来即交易 |

**关键判断因素：**
- 是否有董事/高管派遣权
- 是否存在业务依赖（技术/渠道/原材料）
- 是否有担保/抵押/资金往来
- 是否存在人事兼任

#### 关联方审计程序 checklist

```
获取阶段：
□ 获取完整股东名单（含历史股东追溯12个月）
□ 获取董监高名单（含曾任职单位）
□ 获取主要供应商/客户名单（工商穿透核查）
□ 获取关键管理人员控制的私人企业清单
□ 检查银行账户开户清单（识别隐藏关联方）

识别隐性关联：
□ 核查与大额供应商/客户之间的资金往来
□ 识别账外资金循环（虚构采购/销售）
□ 关注规律性转账（固定日期、整进整出）
□ 核心供应商实际控制人背景调查

交易测试：
□ 获取关联交易合同/订单/发票/付款记录
□ 核查交易价格与独立价格的差异
□ 核查交易条款与市场对比
□ 确认交易的商业必要性
□ 评估对报表的影响（是否需要调整）

披露检查：
□ 关联方关系披露完整性
□ 关联交易金额/条款/条件披露
□ 关键管理人员薪酬披露（不论金额大小必须披露）
□ 合并范围内关联方往来已全额抵消
```

#### 关联交易定价公允性分析方法

| 方法 | 适用场景 | 可接受性 |
|------|---------|---------|
| 独立价格法 | 有可比市场交易 | 高 |
| 成本加成法 | 无市场交易但可计算成本 | 中 |
| 收益法（DCF） | 复杂交易 | 中，需假设支撑 |
| 成本法 | 无法采用其他方法 | 低 |

#### 未披露关联方交易的信号清单（问答6号）

> 类型：判断表
> 来源：审计准则问题解答6号
> 置信度：准则原文

> 与 audit.md「关联方识别方法论 12 维度」互补：12 维度偏工商数据交叉比对，本清单偏交易特征异常。

**未披露关联方"关系"特征：**
- 交易对手曾与被审计单位/实控人/关键管理人员存在关联关系
- 交易对手注册或办公地址、网址/IP/邮箱域名、名称与被审计单位或集团成员相同或接近
- 交易对手实控人/关键人员/购销关键岗位员工姓名与被审计单位管理层或其关联方重合
- 交易标的与交易对手或被审计单位经营范围不相关
- 交易对手系自然人或其控制，规模/性质与交易不匹配
- 仅能提供极有限的交易对手信息，网络也难检索到
- 当年新增/减少的异常重要客户或供应商，关注是否为新设公司且交易时间接近设立时间

**未披露关联方"交易"特征：**
- 金额重大或带来大额利润；交易频次少且接近资产负债表日或集中于特定期间
- 商业条款（价格/交付/付款/结算）与其他客商明显不同（如他人票据结算、该户现金交易）
- 付款人与销售合同/发票客户不一致，或收款人与采购合同/发票供应商不一致
- 与同一客户或其关联公司同时发生销售和采购
- 款项长期以债权债务形式挂账、购销货款久拖不结
- 不合常理且影响财务指标的大额政府补助（用途不明/无条件，真正来源可能是关联方）
- 交易涉及无物质基础或财务能力完成交易的第三方
- 互相提供担保；实控人/控股股东质押被审计单位股份用于自身其他主体
- 无合理理由的大额捐赠或债务豁免
- 银行明细账存在定期/规律性转入转出（可能存在资金集中管理/资金池安排）

#### 确认疑似关联方是否存在的程序（问答6号）

> 类型：流程
> 来源：审计准则问题解答6号
> 置信度：准则原文

舞弊风险较高且必要可行时可考虑：
1. 访谈重要股东/关键管理人员/治理层，或实施函证
2. 以大股东/实控人/关键管理人员为起点做背景调查，识别其控制或关联的主体
3. 获取当期新增客户/供应商清单，比对首次交易时间与对手注册成立时间是否重合
4. 用数据分析工具筛选：与非客商的大额资金往来、月初月末大额收付、规模频率异常、会计记录首次出现的对手
5. 亲自获取被审计单位企业信用报告，核对对外担保等是否已完整披露
6. 检查银行对账单中与疑似关联方/实控人/控股股东的大额资金往来是否有真实合理交易基础（必要时要求实控人提供其本人对账单）
7. 对疑似关联方实施背景调查+实地走访（观察经营场所、货物进出、现场询问），核对与员工/关联方名册
8. 关注关联方被注销或"非关联化"前的交易是否仍按关联交易披露，识别剥离亏损以增利的动机

## 集团审计与尽调通用规则

> 本节是跨阶段的通用判断规则，集团审计、尽调清单、年报分析、报表审查共用。

### 一、八大业务循环框架

业务循环是审计和尽调通用的组织维度——按"业务流"而不是按"科目"分组，便于做穿行测试、内控理解和资料清单生成。

**标准八大循环：**

| 循环 | 主要业务流 | 涉及科目（举例） | 关键控制点 |
| --- | --- | --- | --- |
| 1. 货币资金循环 | 收付款、银行账户管理、现金管理 | 库存现金、银行存款、其他货币资金 | 印鉴分管、对账、大额审批 |
| 2. 销售与收款循环 | 报价 → 合同 → 发货 → 开票 → 回款 | 营业收入、应收账款、合同负债、应收票据 | 信用审批、发货与开票分管、回款核对 |
| 3. 采购与付款循环 | 请购 → 询比价 → 合同 → 收货 → 入库 → 付款 | 应付账款、预付账款、原材料、应付票据 | 三单核对（订单/收货/发票）、付款审批 |
| 4. 生产与仓储循环 / 服务成本循环 | 制造业：领料 → 生产 → 入库 → 销售出库；服务业：人天/工时归集 | 在产品、库存商品、原材料、营业成本 | 工时单、料工费归集、盘点制度 |
| 5. 工薪与人事循环 | 入职 → 考勤 → 计薪 → 发放 → 离职 | 应付职工薪酬、管理费用-工资、销售费用-工资 | 人员名单管理、考勤审批、个税申报 |
| 6. 长期资产管理循环 | 取得 → 折旧/摊销 → 处置 | 固定资产、在建工程、无形资产、长期待摊费用 | 立项审批、转固时点、处置审批 |
| 7. 筹资与投资循环 | 借款 → 投资 → 利息/股利 → 还本付息/退出 | 短/长期借款、长期股权投资、其他权益工具投资 | 借款审批、投资决策、关联方识别 |
| 8. 费用管理循环 | 申请 → 审批 → 报销 → 入账 | 管理费用、销售费用、财务费用、研发费用 | 费用标准、审批权限、票据合规 |

**适用规则：**

- 制造业用"生产与仓储循环"，纯服务业用"服务成本循环"，两者不并存
- 行业特殊业务循环单独识别（如金融业的资金交易循环、保险业的承保赔付循环），不强行套八大
- 用户明确表示某循环不涉及（如纯持有型公司没有生产循环），在尽调清单/底稿框架里标"不适用"，不假装有
- 八大循环 ≠ 科目维度，互补使用：业务循环回答"业务怎么跑"，科目维度回答"账面怎么记"

### 二、主要经营公司判定五标准

集团审计或尽调时，子公司多则需筛选"主要经营公司"作为重点工作对象。**满足任一即可判定：**

| 标准 | 阈值 | 备注 |
| --- | --- | --- |
| 收入 | 最近一会计年度营业收入占集团合并收入的 30% 以上 | 量化指标 |
| 资产 | 最近一会计期末总资产占集团合并总资产的 30% 以上 | 量化指标 |
| 利润 | 最近一会计年度净利润占集团合并净利润的 30% 以上 | 量化指标，注意亏损子公司用绝对值 |
| 业务重要性 | 承担集团核心业务、关键技术或主要市场运营职能 | 定性指标 |
| 风险重要性 | 涉及高风险业务、重大投资或重大或有负债 | 定性指标 |

**应用规则：**

- 量化指标互不独立——一个子公司可能仅占资产 5% 但占利润 40%，仍判主要经营
- 定性指标必须有书面理由，不能"感觉重要就是重要"
- 集团组件重要性双限值（80%/90%）和主要经营公司判定是两件事：前者决定组件重要性怎么算，后者决定哪些子公司纳入工作范围
- 工作产物按"集团合并 / 母公司单体 / 主要经营公司单体"三层分别归档，避免混在一起

### 三、数据对比期间智能判断

提供资料、计算波动、做分析性程序时，是否需要"上一年可比数据"按以下规则判断：

**无需上一年可比数据**（满足任一）：

1. 工作期间已涵盖两个或两个以上完整会计年度（如 2024.1.1 - 2025.12.31）
2. 工作期间已包含完整年度的同期对比（如 2024.1-6 vs 2025.1-6）
3. 用户明确说明已在工作期间内实现同期对比

**需要上一年可比数据**：

1. 工作期间仅为单个完整会计年度（如仅 2025 年）
2. 工作期间为年度内的部分期间且无同期数据（如仅 2025.1-6，无 2024.1-6）
3. 工作期间虽跨年度但无法实现有意义的同期对比

**应用：**

- 尽调资料清单生成时按此规则标注每项财务资料的"期间要求"
- 分析性程序、变动率分析按此规则决定是否拉取上一年数据
- 期间判断有歧义时主动反问用户，不默认全要

## 报表调整体系（试算平衡表）

### 整体架构

报表调整体系是审计项目的"中枢神经"——所有科目的审计调整最终汇集于此，生成审定报表。

**四层结构：**

```
输入层 → 计算层 → 分析层 → 输出层
```

| 层级 | 功能 | 内容 |
|------|------|------|
| 输入层 | 录入原始数据 | 项目参数 + 各主体未审数 + 调整分录 |
| 计算层 | 自动归集计算 | 调整分录汇总 + 审定数生成 + 现金流测算 |
| 分析层 | 波动分析与指标 | 分析性程序 + 财务比率 + 非经常性损益 + EPS |
| 输出层 | 审定报表 | 四表一注（资产负债表/利润表/现金流量表/权益变动表） |

### 输入层

#### 2.1 项目参数

| 参数 | 说明 |
|------|------|
| 被审计单位名称 | 自动过入所有底稿表头 |
| 财务报表截止日/期间 | 上期起止 + 本期起止 |
| 编制人 / 复核人 | 含日期 |
| 金额单位 | 人民币：元 |
| 合并主体列表 | 母公司 + 各子公司（可扩展列） |

#### 2.2 合并工作底稿（未审数录入）

每张报表的工作底稿结构统一：

```
科目名称 | 行次 | 母公司 | 子公司A | 子公司B... | 合计 | 借方调整 | 贷方调整 | 审定数
```

**核心公式：**
- 合计 = 母公司 + 子公司A + 子公司B + ...
- 审定数 = 合计 + 贷方调整 - 借方调整（资产类）
- 审定数 = 合计 + 借方调整 - 贷方调整（负债/权益/收入类，方向相反看科目性质）

**覆盖报表：**

| 报表 | 科目数（含小计） | 说明 |
|------|-----------------|------|
| 资产负债表-资产 | ~100行 | 含坏账准备、净额行 |
| 资产负债表-负债及权益 | ~60行 | 含股东权益 |
| 利润表 | ~70行 | 含其他综合收益、EPS |
| 现金流量表 | ~90行正表 + ~180行补充资料 | 经营/投资/筹资三大类 |
| 所有者权益变动表 | ~30行 × 每个主体 | 12列（股本/资本公积/库存股/OCI/盈余公积/未分配利润等） |

#### 2.3 调整分录录入

**资产负债表/利润表调整分录格式：**

| 字段 | 说明 |
|------|------|
| 编号 | 顺序号 |
| 索引号 | 对应底稿索引 |
| 摘要 | 调整原因 |
| 借方科目 | 从科目名称表选取 |
| 借方二级科目 | 明细（可选） |
| 贷方科目 | 从科目名称表选取 |
| 贷方二级科目 | 明细（可选） |
| 金额 | 调整金额 |
| 影响资产负债表金额 | +/- |
| 影响利润表金额 | +/- |

**现金流量表调整分录格式：**

| 字段 | 说明 |
|------|------|
| 编号 | 顺序号 |
| 索引号 | 对应底稿索引 |
| 摘要 | 调整原因 |
| 借方科目 | 现金流量表项目（借方=收到现金） |
| 贷方科目 | 现金流量表项目（贷方=支付现金） |
| 金额 | 调整金额 |
| 备注 | 补充说明 |

**现金流量表调整方向规则：**
- 正表：借方=收到现金，贷方=支付现金
- 补充资料：借方=减少，贷方=增加
- 期末现金：借方=减少，贷方=增加
- 期初现金：借方=增加，贷方=减少

### 计算层

#### 3.1 调整分录汇总

按科目自动归集所有调整分录：

```
对每个科目：
  借方金额 = SUM(该科目出现在借方的所有分录金额)
  贷方金额 = SUM(该科目出现在贷方的所有分录金额)
  调整数 = 借方金额 - 贷方金额（资产类）
         = 贷方金额 - 借方金额（负债/权益/收入类）
```

科目表覆盖范围：~370个科目（资产负债表科目 + 损益类科目），含：
- 一级科目（货币资金、应收账款...）
- 备抵科目（坏账准备、跌价准备...）
- 净额科目（应收账款净额 = 应收账款 - 坏账准备）

#### 3.2 现金流量表测算（间接法倒推）

对每个现金流量表项目，从审定后的资产负债表/利润表数据倒推：

**示例：销售商品、提供劳务收到的现金**

```
= 主营业务收入
+ 其他业务收入
+ 销项税额
+ 应收票据减少（期初-期末）
+ 应收账款减少（期初-期末）
+ 预收账款增加（期末-期初）
+ 当期收回前期核销坏账
- 票据贴现利息
- 本期计提坏账准备
- 应收票据背书转让金额
- 应收账款债务重组/非现金抵偿
- 应收账款直接抵减费用
- 预收账款中预提费用增加
- 应收账款核销
- 其他非经营活动项目剔除额
```

**测算结果验证：**
```
测试结果 = 按公式计算的金额
报表金额 = 现金流量表中的实际金额
测试差异 = 测试结果 - 报表金额
差异原因 = （需人工填写解释）
```

每个现金流量表项目都有类似的测算公式，涉及的会计科目在公式旁标注。

**现金流量表特殊项目分类规则（实务经验）：**

| 项目 | 分类规则 |
|------|----------|
| 定期存款 < 3个月 | 现金及现金等价物 |
| 定期存款 ≥ 3个月 | 投资活动（购建/收回） |
| 结构性存款（可随时支取） | 现金及现金等价物 |
| 结构性存款（有锁定期） | 投资活动 |
| 票据保证金（票据未到期） | 其他货币资金，不属于现金 |
| 票据保证金（票据已承兑解冻） | 现金流入 |
| 母公司代垫子公司款项 | 合并抵消后不影响净额 |
| 收到的税费返还 | 经营活动 |
| 购买理财产品 | 投资活动（投资支付的现金） |
| 赎回理财产品 | 投资活动（收回投资收到的现金） |
| 理财收益 | 投资活动（取得投资收益收到的现金） |

**现金流量表分析指标（古立特自动计算）：**

| 指标 | 公式 | 判断标准 |
|------|------|----------|
| 净利润现金含量 | 经营活动现金流净额 ÷ 净利润 | >1 利润质量高，<1 存在赊销或虚构可能 |
| 自由现金流 | 经营活动现金流净额 - 资本支出 | >0 可支配现金充裕，<0 依赖外部融资 |
| 造血能力 | 经营活动现金流净额 | >0 正常，<0 营运资本恶化 |

#### 3.3 非经常性损益计算

按证监会定义的 22 项非经常性损益逐项列示：

```
非经常性损益总额 = Σ(22项)
减：所得税影响数
= 非经常性损益净额
减：少数股东权益影响额（税后）
= 归属于公司普通股股东的非经常性损益
```

补充资料区列示各项来源（营业外收入/支出、其他收益、资产处置收益、公允价值变动等），与附注交叉核对。

#### 3.4 EPS 与 ROE 计算

**净资产收益率（两种口径）：**
- 全面摊薄 = P ÷ E（期末净资产）
- 加权平均 = P ÷ (E0 + NP÷2 + Ei×Mi÷M0 - Ej×Mj÷M0 ± Ek×Mk÷M0)

**每股收益（两种口径）：**
- 基本 EPS = P ÷ S
- 稀释 EPS = (P + 稀释性潜在普通股利息×(1-税率) - 转换费用) ÷ (S + 增加的普通股加权平均数)

其中：
- P = 归属于公司普通股股东的净利润（或扣非后）
- E0 = 期初净资产，E = 期末净资产
- S = S0 + S1 + Si×Mi÷M0 - Sj×Mj÷M0 - Sk
- M0 = 报告期月份数（通常12）

### 分析层

#### 4.1 分析性程序（四张报表统一结构）

每张报表的分析底稿结构：

```
科目 | 上年审定金额 | 结构比% | 计划阶段(未审) | 结构比% | 变动金额 | 横比% | 变动结构比 | 完成阶段(审定) | 结构比% | 变动金额 | 横比% | 变动结构比
```

**两个阶段：**
- 计划阶段：上年审定 vs 本期未审 → 识别风险领域，确定审计重点
- 完成阶段：上年审定 vs 本期审定 → 验证审计结论合理性

**关注指标：**
- 横比%（变动幅度）> 阈值 → 异常波动
- 结构比变动 → 报表结构变化
- 从无到有 / 从有到无 → 新增/消失科目

#### 4.2 财务比率分析

23 个核心指标，四列对比：

| 类别 | 指标 | 计算公式 |
|------|------|----------|
| 变现比率 | 流动比率 | 流动资产÷流动负债 |
| | 速动比率 | 速动资产÷流动负债 |
| 资产管理效率 | 存货周转率 | 销售成本÷平均存货 |
| | 存货周转天数 | 360÷存货周转率 |
| | 应收账款周转率 | 营业收入÷平均应收账款 |
| | 应收账款周转天数 | 360÷应收账款周转率 |
| | 流动资产周转率 | 营业收入÷平均流动资产 |
| | 总资产周转率 | 营业收入÷平均总资产 |
| | 营业周期 | 存货周转天数+应收账款周转天数 |
| 负债比率 | 资产负债率 | 负债总额÷资产总额 |
| | 产权比率 | 负债总额÷股东权益 |
| | 有形净值债务率 | 负债总额÷(股东权益-无形资产净值) |
| | 已获利息倍数 | 息税前利润÷利息支出 |
| 盈利能力 | 销售净利率 | 净利润÷收入 |
| | 销售毛利率 | (收入-成本)÷收入 |
| | 资产利润率 | 净利润÷平均总资产 |
| | 净资产收益率 | 净利润÷平均净资产 |
| | 主营业务毛利率 | (主营收入-主营成本)÷主营收入 |
| 其他 | 营业费用率 | 营业费用÷收入 |
| | 管理费用率 | 管理费用÷收入 |
| | 现金流动负债比率 | 经营现金净流入÷流动负债 |
| | 每股经营现金流量 | 经营现金净流入÷股本 |
| | 每股净现金流量 | 现金净流入÷股本 |

对比维度：上年度指标 / 行业指标 / 报告期未审 / 报告期审定

### 输出层

#### 5.1 审定报表（正式报表）

从合并工作底稿的"审定数"列自动引用，生成可打印的正式报表：

| 报表 | 结构 | 列 |
|------|------|-----|
| 合并资产负债表 | 项目 + 附注 + 年末数 + 年初数 | 4列 |
| 合并利润表 | 项目 + 附注 + 本期金额 + 上期金额 | 4列 |
| 合并现金流量表 | 项目 + 附注 + 本期金额 + 上期金额 | 4列 |
| 合并权益变动表 | 项目 + 股本/资本公积/.../少数股东/合计 | 10列 |

每张正式报表底部含勾稽校验行（合计核对）。

#### 5.2 勾稽关系

**资产负债表内部：**
- 流动资产合计 = Σ各流动资产科目
- 非流动资产合计 = Σ各非流动资产科目
- 资产总计 = 流动资产合计 + 非流动资产合计
- 负债合计 + 股东权益合计 = 资产总计

**利润表内部：**
- 营业利润 = 营业总收入 - 营业总成本 + 其他收益 + 投资收益 + 公允价值变动 + 资产处置收益 - 资产减值 - 信用减值
- 利润总额 = 营业利润 + 营业外收入 - 营业外支出
- 净利润 = 利润总额 - 所得税费用
- 综合收益总额 = 净利润 + 其他综合收益税后净额

**跨表勾稽：**
- 资产负债表"未分配利润"期末 = 期初 + 本期净利润 - 分配
- 现金流量表"期末现金" = 资产负债表"货币资金"（需调整非现金等价物）
- 权益变动表"期末余额" = 资产负债表"股东权益"各项

> 来源：《银行函证工作操作指引》第二部分
> 置信度：准则原文

| 判断顺序 | 情形 | 选用格式 |
|---|---|---|
| 1（必填条件） | CPA 已掌握被审计单位账面信息，需银行**核对相符/不符** | 格式一 |
| 2（替代条件） | CPA 仅有账号，需要**银行根据系统填写**具体信息 | 格式二 |
| 3（专用） | **验资业务**（含拟设立公司、外商投资） | 验资业务银行询证函（独立格式） |
| 4（不适用） | 银行间同业往来（同业存放、拆借、转贴现等） | 不适用本格式，参考 CSA 1612 单独发函 |

格式一与格式二**均符合 CSA 1312 中"积极式函证"定义**，纸质和数字方式均可。

### 填写规则

> 类型：规则
> 来源：《银行函证工作操作指引》第二部分（一）
> 置信度：准则原文

1. 一份询证函**只列示一个函证基准日**。
2. 无需函证的项目**应当用斜线划掉**。
3. 确认无此业务的项目**应当填写"无"**——银行**应当对填"无"的信息予以核实并反馈**。
4. **不得留白**——每项必须填写信息、"无"或斜线，三选一。
5. 字号**不小于五号**、黑色、1.5倍行距。
6. 纸质函证**应当加盖骑缝章**（用被审计单位预留银行签章作为骑缝章）；数字函证不强制要求。
7. 来函信息**应当采用打印方式**；回函信息可手工填写或打印，但同一份询证函填写方式应当统一。
8. 手工填写**应当使用不易涂改的签字笔**——直接修改、涂抹、把回函结论填在其他项目栏，**均属于不规范做法**。
9. 询证函**应当填写函证编号**。
10. 银行询证函**空白处不足时**，可另行添加附页列示，并在附页签章或电子签名。

### 集中处理与公示要求

> 类型：规则
> 来源：财会〔2022〕39号 + 操作指引第一部分（一）（二）（三）
> 置信度：准则原文

**对会计师事务所的硬性要求：**

- 应当安排**专门部门或岗位**集中发送、收回银行询证函。
- 应当**采用注协公示地址作为邮寄地址**，直接发出询证函并直接从银行获取回函。
- 应当**始终对询证函全过程保持控制**——确定询证内容、选择被询证者、填写、发出、跟进。
- 应当**及时更新注协统一监管平台中的事务所信息**（地址、联系方式），防止因公示信息过时被退函。
- 对询证函（含回函）所列信息**履行恰当的保密义务**。

**对银行业金融机构的硬性要求：**

- 应当在总行/总部网站、微信公众号等公开渠道公示：受理函证机构与联系方式、受理事项（1—14项及附表）、函证范围（"函证收件人"可填写的总分支机构范围）、回函用章适用范围、收费标准、被审计单位授权预留签章要求、识别函证范围所需信息（格式二适用）。
- **公示后对符合公示条件的申请不应拒绝**。
- 1—13项（含附表）规范业务**应做到集中办理**；个别因特殊原因暂未集中办理的，应在公开渠道清晰公示由集中办理机构以外的经办机构办理。

### 跟函（现场办理）规则

> 类型：流程
> 来源：操作指引第一部分（三）
> 置信度：准则原文

[场景：验资业务、其他时效性强的特殊情况]

1. 跟函**仅适用于时效性强的特殊情况**，不是常规函证方式。
2. 跟函人员信息**应填写在询证函"联系人"栏**。
3. 事务所应向银行提供**经授权加盖事务所印章的介绍信**，跟函人员**携带身份证件**办理。事务所已实现集中函证的，介绍信可加盖集中处理机构印章。
4. 特殊情况需修改联系人时，**应在介绍信中注明**。
5. 银行办理后可**直接交付跟函人员**，或邮寄至询证函指定的事务所收件人。

### 回函时限

> 类型：规则
> 来源：操作指引第一部分（七）
> 置信度：准则原文

| 情形 | 时限 |
|------|------|
| 数字化回函 | **10 个工作日**（优先保障） |
| 纸质回函（已实现集中处理且回函确有困难） | **可适当延长至 14 个工作日**，仅适用于 2025 年底前 |
| 双方协商延期 | 按协商结果，原则上仍按上述时限执行 |

### 退函与通知

> 类型：流程
> 来源：操作指引第一部分（九）
> 置信度：准则原文

[场景：询证函填写不符合规定 / 未成功缴费 / 其他需事务所配合事项]

1. 银行**应当在收到询证函 3 个工作日内**书面通知事务所——可采用纸质、微信小程序、电子邮件等方式。
2. 2025 年底前对已实现集中处理且 3 个工作日内通知确有困难的，年审高峰期**可延长至不超过 5 个工作日**。
3. 通知内容**应清晰描述**：原询证函发函编号、存在问题、需事务所配合的事项。
4. 退函**应采用微信小程序或电子邮件等非口头方式**说明原询证函发函编号及具体原因。
5. 事务所**应保证联系渠道畅通**，配备具有经验的人员及时接收并应对银行通知。

⚠️ 项目经办人手机/邮箱可填写在"联系电话"和"电子邮箱"栏：
- "联系电话"在事务所公示电话基础上，可加填项目经办人手机用于接收银行短信。
- "电子邮件"可填写经办人工作邮箱，用于接收业务通知。

### 回函效力与服务收费

> 类型：规则
> 来源：操作指引第一部分（四）（五）
> 置信度：准则原文

**回函效力：**

- 数字化回函与纸质回函**具有同等法律效力和证明力**。
- 银行**对回函内容的真实性、准确性负责，不得以任何理由、任何方式免责**。
- 数字化回函内容不能覆盖前 13 项询证项目的，**应当以纸质方式辅助回函**。

**服务收费：**

- 收费属市场调节价的，银行**应当依法制定并公示**收费标准（含被审计单位拒付/拖欠的处理办法）。
- 注册会计师**应提请被审计单位提供准确、可扣款的账户**（写入询证函扣款授权栏）。
- 指定账户余额不足以扣划时，银行可现场缴费——通过事务所提请被审计单位现场缴费。
- 被审计单位**不同意支付或未及时补缴**的，银行**可退函**。

### 被审计单位授权预留签章

> 类型：规则
> 来源：操作指引第一部分（六）
> 置信度：准则原文

[场景：纸质询证函或电子授权]

1. 银行可校验的签章 = 被审计单位**开户时的预留签章** + **约定办理函证业务时需加盖的签章**（含具体签章组合）。
2. 被审计单位**应在询证函预留签章处填写日期**。如遇印鉴变更，应尽快在填写日期前完成。
3. 询证函预留签章处按规定加盖签章或采用有效电子授权，**即代表被审计单位同意**就询证函进行函证并扣款。
4. 银行**不得额外要求**加盖会计师事务所印章。

⚠️ 多账户预留签章不同时的处理：
- 若其中一个账户为询证函指定扣款账户，事务所**应要求被审计单位加盖扣款账户预留签章**。
- 银行校验扣款账户预留签章即可办理回函。

⚠️ 销户后签章处理（操作指引第二部分（一）第 5 条）：
- 全部账户已销户、原预留签章未注销 → 校验销户前预留签章办理回函。
- 销户后原预留签章已注销 → 核验以往留存的签章资料。
- 销户后签章发生变更 → 银行可要求事务所补充提供：单位有效证件材料、印鉴变更连续性证明（如账户变更申请书、营业执照、开户许可证、法人身份证、公安机关指定印章厂出具的旧印章变更证明、旧印章销毁证明、新印章刻制证明等）。

### 银行回函内控要求

> 类型：规则
> 来源：操作指引第一部分（八）
> 置信度：准则原文

[场景：审计师执行银行内控测试时的判断依据]

1. 银行回函**应当实现处理人员与复核人员不相容职责的分离**——内部回函操作记录应体现处理过程及复核人员复核签章（含数字签名）。
2. 人工回函时，回函人员、复核人员**应签字并加盖有效印章**。
3. 系统处理回函时，应**具备和使用可靠的数字签名**（即有效电子印章）。已实现集约化或数字化的，**回函可由系统处理自动生成签字代替**。
4. 银行**应规范回函用章种类**，明确、统一回函用章适用范围。鼓励使用**有防伪或校验功能的银行印章**。
5. 集中处理部门回函显示不符、需开户机构进一步解释时，开户机构**可经内部授权后提供说明，加盖开户机构有效印章**。

### 14 项填写要点（避坑速查）

> 类型：判断表
> 来源：操作指引第三部分
> 置信度：准则原文

#### 第 1 项 银行存款

- "银行账号"**应当与银行对账单上显示的账号完全一致**。
- 本外币或多币种账户**按币种分行填列**，"账户余额"填**原币金额**。
- "账户类型"按《人民币银行结算账户管理办法》分类：基本户/一般户/专用户/临时户。专用户可在"备注"栏注明账户性质或资金性质。
- 大额存款、通知存款、定期保证金存款、结构性存款等**比照定期存款填写**。
- "起始日期/终止日期"**仅适用于约定期限账户**——活期存款填"不适用"，"备注"填"活期"。
- "利率"指**函证基准日适用的年化利率**（百分比或数字至小数点后四位）。简单年化利率不适用时，"利率"填"见备注"，"备注"说明浮动利率条款。

⚠️ "其他使用限制"包含什么 / 不包含什么：
- **包含**：反洗钱/案件触发的外部限制、被审计单位自身行为导致的资金无法支取。
- **不包含**：正常设立的资金托管账户、资金监管账户的使用限制。
- 仅部分金额受限的，**应注明截至函证基准日受限部分金额**。

⚠️ 资金归集（资金池）账户的特殊填写：
- "账户余额"填**实际余额**——已实际划出的资金（如已上存集团归集账户的金额）**不应包括在内**。
- 与资金池相关的进一步信息（如已上下归集金额）**填在附表**。
- 归集账户、被归集账户**账号不同的应分别填写附表**。

#### 第 2 项 银行借款

- 包含：一般贷款和垫款、贸易融资项下贷款和垫款、票据垫款。
- 银团贷款情形下，"银行借款"**填被函证银行参贷的自身份额部分金额**。
- 借款人本金或利息**逾期未付**的——"备注"栏说明：逾期起始日期、逾期金额（含本息）。鼓励本息分别列示。
- "借款账号"可填贷款账号、借据编号或银行结算账号——分别逐笔填写。
- "借款日期"按每笔借款放款或垫款的起始日期**逐笔填写**。
- 垫款类业务的"到期日期"可填"不适用"或"见备注"，备注说明垫款事实。

#### 第 3 项 注销的银行存款账户

- 仅函证**已注销的存款账户**——账号未注销、仅变更账户类型的不属于本项目。
- 注册会计师**应填写起止日期**，通常与询证函正文表述的审计年度一致；需追溯以前期间的，起始日期可早于审计年度。
- 账户名称、账号、币种**按注销时的账户信息填写**。

#### 第 4-5 项 委托贷款

- 仅指**一般性委托贷款**——不包括现金管理项下的委托贷款、住房公积金项下的委托贷款。
- "银行结算账号/借据编号/贷款账号"——填委托贷款协议约定的银行结算账号；未约定的填贷款账号或借据编号，"备注"说明编号类型。
- 同一银行结算账号对应多笔委托贷款的，**分别逐行填写**。

#### 第 6 项 担保

⚠️ 区分两类：
- **第（1）类——被审计单位为他人提供、以银行为受益人的担保**："担保余额/到期日"按一般担保 vs 最高额担保分别填列：
  - 一般担保 → 担保债权余额 = **主债权余额**；担保债权到期日 = **主债权到期日**。
  - 最高额担保 → 担保债权余额 = **被担保的全部债权余额**；担保债权到期日 = **被担保主债权到期日**。
  - 抵押或质押方式的，"备注"说明抵质押品类型、数量。
- **第（2）类——银行向被审计单位提供的担保**：保函业务、备用信用证业务等。"担保到期日"按银行开立保函/备用信用证的有效期填写。

#### 第 7 项 银行承兑汇票

- "抵（质）押品"指**保证金存款以外**的抵质押品。
- 到期日为非工作日时，银行按合同约定确认并及时与 CPA 沟通。
- **不包括**银行已垫付款项的情况——垫付款项填在第 2 项"银行借款"。

#### 第 8 项 已贴现未到期商业汇票

- "承兑人名称"指**商业承兑汇票承兑付款人 / 银行承兑汇票承兑付款行**。
- 银行已对该汇票办理转贴出、再贴出业务的，**仍在本项目反馈**。

#### 第 9 项 托收/提示付款的商业汇票

- 截至函证基准日**作为持票人且尚未收到款项**的商业汇票——含银行办理托收的、已提交提示付款申请的。

#### 第 10 项 不可撤销信用证

- "信用证金额" = 银行开立信用证协议（含"溢短装"）中**不可撤销的总金额**。协议约定百分比浮动金额时，按**不可撤销的最高比例、最大金额**填写。
- "未使用金额" = 剩余未被索偿的信用证余额（含未到单金额、已承兑但银行尚未支付的金额），受益人可索偿。

#### 第 11 项 外汇买卖合约

- 仅函证函证基准日**尚未实际交收**的合约——已成交未履约、已展期未履约、未平仓等。
- 函证基准日与交收日期相同**且已交收完毕**的合约**无需填写**。
- 按合约号码和汇率**逐笔填写**——一项合约编号下按合约汇率逐行填写。
- "类别"：即期结售汇/远期结售汇/掉期结售汇/即期外汇买卖/远期外汇买卖/掉期外汇买卖。
- 远期结售汇非固定期限交易：已完成交收的按实际交收日期填，未完成的填交收期间起止日期。

示例：银行向被审计单位卖出 100 美元、买入 700 元人民币，函证基准日未交收 → "贵行卖出币种"填"美元"，"贵行买入币种"填"人民币"，"汇率"填 7.0000。

#### 第 12 项 托管的证券或产权文件

- 仅指被审计单位**作为委托人**与银行签订托管合同的情形——**不包括**被审计单位作为金融产品管理人、银行为金融产品提供托管服务的情形。
- **包含**：未上市流通的股票、未在中央结算机构登记的股票或债券、全球存托凭证、存单、存款证实书、受益凭证、不动产权属证书等。
- **不包含**：保险箱租赁业务、抵质押文件或凭证、可向中央登记结算机构查询的股票债券。

#### 第 13 项 未到期银行理财产品

- 函证范围 = 被审计单位**认购、申购的已起息、未到期或未赎回**的、被函证银行**自己发行**的理财产品。
- 已赎回、已到期但处于待确认或待结算期间的，**按交易信息、最新产品净值如实填写**。
- "产品净值"——非净值型产品填基于持有份额的余额；净值型产品填**持有份额 × 函证基准日产品最新单位净值**。
- 函证基准日为非交易日 → 采用**前最近一个交易日的产品单位净值**。
- 封闭式产品 → "购买日"填实际起息日，"到期日"填投资协议约定到期日。
- 开放式产品 → "购买日/到期日"填"不适用"。

⚠️ 银行已设立理财子公司并由子公司管理的理财产品——银行应公示函证范围**是否包括理财子公司**：
- 不包括 → 注册会计师**应单独向理财子公司发函**。

#### 第 14 项 其他

- 补充说明 1—13 项的内容 + 注册会计师认为重大且应予函证的其他事项。
- 银行应加快集中处理：①已签订合同但尚未提供资金的贷款；②除外汇买卖外的其他衍生品交易、贵金属交易等。
- 具备条件的银行可回复**数字人民币相关信息**（钱包 ID、账户余额）。
- 不存在补充事项的，注册会计师可不函证第 14 项（划线处理）。

#### 附表 资金归集账户具体信息

- 附表是**可选函证内容**——CPA 根据风险评估判断是否函证。
- "截至函证基准日拨入或拨出资金余额" = 函证基准日的**应收应付余额**（不是函证期间的发生额）= 资金归集业务范围内、对银行账户资金调拨后的**累计结果**。
- 仅列示资金池业务**账户间内部划拨的累计净余额**——不涉及资金池业务账户与资金池外其他账户的往来信息。
- 函证基准日**已实际销户或终止资金池业务的账户不涉及填写**。

### 资金池函证案例（实际余额 vs 累计净额）

> 类型：示例
> 来源：操作指引附 4
> 置信度：准则原文

[场景：A 股集团公司 X 与某银行签资金归集协议，对集团内部三级账户进行资金集中管理]

**背景：**
- X 集团母公司 X 主账户：001
- 子公司 B（省级）二级账户：020，每日保留 10 万元，超额自动归集到 001
- 分支机构 C（市级）三级账户：300，零余额管理，每日上划至 020

**截至 20××年 12 月 31 日的实际状态：**

| 项目 | 母公司 X | 子公司 B | 分支机构 C |
|---|---|---|---|
| 资金池-本期划出（用括号表示） | (200,000) | (1,000,000) | (100,000) |
| 资金池-本期划入 | 700,000 | 300,000 | 300,000 |
| 自有资金收支增减 | 1,000,000 | 800,000 | (200,000) |
| **20××.12.31 实际余额** | **1,500,000** | **100,000** | **0** |

**函证三步走：**

**Step 1—主账户（M1 事务所对 X 母公司 001 函证）：**

- 第 1 项"银行存款"："账户余额"填 **1,500,000**（X 主账户实际余额）；"是否资金池账户"填"是"。
- 附表第 1 行：资金提供机构 = 子公司 B（账号 020），资金使用机构 = 母公司 X（账号 001），"截至函证基准日拨入或拨出资金余额"填 **-500,000**（拨入填负数）。

> 解释：B 在 20×× 年累计上划 700,000、X 累计下拨 200,000，**净额** = B 应收 X = 500,000。从 X 视角看是 B 拨入 500,000，按"拨入填负数"填 -500,000——**不是函证期间发生额 700,000 和 200,000**。

**Step 2—二级账户（M2 事务所对 B 公司 020 函证）：**

- 第 1 项"银行存款"："账户余额"填 **100,000**（B 实际余额）。
- 附表第 1 行：资金提供机构 = 子公司 B，资金使用机构 = 母公司 X，金额填 **500,000**（拨出填正数）。
- 附表第 2 行：资金提供机构 = 子公司 B，资金使用机构 = 分支机构 C，金额填 **200,000**（B 净下拨给 C 的累计差额）。

**Step 3—三级账户（C 分支机构 300）：**

C 的 300 账户为零余额、纯日常零星费用支出账户、无业务收入、无借款担保——M2 事务所**评估其满足 CSA 1312 第十二条**"对财务报表不重要且与之相关的重大错报风险很低"，**不再对 C 单独函证**。

⚠️ 关键判断：函证附表填的是**截至函证基准日的累计净额**，不是函证期间的发生额——这是资金池函证最容易踩的坑。

### 验资业务银行询证函

> 类型：流程
> 来源：操作指引第二部分（四）+ 附 3
> 置信度：准则原文

[场景：CPA 按 CSA 1602 执行验资业务——审验股东实缴出资、金融产品投资人入缴投资份额等]

**与一般银行函证的差异：**

- **使用专用格式**（操作指引附 3）——不是格式一/格式二。
- 验资银行函证**时效性强**——**可采用跟函**（现场办理）方式。
- 询证函栏目少，仅含：缴款人、缴入日期、账户名称、账户性质、银行账号、币种、金额、款项用途、款项来源（境内/境外，外商投资适用）。

**填写要点：**

| 情形 | 处理 |
|------|------|
| 截至回函日出资已被转出或未明确标明资金用途 | 银行**根据实际情况回函**——可仅针对资金到位情况回函，加"未写明款项用途"等说明 |
| 缴入款项的账户为**专用账户** | 银行根据与被审计单位约定的具体款项用途确认 |
| 缴入款项的账户为**非专用、非验资临时存款账户** | CPA 填"款项用途"为"不适用"；银行回函可表述"未与单位就款项用途进行约定" |
| 特殊验资场景（如 IPO 募集资金转入承销商账户的资金验证） | CPA**原则上发函前与银行沟通**——避免填写信息与银行无关或银行无法掌握 |

**纸质验资函证 → 加盖骑缝章**（用预留银行签章作骑缝章）；数字函证不强制。

### 货币资金函证实操检查单（操作流程）

> 类型：流程
> 来源：一线审计实务（Kevin 自拟，2026-05-30）
> 置信度：实务经验

[场景：审计现场实际执行银行函证时的动手清单——准则讲"应该怎样"，本清单讲"动手时别踩哪些坑"，与上文格式/法定流程互补]

**第一步：准备条件**

1. 获取银行开户清单、银行对账单（银行流水）。
2. 组织银行函证控制表。
3. 填写函证基准日、开户银行、银行账号、存款余额。
4. **无网银无法电子函证，无公章无法纸质函证**——两个前置硬条件，缺一对应方式走不通。
5. 不同银行预留签章要求不同，发函前应逐家核对。实务案例：济南农商行、北京银行要求在预留印鉴处及骑缝处加盖公章、财务章、法人章。

**第二步：电子函证（优先）**

6. **必须先尝试电子函证，不得跳过电子函证直接用纸质函证**。
7. 电子函证无银行存款余额数据则无法正常回函，**不能填"无"**。

**第三步：纸质函证（电子不可行时）**

8. 纸质函证的银行存款余额**可以填"无"**（与电子函证相反）。
9. 发出条件：格式一有银行存款信息即可发出；格式二有扣款信息即可发出。
10. 打印纸质函证前应与银行核对：抬头、盖章方式、邮寄地址、联系人、联系电话。港澳台及其他地区还要问清函证格式、处理函证部门的电子邮件地址（电话多数打不通）。

**第四步：填写与发出**

11. 询证函联系人**必须留发函人本人的联系方式**（电话、电子邮箱），**不得填事务所总所的统一邮箱**——否则回函/通知收不到。
12. 必须填写经办人（企业出纳/会计）的姓名、职位、联系电话。
13. **不要让客户自己把询证函寄到事务所，应先收回、查验后再由审计人员控制寄发**——保持函证全过程控制（呼应操作指引"注册会计师始终对函证全过程保持控制"）。
14. 万一客户已自行寄出，应获取寄件快递单号和面单照片，以便在函证系统中登记追踪。

**第五步：回收**

15. 及时督促盖章/授权，及时获取、及时解密、及时与账面核对。
16. **每天查询 1 次回函**——及时获取回函文件与回函结果，将回函与企业账面、银行对账单三方核对，核对结果留档；不符项及时汇报。

**第六步：异常处理**

17. 公章丢失未销毁：先补办或寻找公章——无公章既无法发函，也无法打印银行对账单/银行流水，函证程序全部受阻。

⚠️ 电子 vs 纸质的关键差异（最易错点）：
- 电子函证无余额数据 → 无法回函，**不能填"无"**。
- 纸质函证无余额数据 → **可以填"无"**。

⚠️ 第 5、11 条中的"加盖三章""寄到总所"等属特定银行/特定事务所惯例，不是普适规则——实际执行以当地银行公示要求和本所规定为准。

### 古立特生成银行函证的规则

> 类型：流程
> 来源：古立特工具规范
> 置信度：实务经验

当 `confirmation_letter`(type=bank) 被调用时：

1. **格式选择**：默认生成格式一（CPA 填写、银行核对相符/不符）。
2. **必填项目**：第 1 项（银行存款）和第 2 项（银行借款）必须生成。
3. **其他项目**：根据用户提供的数据自动判断——有数据则填具体信息，无数据则填"无"，明确无关的划斜线。
4. **回函栏**：必须包含"经本行核对，所函证项目与本行记载信息相符"和"经本行核对，所函证项目存在以下不符之处"两个选项 + 银行盖章位置 + 经办人/复核人/系统处理签字位置。
5. **被审计单位签章位置**：必须预留——含日期填写位置。
6. **函证编号**：自动生成唯一编号。
7. **回函地址**：取自项目上下文中的事务所公示地址（联系人/电话/邮编/电子邮箱不得空白）。
8. **回函接收地址、扣款账号**：在询证函正文中明示授权，符合操作指引第一部分（一）和（五）要求。
9. **资金池场景**：当 `extra_columns` 含资金池字段时，自动按附表格式生成"截至函证基准日拨入或拨出资金余额"列，并在备注中标明"按资金归集协议，账户余额以实际余额方式呈现"。
10. **验资场景**：识别 `confirmation_type=verify_capital` 时切换到附 3 验资专用格式。

## 函证准则核心规则（CSA 1312）

> 银行函证标准格式（财会39号）见上节；本节是函证程序的准则原文硬规则，适用于所有函证（银行/应收/应付/借款等）。
> 详见：audit_if_lines.md 二/八/九/十四章（函证不回/不符/拒绝的决策树）

### 必须函证的范围

> 类型：规则
> 来源：CSA 1312 第十二、十三条
> 置信度：准则原文

1. **银行存款**（含零余额账户和本期内注销的账户）、**借款**及与金融机构往来的其他重要信息——**应当函证**，除非有充分证据表明对财务报表不重要且相关重大错报风险很低。不函证须在底稿说明理由。
2. **应收账款**——**应当函证**，除非有充分证据表明对财务报表不重要，或函证很可能无效。认为很可能无效的，应实施替代程序；不函证须在底稿说明理由。

### 全过程控制四要求

> 类型：规则
> 来源：CSA 1312 第十四条
> 置信度：准则原文

注册会计师对询证函**保持控制**，包括：①确定需确认/填列的信息 ②选择适当的被询证者 ③设计询证函（正确填列被询证者姓名地址、回函地址）④发出并跟进，必要时再次寄发。

### 积极式 vs 消极式

> 类型：判断表
> 来源：CSA 1312 第六、七、二十二条
> 置信度：准则原文

| 方式 | 定义 | 证据力 |
|------|------|--------|
| 积极式 | 要求被询证者无论是否相符都直接回函（确认信息或填列信息） | 高 |
| 消极式 | 仅在**不同意**时才回函 | 低 |

**消极式作为唯一实质性程序，必须同时满足四个条件（缺一不可）：**
1. 重大错报风险评估为**低水平**，且已获取相关控制运行有效性的充分适当证据
2. 总体由**大量小额、同质**的账户余额/交易/事项构成
3. 预期不符事项发生率**很低**
4. 没有迹象表明接收方不认真对待函证

### 回函结果处理规则

> 类型：流程
> 来源：CSA 1312 第十五至二十一条
> 置信度：准则原文

| 情形 | 处理 |
|------|------|
| 管理层不允许寄发询证函 | ①询问原因并就正当合理性收集证据 ②评价对重大错报风险（含舞弊）和其他程序的影响 ③实施替代程序。原因不合理或替代程序无法获取证据→与治理层沟通+评价对审计意见影响 |
| 对回函可靠性产生疑虑 | 进一步获取审计证据以消除疑虑 |
| 认为回函不可靠 | 评价对重大错报风险（含舞弊）和其他程序性质/时间/范围的影响 |
| 未回函 | 实施替代程序获取相关、可靠证据 |
| 积极式函证是必要程序且未回函 | 替代程序不能替代→按 CSA 1502 确定对审计工作和审计意见的影响 |
| 存在不符事项 | **调查**不符事项以确定是否表明存在错报 |

⚠️ 函证程序**不适用于**诉讼和索赔事项的询问（那属于 CSA 1311）。

### 回函可靠性判断（函证程序应用指南）

> 类型：判断表
> 来源：函证程序应用指南
> 置信度：准则原文

所有回函都存在被拦截、更改或舞弊的风险（无论纸质/电子）。**显示可靠性存疑的因素：**
- 注册会计师**间接**收到回函（如对方误寄给被审计单位再转交）
- 回函看起来**不是来自预期的被询证者**

**处理规则：**
1. **电子回函**（传真/邮件）发件人身份和授权难确定、更改难发觉→可靠性有风险。采用加密、电子签名、网页认证等安全技术并确信受控，可提高可靠性。
2. 可与被询证者**联系核实**回函来源和内容（如收到邮件回函后电话确认是否本人发送）。
3. 间接收到回函→要求被询证者**直接书面回复**。
4. **口头回复不符合函证要求**（非直接书面回复）→需要求提供书面回复；未回函则实施替代程序。
5. 回函中**对使用作出限制的措辞，不一定使其失去可靠性**。
6. **某些不符事项不表明错报**——如时间安排、计量或书写错误造成的差异；但识别出错报时须评价是否表明舞弊或内控缺陷。

## 供应商风险分级与供应商审计（采购循环补充）

> 采购与付款循环的内控和实质性程序见前文八大循环；本节补充供应链质量管理视角的供应商风险分级方法，用于尽调、内控评价和采购循环风险评估。
> 来源：Lance B. Coleman《供应商审计与组织风险》

### 供应商组合分级矩阵（Kraljic 模型）

> 类型：判断表
> 来源：Kraljic 投资组合细分模型（供应商审计与组织风险）
> 置信度：实务经验

按"利润影响"×"供应风险/关键性"两维度把供应商分到四象限，决定控制和采购策略：

| 象限 | 利润影响 | 供应风险 | 采购策略 |
|------|---------|---------|---------|
| 战略型（Strategic） | 高 | 高 | 长期供应关系、定期风险分析、应急预案、必要时自产，高管层关注 |
| 杠杆型（Leverage） | 高 | 低 | 用足采购议价权、开发替代供应商、大批量下单 |
| 瓶颈型（Bottleneck） | 低 | 高 | 可获得时多备货、寻找控制供应商的办法 |
| 常规型（Routine） | 低 | 低 | 标准化产品、优化订货量和库存、交易型管理 |

⚠️ 价值创造的 50% 以上往往发生在组织围墙之外（供应商处），供应链薄弱环节会引发质量、交付双重风险——重大召回/食安事件多源于此。

### 供应商准入（onboarding）风险问题清单

> 类型：流程
> 来源：供应商审计与组织风险
> 置信度：实务经验

- 是否与该供应商有合作历史？
- 是否需要审计？现场审计、书面审计还是两者都要？审计频率多高？
- 该供应商产品的可接受质量限（AQL）是多少？
- 如何监控与分析供应商绩效？触发纠正行动的阈值是什么？

### 能力 vs 产能 + 绩效监控双维度

> 类型：判断表
> 来源：供应商审计与组织风险
> 置信度：实务经验

| 概念 | 含义 |
|------|------|
| 能力（Capability） | 持续、可预测地产出满足要求产品的能力 |
| 产能（Capacity） | 满足当前需求并能按需扩产的能力 |

供应商绩效监控应同时覆盖：
- **有形指标**：质量、准时交付、足量交付、审计结果、纠正措施请求数
- **无形指标**：协作态度、对问题的响应度、信息的开放透明度（无形因素风险更易被忽略）

## 科目表标准体系

### 资产负债表科目（约 370 个）

按以下层级组织：

```
一级分类（流动资产/非流动资产/流动负债/非流动负债/股东权益）
  └─ 报表项目（货币资金/应收账款/...）
       └─ 明细科目（库存现金/银行存款/其他货币资金）
            └─ 备抵科目（坏账准备/跌价准备）
```

**净额计算规则：**
- 应收账款净额 = 应收账款 - 应收账款坏账准备
- 存货净额 = 存货 - 存货跌价准备
- 固定资产净额 = 固定资产原值 - 累计折旧 - 固定资产减值准备 + 固定资产清理

### 现金流量表科目（约 90 个）

三大类 + 补充资料：
- 经营活动（收入15项 + 支出10项）
- 投资活动（收入5项 + 支出5项）
- 筹资活动（收入5项 + 支出5项）
- 补充资料（间接法调整项 ~40项）
- 现金及现金等价物净增加额

## 古立特输出规范

当用户要求"做报表调整"或"生成审定报表"时，古立特按以下结构输出：

### 输出格式（结构化 JSON 或 Excel 多 Sheet）

```json
{
  "project_info": {
    "entity_name": "XX公司",
    "period": "2025年度",
    "preparer": "...",
    "reviewer": "..."
  },
  "trial_balance": {
    "科目名称": {"未审借方": 0, "未审贷方": 0, "调整借方": 0, "调整贷方": 0, "审定借方": 0, "审定贷方": 0}
  },
  "adjustments": [
    {"编号": 1, "摘要": "...", "借方科目": "...", "贷方科目": "...", "金额": 0}
  ],
  "financial_statements": {
    "balance_sheet": {"资产": {...}, "负债": {...}, "权益": {...}},
    "income_statement": {...},
    "cashflow_statement": {...},
    "equity_changes": {...}
  },
  "analysis": {
    "fluctuation": [...],
    "ratios": {...},
    "non_recurring": {...},
    "eps_roe": {...}
  },
  "validation": {
    "balance_check": true,
    "cross_check": [...]
  }
}
```

### 古立特执行规则

1. **收到数据后先做平衡校验** — 借贷必须平衡，不平衡则报错
2. **调整分录逐笔校验** — 每笔借贷相等，科目名称在标准科目表内
3. **自动计算审定数** — 未审 + 调整 = 审定
4. **自动生成分析** — 波动分析、财务比率、非经常性损益
5. **输出时标注异常** — 波动超阈值的科目、比率异常的指标
6. **勾稽关系自动校验** — 表内平衡 + 跨表一致
