# W17：个股排雷（一句话端到端）

> 定位：用户一句话"帮我排雷 XX"，古立特自动跑完——行情异动 → 拉财务 → Red Flags 扫描 → 出排雷报告
> 触发：排雷、排雷 XX、这股有没有雷、XX 财报有没有问题、帮我看看 XX 靠不靠谱、XX 会不会暴雷
> 适用判断：用户给一个上市公司名/代码 + 想知道"有没有风险/雷"。**不是**要精算估值（那走 investment.md），**不是**要做底稿（那走审计工作流）。
> 依赖工具：market_quote / report_download / stock_financial / stock_history / financial_ratios / analytical_procedures
> 依赖知识：financial_fraud.md（Red Flags 体系 + 23 案例）、statement.md（勾稽 + 行业基准）、equity_research.md（研究框架）
> 性质：实操

---

## 设计理念

排雷 ≠ 估值。排雷回答一个问题：**这家公司的财报数字可信吗，有没有藏雷。**

古立特的排雷不是堆数据，是**用反舞弊视角主动找雷**——对抗者视角（见 SKILL.md 工作行为规则）落地到一条工具链：先看价异动（市场在不在怀疑它），再看财报硬伤（数字自己打不打架），最后对照 Red Flags 库（像不像已暴雷的先例）。

**一句话触发，但每步报告、关键处确认**——不黑箱跑完甩个结论，而是边跑边给业务判断。

---

## 工具未配置时的降级

排雷依赖 market_quote / stock_financial 等 MCP 工具。**MCP 未配置（只装 Skill）时**：古立特退回知识指引——告诉用户"自动排雷需要 gridman-app 工具层；当前未检测到，我可以基于你提供的财报数据用 financial_fraud.md 的 Red Flags 体系人工排查"，然后按知识库的红旗清单逐项问、逐项判。不假装跑了工具。

---

## 五步流程

### Step 1：定位标的 + 看价异动（入口扫描）

调 `market_quote`，输入公司名或代码（支持 A股/港股/美股）。

- 拿到：当前价、涨跌幅、振幅、**异动信号**（接近涨跌停 / 振幅异常）。
- **排雷意义**：市场往往先于财报反应。连续异动、闪崩、长期阴跌——市场可能已在用脚投票。异动信号是排雷的第一道"烟味"。
- 报告用户：行情快照 + 一句判断（如"近期有大幅波动，市场情绪需关注"或"价格平稳，无明显市场预警"）。

> ⚠️ 单次快照只能看当下。要看"连续异动"需配合 Step 2 的历史数据。

### Step 2：拉财务数据 + 历史行情

并行两路：
- `stock_financial`（report_type 分别取 income / balance / cashflow）→ 拿三大报表近 8 期。
- `stock_history`（period=monthly，近 1-3 年）→ 看价格趋势与成交变化，确认 Step 1 的异动是否持续。

报告用户：营收/利润/经营现金流近 3 年趋势一句话概括。

### Step 3：硬伤扫描（数字自己打不打架）—— 核心

这一步是排雷的心脏。调 `financial_ratios` 算比率，再**对照 `financial_fraud.md` 的 Red Flags 体系**逐项过：

| 排雷维度 | 看什么 | 雷信号 | 知识库依据 |
|---|---|---|---|
| 利润含金量 | 净现比（经营现金流/净利润） | 持续 < 1、利润高但现金流差 | financial_fraud.md 反舞弊思路一 |
| 收入真实性 | 收现比、应收增速 vs 收入增速 | 应收暴增超过收入、收现比持续偏低 | financial_fraud.md 高风险科目-应收 |
| 存贷双高 | 货币资金 vs 有息负债 | 大额存款 + 大额借款并存 | financial_fraud.md 案例库共性 |
| 毛利率异常 | 毛利率 vs 同行/历史 | 显著高于同行且无合理解释 | statement.md 行业基准 |
| 商誉/其他应收 | 占净资产比 | 商誉/其他应收占净资产过高 | financial_fraud.md 掏空指标 |
| 利润结构 | 扣非净利 vs 净利 | 靠非经常性损益撑盈利 | financial_fraud.md 财务洗澡 |

用 `analytical_procedures` 跑本期 vs 上期波动，标记异常科目（按 README 的"风险持续性升级"规则判等级）。

报告用户：**亮红的雷点清单**，每条标严重程度 + 依据。

### Step 4：对照先例（像不像已暴雷的）

把 Step 3 的雷点，对照 `financial_fraud.md` 案例库 23 例的"标志手法 + 特有红旗"：

- 净现比长期 < 1 + 应收暴增 → 像不像虚增收入型（紫晶/卓朗）？
- 存贷双高 → 像不像资金占用型（康得新/ST苏吴）？
- 利润虚增占比疑似 > 100% → 算"剔除可疑后真实利润"，常为负。

报告用户：**最像哪类先例**（如"收入-现金流背离模式，与 XX 案例特征相似，建议穿透核查回款真实性"）。不下"它造假"的结论——给"像不像、该查什么"。

### Step 5：出排雷报告（落盘）

汇总成 `排雷报告_{公司}_{日期}.md`，存 `gridman-mind/outputs/`。结构：

```
# {公司} 排雷报告（{日期}）
## 一、市场信号    —— 行情异动 + 趋势
## 二、财务硬伤    —— 雷点清单（按严重程度排序，每条带依据）
## 三、先例比对    —— 最像哪类暴雷模式
## 四、整体结论    —— 红/黄/绿灯 + 一句话 + 建议下一步
## 五、免责声明    —— 基于公开数据的风险提示，不构成投资建议
```

## 红黄绿灯判定（整体结论）

| 灯 | 含义 | 触发 |
|---|---|---|
| 🟢 绿 | 未见明显雷点 | 比率正常、无持续异常、不像任何先例 |
| 🟡 黄 | 有疑点需关注 | 1-2 个单期异常，或单一维度偏离但有合理解释空间 |
| 🔴 红 | 高风险 | 多维度雷点叠加 / 持续 2 期以上异常 / 高度吻合某暴雷先例 |

---

## 铁律（排雷不可破）

1. **不下"造假"定性**——古立特给的是"风险信号 + 像不像先例 + 建议查什么"，不替监管/审计下定论。守 SKILL.md 铁律 5。
2. **不编造数据**——拉不到的数据如实说"未取到"，不脑补。守铁律 1。
3. **对抗者视角，不扮演造假方**——预判"如果造假会从哪下手"去查，不下场编造假账。守对抗者视角规则。
4. **数据时效**——行情是快照、财报是定期报告，结论标注"截至 X 期数据"。

## Few-shot

**示例（典型触发）**：
用户："帮我排雷东方集团"
古立特：
→ 先把"东方集团"转代码 600811（拿不准则反问/查证），再 Step1 market_quote("600811")：价格 + 异动信号
→ Step2 拉三表 + 月线
→ Step3 算比率：净现比 0.3（持续<1）、应收暴增 220% 远超收入 → 红旗
→ Step4 对照：高度吻合"融资性贸易虚增收入"模式（东方集团本身即此类先例）
→ Step5 出报告：🔴 红灯，"收入-现金流严重背离，建议穿透贸易链条真实性"
