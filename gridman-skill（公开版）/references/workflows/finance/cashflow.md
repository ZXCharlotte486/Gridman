# W15: 现金流量表编制

**触发场景：** "帮我做现金流量表"、"间接法倒推"、"直接法现金流"、"现金流编制"、"序时账归类现金流"

**业务背景：**

古立特支持两种现金流编制路径，根据用户数据情况选择：
- **间接法（cashflow_test）**：从净利润出发，通过调整非现金项目倒推经营性现金流，再结合 TB 科目变动估算三大活动现金流，并与报表对比验证。适合有三大报表但无细流水的场景。
- **直接法（cashflow_direct）**：穿透序时账逐笔归类至经营/投资/筹资活动。适合有完整序时账的场景，结果更精确。

**步骤（先判断走哪条路）：**

```
[输入判断]
   ├── 有序时账（凭证流水） → 走直接法（Step A）
   └── 只有三大报表/TB    → 走间接法（Step B）

─── 直接法路径（Step A）───────────────────────────────────

Step A1: data_file_overview（确认序时账列名：日期/凭证号/科目/摘要/借贷金额）
   ↓
Step A2: cashflow_direct
        - 输入：序时账文件 + 会计期间
        - 输出：直接法现金流量表（三大活动分类 + 合计 + 净增减额）
        - 内置 20 条分类规则（银行存款↔应付账款=经营，银行存款↔固定资产=投资，等）
        - 注意：复杂业务（如融资性租赁）需人工复核分类
   ↓
Step A3: 业务结论
        - 核实净增减额与期末期初货币资金差额是否一致
        - 报告三大活动金额和比例结构

─── 间接法路径（Step B）───────────────────────────────────

Step B1: data_file_overview（确认三大报表和TB的数据结构）
   ↓
Step B2: cashflow_test
        - 输入：资产负债表（两期）+ 利润表 + TB
        - 输出：间接法现金流测算表 + 与已编报表的差异分析
        - 自动估算：折旧摊销、非现金项目调整、营运资本变动
        - 含行业假设（VAT 13%等），输出 message 中有说明
   ↓
Step B3: 差异分析
        - 测算值 vs 报表值差异 > 5% → 需查原因（可能是特殊业务或假设偏差）
        - 差异科目列表 + 金额排序
   ↓
Step B4: 业务结论
        - 经营性现金流 vs 净利润比值（理想 > 0.8）
        - 如 < 0.5，说明利润"含金量"低，重点关注应收/存货
        - 投资/筹资活动结构分析
   ↓
[输出]
   ▸ 直接法：现金流量表.xlsx（三大活动明细 + 合计）
   ▸ 间接法：现金流测算底稿.xlsx（测算值 + 报表值 + 差异）
```

**关键判断：**

| 指标 | 判断标准 |
|------|---------|
| 经营性现金流 / 净利润 | > 0.8 → 健康；0.5–0.8 → 关注；< 0.5 → 预警 |
| 投资性现金流持续为负 | 正常扩张信号，但需看资金来源 |
| 筹资性现金流持续为正 | 长期债务融资依赖，关注偿债压力 |
| 直接法归类失败笔数 | > 10% → 建议人工复核分类规则 |

**注意（cashflow_test 局限性）：**
- 间接法工具含行业默认假设，结果是"估算"不是"精确"
- 建议配合 W9 报表勾稽校验，先确认三表数据准确

**与其他工作流的关系：**

- **上游**：W6 月结（Step 7 现金流质量核查调用 cashflow_test）
- **平行**：W7 上市公司财报分析（现金流是盈利质量判断的重要依据）
- **下游**：现金流异常 → W5 总账分析追查来源
