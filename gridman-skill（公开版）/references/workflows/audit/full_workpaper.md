# W13: 底稿全流程（半自动）

**触发场景：** "帮我做一套完整的底稿"、"从计划到报告"、"新项目底稿框架"、"全套审计底稿"

**业务背景：**

底稿编制是审计师的核心交付物。这个工作流不是"全自动出底稿"——古立特没有事务所内部模板，也无法替代审计师做实地盘点、询问管理层、形成审计意见这些工作。

但古立特能做"半自动"——**搭起骨架，让人填**：
- 自动建立标准目录结构（按手册第八部分）
- 生成 12+ 份空白模板（含计划、风险、判断、完成阶段全套）
- 把现有 24 个审计工具串起来按手册顺序跑
- 各程序输出自动归档到对应文件夹
- 关键参数（重要性、阈值）由工具计算好填入

**适用前提：**

- 古立特生成的是"通用骨架"。事务所有内部模板的，自行替换。
- 配色用古立特灰紫，可改。
- 封面、抬头、签字栏用户自行加。

**步骤：**

```
[输入] 客户名称 + 审计年度 + 客户提供的电子资料
   ↓
Step 1: workpaper_init
        - 输入：客户名 + 年度 + 是否集团审计 + 是否做控制测试
        - 输出：完整目录结构 + 空白模板（共 13-14 个文件）
        - 包含：业务承接评价表 / 总体审计策略 / 审计计划 /
                风险评估 / 自动化工具记录 / 职业判断 /
                未更正错报 / 复核记录 / 期后事项 /
                审计总结 / 审计报告草稿 / 索引 / README
   ↓
Step 2: 填写业务承接评价表
        - 客户基本情况 / 管理层诚信 / 独立性 / 资源充足性
        - 集团审计：组织结构 / 组成部分识别
        - ※ 此步骤需人工询问 + 判断，工具不替代
   ↓
Step 3: materiality_calculator
        - 输入：税前利润 / 总资产 / 营业收入
        - 输出：整体重要性 / 实际执行重要性 / 微小错报临界值
                + 职业判断说明（直接复制到底稿）
        - auto 模式：盈利稳定用税前利润，否则用总资产
        - 集团审计：自动应用双限值（组件 ≤ 集团 × 80%）
   ↓
Step 4: 把重要性结果填入【总体审计策略】
   ↓
Step 5: data_file_overview + analytical_procedures
        - 看科目余额表/序时账数据结构
        - 本期 vs 上期波动分析
        - 自动标记异常波动（> 10%）
        - 输出风险评估的输入材料
   ↓
Step 6: 风险评估结果汇总（基于第 5 步输出 + 询问管理层）
        - 识别特别风险（如收入舞弊、管理层凌驾）
        - 设计应对措施
   ↓
Step 7: 各科目实质性程序（按【具体审计计划】顺序）
        ① 货币资金   → bank_reconciliation + cutoff_test
        ② 应收账款   → aging_analysis + audit_sampling + confirmation_letter + cutoff_test
        ③ 存货       → cutoff_test + 实地盘点（人工）
        ④ 固定资产   → depreciation_check + audit_sampling
        ⑤ 收入       → cutoff_test + benford_law_check + 分析性程序
        ⑥ 费用       → cutoff_test + audit_sampling
        ⑦ 应付账款   → aging_analysis + audit_sampling + confirmation_letter
        ⑧ 在建工程   → data_filter（前 10 大）+ 资本化利息测试（人工）
        各程序输出存入对应的 3-实质性程序/[科目]/ 子目录
   ↓
Step 8: 边做边记录【自动化工具使用记录】（每用一次工具就更新）
        每用一次 MCP 工具，登记：工具名 / 版本 / 用途 / 数据来源 /
        验证步骤 / 人工复核逻辑 / 执行人 / 执行日期
   ↓
Step 9: 边做边记录【重大职业判断记录单】
        关键判断必填：重要性确定 / 特别风险识别 / 会计估计合理性 /
        持续经营评估 / 以前年度损益调整等
   ↓
Step 10: 汇总调整分录 + 三栏 TB
         - 收集本次审计发现的所有调整分录
         - audit_adjustment：未审 TB + 调整分录 → 审定 TB
         - 借贷平衡校验
         - 输出存入 6-完成阶段/
   ↓
Step 11: 完成阶段
         - 未更正错报汇总（管理层不调整的错报，评估累计影响）
         - 期后事项复核
         - 复核记录（项目组互审 / 项目经理 / 质量复核）
         - 撰写审计总结
         - 起草审计报告
   ↓
[输出] 完整的审计底稿包（按手册七大文件夹结构归档）
```

**关键判断规则：**

1. **古立特只做能机器化的部分**
   - ✓ AI 能做：数据计算、模板填充、异常标记、报表勾稽校验
   - ✗ AI 不能做：实地盘点、询问管理层、检查实物凭证、形成审计意见

2. **每用一次自动化工具，记录一次**（ISA 500 修订要求）
   - 自动化工具使用记录是 2026 年准则的强制要求
   - 古立特生成的【自动化工具使用记录】模板已预填工具清单，用户填执行人和日期即可

3. **职业判断必须由人完成**
   - 古立特提供"判断框架"和"参考数据"
   - 但所有"判断结论"必须审计师签字确认
   - 重大职业判断记录单是质量控制的核心

4. **配色和模板格式可改**
   - 古立特用统一灰紫配色，事务所有规范的可调
   - 封面/抬头/装订由用户自行处理

**与其他工作流的关系：**

- **包含**：W1（应收）/ W2（银行）/ W3（固资）/ W6（月结）/ W8（函证）/ W11（三栏TB）/ W12（截止测试）都是 W13 的子流程
- **W13 是元工作流**：把所有底稿层面的工作流串成一个整体
- **执行频率**：每个项目跑一次

**估算时间：**

- workpaper_init + materiality + analytical：5-10 分钟（机器执行）
- 各科目实质性程序：1-3 天（含人工部分）
- 完成阶段：1-2 天（撰写、复核）
- 总计：通常 1-2 周（小型项目）至 1 个月（大型项目）

## 验证清单

- [ ] workpaper_init 已生成完整目录结构 + 13-14 个模板
- [ ] 业务承接评价表已填（人工询问+判断，工具不替代）
- [ ] materiality_calculator 已算三档重要性，结果填入总体审计策略
- [ ] 各科目实质性程序按计划顺序执行，输出归档到对应子目录
- [ ] 【自动化工具使用记录】每用一次工具就登记（ISA 500 强制要求）
- [ ] 【重大职业判断记录单】关键判断已记录待审计师签字
- [ ] 调整分录已汇总，audit_adjustment 生成审定 TB 且借贷平衡
- [ ] 完成阶段：未更正错报汇总/期后事项/复核记录/审计总结/报告草稿
- [ ] AI 不能做的部分（盘点/询问管理层/查实物/形成意见）已明确交回人工
