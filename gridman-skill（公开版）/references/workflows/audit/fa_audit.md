# W3: 固定资产审计套件

**触发场景：** "帮我审固定资产"、"折旧算得对不对"、"固定资产盘点"

**可单独执行每一步。**

**步骤：**

```
[输入] 固定资产明细表（含原值、净残值率、使用年限、入账日期、折旧方法、账面累计折旧）
   ↓
Step 1: 数据概览与核对
        工具：data_file_overview
        输入：固定资产明细表
        输出：列名确认、资产总数、数据完整性
        验证：明细表合计 vs 总账余额是否一致
   ↓
Step 2: 折旧重算
        工具：depreciation_check
        输入：固定资产明细表 + 检查日期
        输出：每项资产的重算累计折旧 vs 账面累计折旧 + 差异
        判断：差异方向（账面少计=利润高估，账面多计=利润低估）
   ↓
Step 3: 差异分析
        工具：data_filter（差异 > 1元）
        输入：折旧重算结果
        输出：差异清单（按差异金额排序）
        判断：累计差异占总折旧比例 → 评估是否超过重要性
   ↓
Step 4: 本期增加检查
        工具：data_filter（本期新增资产）
        输入：固定资产明细表（入账日期在本期）
        输出：本期新增资产清单（大额项目需查合同/发票/验收单）
   ↓
Step 5: 本期减少检查
        工具：data_filter（本期处置/报废资产）
        输入：固定资产明细表
        输出：本期减少清单（需查处置审批/收入确认）
   ↓
Step 6: 抽样盘点
        工具：audit_sampling
        输入：固定资产明细表
        输出：盘点抽样清单（从账到实 + 从实到账）
        注意：实地盘点需人工执行
   ↓
Step 7: 在建工程转固检查
        工具：data_filter（在建工程明细中已完工项目）
        输入：在建工程明细
        输出：应转固但未转固的项目清单
        判断：达到预定可使用状态但未转固 → 少计折旧
   ↓
[输出]
   ▸ 折旧重算表.xlsx（含差异标记）
   ▸ 差异清单.xlsx
   ▸ 本期增加清单.xlsx
   ▸ 本期减少清单.xlsx
   ▸ 盘点抽样清单.xlsx
   ▸ 在建工程转固检查.xlsx
```

**关键判断：**

| 项目 | 判断标准 |
|------|---------|
| 折旧差异方向 | 少计=利润高估（更严重），多计=利润低估 |
| 差异重要性 | 累计差异 > 实际执行重要性 → 建议调整 |
| 转固时点 | 实体建造完成+安装调试完毕+试运行合格（不以竣工验收为准） |
| 抽样优先级 | 差异大的 + 原值大的 + 新增的 |

## 验证清单

> 从本工作流关键判断 + 步骤检查点提取，交付前逐条核对。

- [ ] 明细表合计与总账余额已核对一致
- [ ] 折旧重算完成，差异方向已判定（少计=利润高估更严重）
- [ ] 累计差异占比已评估是否超过实际执行重要性
- [ ] 本期增加项已查合同/发票/验收单（大额）
- [ ] 本期减少项已查处置审批/收入确认
- [ ] 抽样盘点清单已生成（实地盘点需人工，已提示）
- [ ] 在建工程已达可使用状态但未转固的项目已识别
- [ ] 转固时点按"建造完成+调试+试运行合格"判断，非竣工验收
- [ ] 输出文件已存入 gridman-mind/outputs/
