# W2: 银行存款审计套件

**触发场景：** "帮我对一下银行账"、"银行存款审计"、"做银行调节表"

**可单独执行每一步，也可以说"帮我跑完整个银行审计"一次性执行全部。**

**步骤：**

```
[输入] 银行对账单 + 银行日记账 + 科目余额表（货币资金）
   ↓
Step 1: 数据概览与核对
        工具：data_file_overview
        输入：银行对账单 + 银行日记账
        输出：列名确认、行数、数据类型、缺失值
        验证：对账单期末余额 vs 日记账期末余额是否一致
   ↓
Step 2: 银行余额调节表
        工具：bank_reconciliation
        输入：银行对账单 + 银行日记账
        输出：调节表（企业未达账项 + 银行未达账项 + 调节后余额）
        判断：调节后余额必须相等，否则存在差错
   ↓
Step 3: 未达账项分析
        工具：data_filter（筛选未达账项中 > 3个月的长期项目）
        输入：调节表中的未达账项清单
        输出：长期未达账项清单 + 风险标记
        判断：长期未达账项可能是已作废支票、错误记账、或舞弊信号
   ↓
Step 4: 大额异常交易筛选
        工具：data_filter（金额 > 重要性水平）+ benford_law_check
        输入：银行日记账
        输出：大额交易清单 + 本福特检验结果
        判断：首位数字偏离 → 可能存在人为编造的交易
   ↓
Step 5: 截止测试
        工具：cutoff_test
        输入：银行日记账 + 截止日
        输出：期末前后 7 天的收付款清单 + 可疑项标记
        判断：期末最后几笔大额收款 → 可能提前确认收入
   ↓
Step 6: 银行函证
        工具：confirmation_letter(type=bank)
        输入：银行账户信息（账号/开户行/余额）
        输出：标准格式银行询证函（14项）+ 函证清单
        注意：按财会〔2022〕39号标准格式生成
   ↓
Step 7: 受限资金识别
        工具：data_filter（筛选定期存款/保证金/冻结）
        输入：银行存款明细
        输出：受限资金清单（需在附注披露）
   ↓
[输出]
   ▸ 银行余额调节表.xlsx
   ▸ 未达账项分析.xlsx（含长期未达标记）
   ▸ 大额异常交易清单.xlsx
   ▸ 本福特检验报告.xlsx
   ▸ 截止测试报告.xlsx
   ▸ 银行询证函.docx（标准14项格式）
   ▸ 函证清单.xlsx
   ▸ 受限资金清单.xlsx
```

**关键判断：**

| 项目 | 判断标准 |
|------|---------|
| 长期未达账项 | > 3 个月未清 → 重点核实 |
| 本福特偏离 | 首位数字 1 过度集中或不足 → 舞弊信号 |
| 整数大额交易 | 恰好 100,000 / 500,000 等 → 重点核查 |
| 截止测试窗口 | 前后 7 天（高风险缩到 3 天） |
| 受限资金 | 定期存款/保证金/冻结 → 附注披露 |

## 验证清单

> 从本工作流关键判断 + 步骤检查点提取，交付前逐条核对。

- [ ] 对账单期末余额 vs 日记账期末余额已核对
- [ ] 调节后余额两边相等（不等则有差错，未平不交付）
- [ ] 长期未达账项（> 3 个月）已标记并说明
- [ ] 本福特检验完成，首位数字异常集中已标注
- [ ] 大额/整数交易（10万/50万等）已重点核查
- [ ] 截止测试窗口已确认（默认 7 天，高风险 3 天）
- [ ] 银行询证函按财会〔2022〕39号 14 项标准格式生成
- [ ] 受限资金（定期/保证金/冻结）已识别，提示附注披露
- [ ] 输出文件已存入 gridman-mind/outputs/
