# W11: 审计调整与底稿三栏 TB

**触发场景：** "帮我做底稿"、"调整分录汇总"、"未审到审定"、"三栏 TB"、"试算平衡表"

**业务背景：**

底稿核心公式：**未审数 + 调整数 = 审定数**。这是审计底稿"试算平衡表"工作底稿（B级，A底稿之一）的核心逻辑。古立特的工具完成"内容生成"，模板套用（各所自己的封面格式）由用户自己处理。

**步骤：**

```
[输入] 未审 TB（科目代码/名称/借贷余额） + 调整分录表（分录号/科目/借贷/摘要）
   ↓
Step 1: data_file_overview（先看 TB 文件，确认列名和数据范围）
   ↓
Step 2: balance_sheet_process（如果是多级科目，先拆到末级；已是末级 TB 跳过）
   ↓
Step 3: 与用户确认调整分录表
        - 分录号字段是否存在（同一笔分录的多行用同一编号关联）
        - 借/贷字段是否分列
        - 是否有摘要说明每笔调整的依据
   ↓
Step 4: audit_adjustment
        - 输入：未审 TB + 调整分录
        - 输出：三栏 TB（未审/调整/审定）+ 调整分录平衡校验 + 总借贷平衡校验
        - 自动校验：
          ✓ 每笔调整分录借贷是否平衡
          ✓ 未审 TB 借贷是否平衡
          ✓ 审定 TB 借贷是否平衡
          ✓ 是否有调整分录引入了 TB 中不存在的新科目
   ↓
Step 5: 业务结论
        - 报告调整分录笔数、影响科目数、调整总额
        - 列出未平衡的调整分录（如有）
        - 列出新增科目（如有）— 这种情况通常说明用户漏录了相关科目
        - 给出审定数对关键指标的影响（如收入/利润变动）
   ↓
[输出] 三栏 TB.xlsx（古立特配色，分三个 Sheet：三栏TB / 调整分录汇总 / 校验报告）
```

**关键判断规则：**

1. **借贷差容差**：1 元以内视为平衡（账面四舍五入）
2. **新增科目**：调整分录涉及但 TB 中没有的科目，要警告用户——通常是用户漏录或调整分录写错了科目
3. **审定净额方向**：未审借 - 未审贷 + 调整借 - 调整贷 = 审定净额，正数落审定借方，负数落审定贷方
4. **调整分录号**：强烈建议有"分录号"列（如"调1/调2"），便于按笔校验借贷平衡

**与其他工作流的关系：**

- **上游**：W2 银行调节表 / W3 固定资产折旧 / W6 月结，这些工作流会产出"调整建议"，可作为本工作流的调整分录输入
- **下游**：审定 TB 是后续生成正式报表（资产负债表/利润表）的基础

## 验证清单

> 从本工作流关键判断 + 步骤检查点提取，交付前逐条核对。

- [ ] 多级科目已拆到末级（已是末级 TB 则跳过）
- [ ] 调整分录表确认：分录号列存在、借贷分列、有摘要依据
- [ ] 每笔调整分录借贷平衡（1 元内容差）
- [ ] 未审 TB 借贷平衡、审定 TB 借贷平衡
- [ ] 调整分录引入的新科目（TB 中不存在）已警告用户
- [ ] 审定净额方向正确（净额为正落借方，为负落贷方）
- [ ] 已报告调整笔数、影响科目数、调整总额、对关键指标影响
- [ ] 输出三 Sheet（三栏TB/调整分录汇总/校验报告），存入 outputs/
