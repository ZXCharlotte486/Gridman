# W1: 应收账款审计套件

**触发场景：** "帮我做应收账款的审计"、"我有应收明细，帮我跑完账龄+函证"

**业务背景（参考天职国际指引 2008 版的 9 步法，结合 2026 准则）：**

应收账款审计是审计中最重要的循环之一。本工作流覆盖天职国际指引第 4 章应收账款的核心步骤，并结合古立特工具自动化。

**步骤（9 步法）：**

```
[输入] 应收账款明细表（含客户名称、欠款金额、欠款日期）
       + 总账/科目余额表
       + 销售合同/账龄分析表（如有）
       + 上年审计调整数（连续审计）
       + 关联方名录
   ↓
Step 1: 获取并复核明细表（古立特：data_file_overview）
        - 确认列名 + 数据完整性
        - 标记审计标识：S（与明细账核对）+ G（与总账核对）+ B（与上年审定数核对）
        - 检查贷方余额项目（可能需重分类至预收账款）
   ↓
Step 2: 财务指标分析（古立特：analytical_procedures）
        - 应收账款周转率、周转天数
        - 应收/营业收入比率
        - 与上期 + 行业对比，标记异常波动
        - 重大异常需说明原因
   ↓
Step 3: 账龄分析（古立特：aging_analysis）
        - 先进先出法生成账龄分布
        - 重点关注 ≥ 1 年的长账龄项目
        - 与上年账龄分析表核对（连续审计）
   ↓
Step 4: 函证样本选取（古立特：audit_sampling method=mus）
        - 选取所有超过实际执行重要性 50% 的明细
        - MUS 抽样保证金额覆盖率 ≥ 50%
        - 关联方/异常款项必抽
   ↓
Step 5: 函证发出（古立特：address_split + confirmation_letter）
        - 拆分地址为省市区（顺丰发函需要）
        - 生成 Word 询证函（receivable 类型）
        - 生成 Excel 函证清单（含编号/对方/金额/状态）
        - 函证由审计人员亲自收发
        - 标记审计标识：C（已发函）→ ?（已回函）
   ↓
Step 6: 替代测试（手工，针对未回函项目）
        - 检查期后收款的银行流水
        - 检查销售合同、发运单、客户签收单
        - 检查发票
        - 不构成回函证据，但可降低风险
   ↓
Step 7: 坏账准备复核（手工 + 古立特工具辅助）
        - 复核坏账准备计提政策（与上年是否一致）
        - 单项金额重大、组合风险较大的客户单独评估
        - 新准则下用 ECL（预期信用损失）模型
        - 历史损失率、前瞻性调整、客户违约概率
   ↓
Step 8: 关联方往来检查（古立特：audit.md 12 维度方法论）
        - 股权穿透 / 共同股东 / 实控人
        - 法人交叉 / 联系方式共用
        - 工商信息异常
        - 见 audit.md "关联方识别 12 维度"
   ↓
Step 9: 截止测试 + 列报检查（古立特：cutoff_test）
        - cutoff_test test_type=receivable
        - 检查 12-31 前后 7 天的应收交易
        - 标记可疑项（临近截止日 / 大额 / 整数 / 月末月初）
        - 检查附注披露是否完整（账龄、坏账准备、前五大客户）
   ↓
[输出]
   ▸ 应收账款明细表（标记 S/G/B 审计标识）
   ▸ 账龄分析表
   ▸ 财务指标分析报告
   ▸ 函证样本清单 + 批量函证 Word
   ▸ 替代测试记录（手工补充）
   ▸ 坏账准备复核底稿（手工补充）
   ▸ 关联方排查报告
   ▸ 截止测试报告
```

**关键判断（综合天职国际指引 + CSA 准则）：**

| 项目 | 判断标准 |
|------|---------|
| 函证选取标准 | 超过实际执行重要性 50% 的明细 + 大额异常 + 关联方 |
| 函证覆盖率 | ≥ 50% 金额覆盖率 |
| 长账龄阈值 | ≥ 1 年 |
| 截止测试窗口 | 前后 7 天（高风险时缩到 3 天） |
| 替代测试触发 | 资产方余额 > 报表层重要性 + 未回函 |
| 坏账模型 | 新准则用 ECL（预期信用损失），不再用已发生损失 |

**审计标识参考（标到底稿明细表）：**

```
B   期初余额与上年审定数核对   →   明细表期初余额行
G   与总账核对相符              →   明细表合计行
S   与明细分类账核对相符        →   明细表数据行
T/B 与试算平衡表核对相符        →   合计行
C   已发询证函                   →   函证清单
?   已收回询证函                 →   函证清单
AJE 调整分录                     →   重分类调整行
```

**输出文件命名建议（一级索引 ZD）：**
- `ZD0 应收账款审定表.xlsx`
- `ZD1 应收账款明细表.xlsx`
- `ZD2 函证控制表.xlsx`
- `ZD2-3 函证结果调节表.xlsx`
- `ZD3 应收账款账龄分析.xlsx`
- `ZD4 坏账准备计提政策查验表.xlsx`
- `ZD5 截止测试.xlsx`

**与其他工作流的关系：**

- **上游**：W13 底稿全流程 Step 7（应收账款实质性程序）
- **平行**：W8 函证全流程（本工作流的 Step 5 是 W8 的简化版）
- **下游**：调整分录汇总到 W11（三栏 TB）

## 验证清单

> 从本工作流关键判断 + 步骤检查点提取，交付前逐条核对。

- [ ] 明细表合计与总账核对相符（已标 G），期初与上年审定数核对（已标 B）
- [ ] 贷方余额项目已识别，需重分类至预收账款的已处理
- [ ] 账龄分析覆盖全部明细，长账龄（≥1 年）已单独标记
- [ ] 函证抽样金额覆盖率 ≥ 50%，关联方/大额异常款项已纳入必抽
- [ ] 地址已拆分省市区，Word 询证函 + Excel 清单已生成
- [ ] 未回函项目已安排替代测试（期后收款/合同/签收单）
- [ ] 坏账准备按 ECL 模型复核，计提政策与上年一致性已确认
- [ ] 截止测试窗口已与用户确认（默认 7 天，高风险 3 天）
- [ ] 输出文件命名符合 ZD 一级索引规范
- [ ] 关键参数（抽样比例、截止窗口）执行前已向用户确认
