# W12: 截止测试

**触发场景：** "做截止测试"、"看跨期"、"年末有没有提前/延后确认"

**业务背景：**

截止测试（Cutoff Test）是审计中最常用的程序之一，目标是验证交易是否记录在正确的会计期间。重点查两类风险：
- 截止日前的交易：是否有"提前确认收入/费用"操纵利润
- 截止日后的交易：是否有"应当属于本期但延后入账"

**步骤：**

```
[输入] 序时账（凭证流水，含日期/凭证号/科目/摘要/金额）+ 截止日 + 测试类型
   ↓
Step 1: data_file_overview（先看数据范围和列名是否匹配）
   ↓
Step 2: 与用户确认测试参数
        - 截止日（YYYY-MM-DD，通常是 12-31）
        - 窗口期（默认前后各 7 天，重点关注期可缩到 3 天）
        - 测试类型（revenue / purchase / expense / receivable / payable / general）
        - 金额阈值（可选，过滤小额交易，聚焦重要项）
   ↓
Step 3: cutoff_test
        - 自动按 test_type 筛选相关科目
        - 自动标记可疑项：
          ✓ 临近截止日（≤3 天）
          ✓ 大额（前 5%）
          ✓ 整数（金额尾数 4 个 0）
          ✓ 月末/月初
        - 输出三个 Sheet：可疑项 / 截止日前 / 截止日后
   ↓
Step 4: 业务结论
        - 报告窗口期总笔数、可疑项笔数
        - 重点标注金额最大的 3-5 笔可疑项
        - 提示用户：可疑项需要核实凭证原件、出库单、合同等支持文件
        - 建议下一步：对可疑项做凭证抽查 / 找业务部门询问
   ↓
[输出] 截止测试报告.xlsx（古立特配色，4 个 Sheet：摘要 / 可疑项 / 截止日前 / 截止日后）
```

**测试类型选择指南：**

| 关注风险 | 推荐 test_type | 重点查 |
|---------|---------------|--------|
| 收入虚增 | revenue | 12-31 前的大额收入凭证（提前确认） |
| 收入低估 | revenue | 1 月初的收入凭证（应属上年的延后确认） |
| 成本不实 | purchase | 截止日前后的采购入库凭证 |
| 期间费用截止 | expense | 截止日后才入账的费用（应属本期） |
| 应收/应付截止 | receivable / payable | 跨期的往来余额变动 |

**关键判断规则：**

1. **窗口期默认 7 天，重要性高时缩到 3 天**：重点关注最临近截止日的交易
2. **可疑项不等于错误项**：标记的是"需要进一步检查"，不是"一定有问题"
3. **大额阈值用 95 分位数**：自动适应不同业务规模，不需要手动设定
4. **整数检测**：审计中的经验法则——很多虚假交易喜欢凑整数（10 万、100 万）

**与其他工作流的关系：**

- **上游**：W11 调整分录可能由本工作流的发现触发（如发现 12-31 的收入应冲销）
- **平行**：与 W5 总账分析互补（W5 看全年异常，W12 聚焦截止日窗口）
