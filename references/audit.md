# 第一编 审计基本原理与职业道德

## 第1章 审计概述

### 核心概念

| 概念 | 定义 |
|------|------|
| 财务报表审计 | 注册会计师对财务报表是否不存在重大错报提供**合理保证**，以积极方式提出意见 |
| 合理保证 | 高水平保证，非绝对保证（审计存在固有限制） |
| 有限保证 | 审阅业务，以消极方式提出结论，保证水平低于审计 |

### 审计要素（五要素）

1. **三方关系人**：注册会计师、管理层（责任方）、预期使用者
2. **财务报表**：审计对象信息的载体
3. **财务报告编制基础**：通用目的（企业会计准则）/ 特殊目的（计税基础等）
4. **审计证据**：充分性（数量）+ 适当性（质量 = 相关性 + 可靠性）
5. **审计报告**：最终产品

### 执行审计工作的前提（管理层责任）

管理层认可并理解其承担：
1. 按适用的财务报告编制基础编制财务报表
2. 设计、执行和维护必要的内部控制
3. 向注册会计师提供必要的工作条件（不受限制地接触信息和人员）

### 认定体系

**交易和事项认定**：发生、完整性、准确性、截止、分类、列报

**期末余额认定**：存在、权利和义务、完整性、准确性/计价和分摊、分类、列报

| 认定 | 关注方向 | 典型审计程序 |
|------|----------|-------------|
| 发生/存在 | 高估（多记/虚构） | 从账到证（检查凭证支持） |
| 完整性 | 低估（漏记） | 从证到账（检查原始单据是否入账） |
| 准确性 | 金额错误 | 重新计算、核对 |
| 截止 | 跨期 | 检查资产负债表日前后交易 |
| 权利和义务 | 归属 | 查阅产权证书、合同 |
| 计价和分摊 | 金额合理性 | 评估减值/跌价准备 |

### 审计风险模型

```
审计风险 = 重大错报风险 × 检查风险
重大错报风险 = 固有风险 × 控制风险
```

| 风险类型 | 含义 | 可否控制 |
|----------|------|----------|
| 固有风险 | 不考虑控制时，认定易于发生错报的可能性 | 不可控（客观存在） |
| 控制风险 | 错报未被内控及时防止或发现纠正的可能性 | 不可控（被审计单位决定） |
| 检查风险 | 注册会计师实施程序后未发现错报的风险 | **可控**（通过审计程序应对） |

**反向关系**：评估的重大错报风险越高 → 可接受的检查风险越低 → 需要更多/更有效的审计程序

### 审计的固有限制

1. **财务报告的性质**：会计估计涉及主观判断，存在固有变动幅度
2. **审计程序的性质**：管理层可能不提供完整信息；舞弊涉及串通伪造
3. **成本效益权衡**：需在合理时间内以合理成本完成

### 审计基本要求（五项）

1. 维护公共利益
2. 遵守审计准则
3. 遵守职业道德守则
4. **保持职业怀疑**：质疑的思维方式 + 对异常保持警觉 + 审慎评价证据
5. **运用职业判断**：在准则框架下综合运用知识、技能和经验作出决策

### 审计过程五阶段

1. 接受业务委托
2. 计划审计工作
3. 识别和评估重大错报风险
4. 应对重大错报风险（控制测试 + 实质性程序）
5. 编制审计报告

### 注册会计师执业责任

| 责任类型 | 法律依据 | 后果 |
|----------|----------|------|
| 行政责任 | 注册会计师法、证券法 | 警告、没收违法所得、暂停/撤销执业 |
| 民事责任 | 注册会计师法第42条、证券法第163条 | 连带赔偿（能证明无过错除外） |
| 刑事责任 | 刑法第229条 | 故意提供虚假证明：5年以下/5-10年；重大失实：3年以下 |


## 第2章 职业道德基本原则和概念框架

### 六项基本原则

| 原则 | 核心要求 |
|------|----------|
| 诚信 | 正直、诚实可信，不与有问题的信息发生关联 |
| 客观公正 | 不受偏见、利益冲突或不当影响损害职业判断 |
| 独立性 | 实质独立 + 形式独立（鉴证业务必须） |
| 专业胜任能力和勤勉尽责 | 获取并保持专业能力，认真全面及时完成工作 |
| 保密 | 对职业活动中获知的涉密信息保密 |
| 良好职业行为 | 遵守法律法规，避免损害职业声誉 |

### 职业道德概念框架（三步法）

1. **识别**不利影响
2. **评价**不利影响的严重程度（理性且掌握充分信息的第三方测试）
3. **应对**：消除 / 防范措施降低至可接受水平 / 拒绝或终止

### 五类不利影响

| 类型 | 含义 | 典型情形 |
|------|------|----------|
| 自身利益 | 经济利益或其他利益不当影响判断 | 在客户中拥有直接经济利益；收入过分依赖某一客户 |
| 自我评价 | 需依赖自己以往的判断或结论 | 设计系统后又对其出具鉴证报告 |
| 推介或代理 | 倾向客户立场 | 推介客户产品；为客户辩护 |
| 密切关系 | 长期或密切关系导致偏向 | 近亲属担任客户高管；长期业务关系 |
| 外在压力 | 迫于压力无法客观行事 | 被威胁解除业务关系；被施压同意不恰当会计处理 |

### 利益冲突处理

- 识别各方利益和关系的性质
- 评价不利影响严重程度
- 防范措施：不同项目组分别服务 + 保密隔离 + 适当复核
- 必要时向客户披露并获取明确同意

### 收费规则

- 报价过低 → 自身利益对专业胜任能力产生不利影响
- **鉴证业务禁止或有收费**（收费不得以鉴证结果为条件）
- **禁止收取/支付介绍费或佣金**

### 保密原则的例外（可以披露）

1. 法律法规允许 + 客户授权
2. 法律法规要求（诉讼、向监管机构报告违法行为）
3. 法律诉讼中维护自身合法权益
4. 接受协会/监管机构质量检查

## 第3章 审计业务的独立性要求

### 独立性的两个层面

| 层面 | 含义 |
|------|------|
| 实质上的独立 | 在发表意见时保持客观公正的精神状态 |
| 形式上的独立 | 避免出现使理性第三方认为独立性受损的情形 |

### 独立性适用范围

- **审计业务期间**：从项目组开始执行审计业务时起，至出具审计报告时止
- **财务报表涵盖期间**：如果客户是新客户，独立性要求从项目组开始执行审计业务时起适用

### 经济利益

| 类型 | 规则 |
|------|------|
| 直接经济利益 | 审计项目团队成员及其主要近亲属 → **禁止** |
| 间接经济利益（重大） | 审计项目团队成员及其主要近亲属 → **禁止** |
| 间接经济利益（非重大） | 通常可接受 |

**直接经济利益**：个人直接拥有的股票、债券等
**间接经济利益**：通过基金、信托等中介持有

### 贷款和担保

- 审计项目团队成员向审计客户（银行/类似金融机构）贷款 → 按正常商业条款可接受
- 审计项目团队成员向非金融机构审计客户贷款/担保 → **禁止**

### 商业关系

- 审计项目团队成员与审计客户存在密切商业关系 → 产生自身利益或外在压力不利影响
- 防范措施：终止商业关系 / 将该成员调离项目组

### 家庭和私人关系

| 关系人 | 在客户担任职务 | 影响 |
|--------|---------------|------|
| 主要近亲属 | 董事/高管/能对财务报表施加重大影响 | 严重不利影响，通常需调离 |
| 其他近亲属 | 能对财务报表施加重大影响 | 评价严重程度，必要时调离 |

### 雇佣关系

**前任合伙人/员工加入客户**：
- 如果前任合伙人加入审计客户担任董事/高管 → 评价不利影响
- 关键：是否仍与事务所保持联系、是否参与审计业务

**客户人员加入事务所**：
- 如果客户前任董事/高管加入事务所 → 不得担任该客户审计项目团队成员（冷却期）

### 长期关联（关键审计合伙人轮换）

| 客户类型 | 轮换要求 |
|----------|----------|
| 上市公司/公众利益实体 | 关键审计合伙人连续服务不超过**5年**，冷却期**2年** |
| 非上市公司 | 评价长期关联产生的密切关系不利影响 |

**关键审计合伙人**：项目合伙人、项目质量复核合伙人、其他在审计中作出重大判断或决策的合伙人

### 非鉴证服务对独立性的影响

**总原则**：向审计客户提供非鉴证服务时，不得承担管理层责任

| 服务类型 | 对上市公司审计客户 | 对非上市公司审计客户 |
|----------|-------------------|---------------------|
| 编制财务报表 | 禁止（除非仅为机械性工作） | 可接受（需防范措施） |
| 评估服务 | 禁止（如果对财务报表有重大影响） | 评价不利影响 |
| 税务服务 | 通常可接受（纳税申报、税务筹划） | 可接受 |
| IT系统服务 | 禁止（如果涉及财务报告系统设计） | 评价不利影响 |
| 内部审计服务 | 禁止（如果涉及内控有效性评价） | 评价不利影响 |
| 法律服务 | 禁止（如果涉及诉讼代理） | 评价不利影响 |

### 违反独立性的处理

1. 发现违反 → 立即评估影响
2. 确定是否需要终止业务或采取补救措施
3. 与治理层沟通
4. 记录违反事项、评价结果和采取的措施


## 第4章 审计证据

### 审计证据的构成

1. **会计记录中的信息**：原始凭证、记账凭证、总分类账/明细账、调整分录、计算表
2. **其他信息**：会议记录、内部控制手册、询证函回函、分析师报告、注册会计师编制的分析表等

### 充分性与适当性

| 维度 | 含义 | 影响因素 |
|------|------|----------|
| 充分性 | 数量衡量（样本量） | 重大错报风险越高→需要越多；证据质量越高→需要越少 |
| 适当性 | 质量衡量 = 相关性 + 可靠性 | 来源、性质、获取环境 |

**关键原则**：更多数量无法弥补质量缺陷

### 审计证据可靠性层级（从高到低）

1. 外部独立来源 > 内部来源
2. 内控有效时内部生成 > 内控薄弱时内部生成
3. 直接获取 > 间接获取或推论
4. 文件记录形式 > 口头形式
5. 原件 > 传真件/复印件

### 七种审计程序

| 程序 | 含义 | 典型用途 |
|------|------|----------|
| 检查 | 审查记录/文件或实物 | 检查合同→收入确认；检查有形资产→存在 |
| 观察 | 查看他人正在执行的活动 | 观察盘点、观察控制执行 |
| 询问 | 向知情人获取信息 | 贯穿全过程，需佐证 |
| 函证 | 直接从第三方获取书面答复 | 银行存款、应收账款、借款 |
| 重新计算 | 核对数据计算准确性 | 折旧重算、利息重算 |
| 重新执行 | 独立执行原属内控的程序 | 重新执行银行调节 |
| 分析程序 | 分析数据间内在关系 | 比率分析、趋势分析 |

### 函证规则

**必须函证**：银行存款（含零余额和已注销账户）、借款、应收账款（除非不重要或函证很可能无效）

**函证方式**：

| 方式 | 特点 | 适用场景 |
|------|------|----------|
| 积极式（列明余额） | 要求必须回函确认 | 大额、高风险项目 |
| 积极式（不列余额） | 要求填写信息 | 测试完整性 |
| 消极式 | 仅不同意时回函 | 低风险 + 大量小额 + 预期错误少 |

**函证控制要求**：
- 注册会计师全程控制（发出、收回）
- 询证函经被审计单位盖章后由注册会计师直接发出
- 回函直接寄回事务所
- 核实被询证者地址的真实性

**未收到回函处理**：
1. 再次寄发
2. 实施替代程序（检查期后回款、核对发货单据等）
3. 如果积极式函证是必要程序且无法获取回函→考虑对审计意见的影响

### 分析程序

**三种用途**：
1. 风险评估阶段：了解被审计单位（必须使用）
2. 实质性程序：作为实质性分析程序获取证据
3. 总体复核阶段：评价财务报表整体合理性（必须使用）

**实质性分析程序要素**：
- 建立预期值（使用可靠数据）
- 确定可接受差异额
- 将实际值与预期值比较
- 对超出可接受差异的波动进行调查

## 第5章 审计抽样方法

### 基本概念

| 概念 | 定义 |
|------|------|
| 审计抽样 | 对总体中低于100%的项目实施审计程序，使所有抽样单元都有被选取的机会 |
| 抽样风险 | 基于样本得出的结论与对总体100%检查得出的结论不同的可能性 |
| 非抽样风险 | 与抽样无关的风险（如选择不恰当的程序、误解证据） |

### 抽样风险的两种类型

**控制测试中**：
- 信赖过度风险（效果）：实际偏差率高于样本偏差率 → 可能导致不恰当的审计意见
- 信赖不足风险（效率）：实际偏差率低于样本偏差率 → 导致额外工作

**细节测试中**：
- 误受风险（效果）：实际存在重大错报但样本未发现 → 可能导致不恰当的审计意见
- 误拒风险（效率）：实际不存在重大错报但样本显示存在 → 导致额外工作

### 三种样本选取方法

| 方法 | 特点 | 适用 |
|------|------|------|
| 随机选样 | 每个单元被选中概率相等 | 统计抽样必须使用 |
| 系统选样 | 等距选取（间隔=总体/样本量） | 总体无周期性规律 |
| 随意选样 | 不使用结构化方法 | 非统计抽样 |

### 统计抽样 vs 非统计抽样

| 维度 | 统计抽样 | 非统计抽样 |
|------|----------|------------|
| 样本选取 | 随机选取 | 可随意选取 |
| 结果评价 | 运用概率论量化抽样风险 | 运用职业判断 |
| 适用 | 需要量化风险时 | 其他情况 |

### 控制测试中的抽样

**样本量影响因素**：

| 因素 | 与样本量关系 |
|------|-------------|
| 可接受的信赖过度风险 | 反向（风险越低→样本越大） |
| 可容忍偏差率 | 反向（容忍度越低→样本越大） |
| 预计总体偏差率 | 正向（预计越高→样本越大） |
| 总体规模 | 正向（但影响较小） |

### 细节测试中的抽样

**样本量影响因素**：

| 因素 | 与样本量关系 |
|------|-------------|
| 可接受的误受风险 | 反向 |
| 可容忍错报 | 反向 |
| 预计总体错报 | 正向 |
| 总体规模 | 正向 |
| 分层 | 适当分层可减少样本量 |

### 金额单位抽样（MUS/PPS）

- 以货币单位为抽样单元
- 大金额项目被选中概率更高
- 特别适合测试高估错报（存在/发生认定）
- 对低估错报检测能力较弱

### 评价样本结果

1. 分析样本中发现的偏差/错报的性质和原因
2. 将样本结果推断至总体
3. 评价抽样风险是否可接受
4. 如果推断的错报接近或超过可容忍错报 → 扩大样本或实施替代程序

# 第二编 风险导向审计流程

## 第六章 审计计划

### 初步业务活动

**目的（三项）：**
1. 具备执行业务所需的独立性和能力
2. 不存在因管理层诚信问题影响保持业务意愿的事项
3. 与被审计单位之间不存在对业务约定条款的误解

**内容（三项）：**
1. 针对保持客户关系和具体审计业务实施质量管理程序
2. 评价遵守相关职业道德要求（含独立性）
3. 就审计业务约定条款达成一致意见

**审计的前提条件：**

| 要素 | 内容 |
|------|------|
| 财务报告编制基础 | 管理层采用可接受的编制基础 |
| 管理层责任 | ①按编制基础编制报表并公允反映；②设计执行维护必要内控；③提供必要工作条件和信息 |
| 确认形式 | 书面声明；管理层不认可→不能获取充分适当证据→不应承接（法律另有规定除外） |

**审计业务约定书基本内容：**
1. 审计目标与范围
2. 注册会计师的责任
3. 管理层的责任
4. 适用的财务报告编制基础
5. 拟出具审计报告的预期形式和内容

**连续审计中需修改约定条款的情形：**
- 被审计单位误解审计目标和范围
- 高级管理人员/所有权发生重大变动
- 业务性质或规模发生重大变化
- 法律法规/编制基础/报告要求发生变化

**变更业务约定条款的判断规则：**
- 合理理由：环境变化影响需求、对原业务性质存在误解
- 不合理：与错误/不完整/不令人满意的信息有关
- 不同意变更且管理层不允许继续→解除约定/报告该事项

### 总体审计策略与具体审计计划

**总体审计策略四大事项：**

| 事项 | 核心内容 |
|------|----------|
| 审计范围 | 编制基础、行业报告要求、组成部分数量及地点、内审可利用性、服务机构使用 |
| 报告目标与时间安排 | 对外报告时间表、与管理层/治理层沟通安排 |
| 审计方向 | 重要性水平、高风险领域、重要组成部分、内控有效性证据需求 |
| 审计资源 | 人员分配（经验匹配风险）、时间预算、指导监督复核安排 |

**具体审计计划三部分：**
1. **风险评估程序**——计划实施的性质、时间安排和范围
2. **进一步审计程序**——针对认定层次重大错报风险的控制测试和实质性程序
3. **其他审计程序**——舞弊、持续经营、法律法规、关联方等特定项目

**计划更改规则：** 审计过程中因未预期事项/条件变化/新证据，应在必要时更新修改总体策略和具体计划，重大修改应记录理由。

### 重要性

**重要性概念三要点：**
1. 错报单独或汇总可能影响使用者经济决策→重大
2. 根据具体环境判断，受金额或性质（或两者共同）影响
3. 考虑使用者整体共同需求，不考虑个别使用者

**重要性层次体系：**

| 层次 | 定义 | 用途 |
|------|------|------|
| 财务报表整体重要性 | 基准 × 百分比 | 评价错报对报表和审计意见的影响 |
| 特定交易类别/账户余额/披露的重要性 | 低于整体重要性 | 特定项目错报虽低于整体重要性但影响使用者决策 |
| 实际执行的重要性 | 低于整体重要性（通常50%~75%） | 评估风险、确定进一步审计程序的性质/时间/范围 |
| 明显微小错报临界值 | 通常为整体重要性的3%~5%（不超过10%） | 低于此值的错报不需累积 |

**基准选择规则：**

| 被审计单位情况 | 可能选择的基准 |
|----------------|----------------|
| 盈利水平稳定 | 经常性业务税前利润 |
| 盈亏大幅波动/微利微亏 | 过去3~5年平均税前利润或营业收入 |
| 新设企业/开办期 | 总资产 |
| 新兴行业/侧重市场份额 | 营业收入 |
| 开放式基金 | 净资产 |
| 研发中心（成本加成） | 成本与营业费用总额 |

**实际执行的重要性确定规则：**
- 选择较低百分比（接近50%）：首次接受委托、以前年度调整较多、总体风险较高、存在内控缺陷
- 选择较高百分比（接近75%）：连续审计调整较少、总体风险低到中等、内控运行有效

**错报分类：**
- **事实错报**：毋庸置疑的错报（数据错误、事实忽略、舞弊）
- **判断错报**：管理层与CPA对会计估计值或会计政策选用的判断差异
- **推断错报**：根据样本推断总体的错报（总体推断错报 - 已识别具体错报）

## 第七章 风险评估

### 风险评估程序与信息来源

**风险评估程序（三类）：**
1. **询问**管理层和被审计单位内部其他合适人员（含内审人员）
2. **分析程序**（研究财务/非财务数据间关系）
3. **观察和检查**（经营活动、文件记录、管理层报告、实地察看、穿行测试）

**询问对象与信息价值：**

| 询问对象 | 可获取的信息 |
|----------|-------------|
| 治理层 | 对管理层编制报表的监督程度 |
| 复杂交易处理员工 | 会计政策选用恰当性 |
| 内部法律顾问 | 诉讼、合规、舞弊嫌疑、合同条款 |
| 营销人员 | 营销策略变化、销售趋势、客户合同 |
| 风险管理人员 | 经营和监管风险 |
| 信息技术人员 | 系统变更、控制失效、IT相关风险 |
| 内部审计人员 | 被审计单位环境及内控 |

**项目组内部讨论要点：**
- 参与人员：关键成员必须参与，IT专家等可按需参与
- 讨论内容：①分享了解的信息（性质、舞弊因素）；②分享审计思路（舞弊场景设想）；③指明审计方向（职业怀疑、不可预见性）
- 时间：整个审计过程中持续进行

### 了解被审计单位及其环境

**了解的三大方面：**
1. 被审计单位及其环境（组织结构/所有权/治理/业务模式、行业/法律/监管/外部因素、财务业绩衡量标准）
2. 适用的财务报告编制基础、会计政策及变更原因
3. 内部控制体系各要素

**固有风险因素（五类）：**

| 因素 | 含义 | 示例 |
|------|------|------|
| 复杂性 | 信息性质或编制方式导致 | 复杂监管环境、表外融资、复杂会计计量 |
| 主观性 | 知识/信息可获得性受限，需主观判断 | 多种估值方法可选、折旧方法选择 |
| 变化 | 经营/经济/会计/监管环境变化 | 新系统实施、新准则采用、重大收购重组 |
| 不确定性 | 无法仅通过直接观察编制信息 | 重大计量不确定性、未决诉讼、或有负债 |
| 管理层偏向和其他舞弊风险因素 | 管理层未保持中立 | 关联方交易、期末大量收入、按特定意图记录交易 |

### 了解内部控制

**内部控制五要素：**
1. 内部环境（控制环境）
2. 风险评估
3. 信息与沟通（信息系统与沟通）
4. 控制活动
5. 内部监督

**直接控制 vs 间接控制：**

| 类型 | 定义 | 主要来源要素 | 影响层次 |
|------|------|-------------|----------|
| 直接控制 | 足以精准防止/发现/纠正认定层次错报 | 信息系统与沟通、控制活动 | 认定层次 |
| 间接控制 | 不足以精准防止/发现/纠正认定层次错报，但支持其他控制 | 内部环境、风险评估、内部监督 | 财务报表层次（也可能影响认定层次） |

**了解内部控制的程度：**
- 评价控制设计的有效性 + 确定控制是否得到执行
- **不包括**对控制是否得到一贯执行的测试（那是控制测试的目的）
- 程序：询问 + 观察 + 检查 + 穿行测试（询问本身不充分）

**IT对内部控制的影响：**
- 优势：一贯运用业务规则、提高信息及时性/准确性、降低控制被规避风险、提高职责分离有效性
- 风险：系统/程序错误处理数据、未经授权访问、IT人员超权限、未授权修改主文档/程序、数据丢失

### 识别和评估重大错报风险

**两个层次：**
1. **财务报表层次**——与报表整体广泛相关，影响多项认定（通常源于薄弱控制环境）
2. **认定层次**——与特定交易类别/账户余额/披露的认定相关

**特别风险的识别条件：**
- 需要特别考虑的重大错报风险
- 识别时不考虑相关控制的影响
- 考虑因素：是否为舞弊风险、是否与重大异常交易相关、涉及重大判断的主观性程度、涉及重大关联方交易

**特别风险的应对要求：**
- 了解与特别风险相关的控制（含设计和执行）
- 如拟信赖相关控制→必须在本期测试运行有效性（不得依赖以前审计证据）
- 实质性程序应包括细节测试（或细节测试+实质性分析程序），不能仅用实质性分析程序

## 第八章 风险应对

### 总体应对措施（针对财务报表层次）

**五项总体应对措施：**
1. 向项目组强调保持职业怀疑的必要性
2. 指派更有经验或具有特殊技能的人员/利用专家工作
3. 对指导、监督和复核的性质/时间/范围作出调整
4. 在选择进一步审计程序时融入更多不可预见因素
5. 对总体审计策略或拟实施的审计程序作出调整

**增加不可预见性的方法：**
- 对未测试过的低风险账户/认定实施实质性程序
- 调整实施审计程序的时间（使被审计单位不可预期）
- 采取不同审计抽样方法
- 选取不同地点/预先不告知测试地点

**控制环境缺陷时的总体修改：**
- 在期末而非期中实施更多审计程序
- 通过实质性程序获取更广泛的审计证据
- 增加拟纳入审计范围的经营地点数量

**总体审计方案：**
- **实质性方案**：以实质性程序为主（财务报表层次风险高时倾向采用）
- **综合性方案**：控制测试 + 实质性程序结合使用

### 进一步审计程序（针对认定层次）

**性质、时间、范围三要素：**

| 要素 | 含义 | 核心考虑 |
|------|------|----------|
| 性质 | 目的（控制测试/实质性程序）和类型（检查/观察/询问/函证/重新计算/重新执行/分析程序） | 最重要；与特定风险的相关性 |
| 时间 | 何时实施/审计证据适用的期间或时点 | 风险越高→越应在期末或接近期末实施 |
| 范围 | 数量（样本量、观察次数等） | 重要性水平越低/风险越高/保证程度越高→范围越广 |

**设计进一步审计程序的考虑因素：**
1. 风险的重要性（后果严重程度）
2. 重大错报发生的可能性
3. 涉及的交易/余额/披露的特征
4. 被审计单位采用的特定控制的性质
5. 是否拟获取内控有效性的审计证据

### 控制测试

**实施控制测试的两种情形：**
1. 评估认定层次重大错报风险时，预期控制运行有效
2. 仅实施实质性程序不能提供认定层次充分适当的审计证据

**控制测试的审计程序类型：**

| 类型 | 适用情况 | 注意事项 |
|------|----------|----------|
| 询问 | 获取控制运行信息 | 单独不充分，必须结合其他程序 |
| 观察 | 不留书面记录的控制（如职责分离） | 仅限观察时点，不在场时可能未执行 |
| 检查 | 留有书面证据的控制 | 签字/标记/偏差报告 |
| 重新执行 | 需验证控制实际执行效果 | 如核对价格一致性 |

**控制测试的时间规则：**
- 期中测试后→需获取剩余期间补充证据（考虑：风险严重程度、控制变化、期中证据程度、剩余期间长度、拟缩小实质性程序范围、控制环境）
- 以前审计证据→控制未变化且非特别风险控制→每三年至少测试一次；每次审计选取足够数量控制测试
- **特别风险控制→不得依赖以前审计证据，必须在本期测试**

**自动化控制测试的特别考虑：**
- IT处理内在一贯性→确定控制得到执行 + IT一般控制有效→可作为运行有效性证据
- 无需增加测试范围（除非系统变动）
- 需测试：相关IT一般控制有效性、系统是否变动及变动控制、是否使用授权软件版本

### 实质性程序

**必须实施的实质性程序：**
- 无论风险评估结果如何，都应针对所有重大交易类别/账户余额/披露实施实质性程序
- 将财务报表信息与会计记录核对/调节
- 检查编制过程中的重大会计分录和其他调整

**针对特别风险的实质性程序：**
- 应专门针对该风险设计
- 应包括细节测试（或细节测试+实质性分析程序）
- 仅实施实质性分析程序不充分

**实质性程序的时间选择：**
- 期中实施的考虑因素：控制环境、信息可获得性、程序目的、评估的风险、认定性质、能否降低期末未发现错报风险
- 期中实施后→针对剩余期间实施进一步实质性程序或结合控制测试
- 舞弊导致的重大错报风险→应在期末或接近期末实施
- 以前审计的实质性程序证据→通常对本期只有很弱或没有证据效力

**实质性分析程序设计要点：**
1. 对特定认定的适用性
2. 数据可靠性（来源、可比性、性质、相关控制）
3. 预期值的准确程度
4. 已记录金额与预期值之间可接受的差异额

## 第九章 信息技术对审计的影响

### IT对财务报告和内部控制的影响

**IT对财务报告的主要变化：**
- 计算机输入输出代替人工记录
- 电子影像代替纸质凭证
- 计算机文档代替纸质账簿
- 管理需求固化到应用程序中
- 系统性问题比偶然性误差更普遍

**有效信息系统应实现的功能：**
1. 识别和记录全部经授权的交易
2. 及时详细记录并恰当分类
3. 恰当衡量交易价值
4. 确定交易期间并记录在恰当会计期间
5. 恰当披露交易信息

**IT对内部控制的双重影响：**

| 优势 | 风险 |
|------|------|
| 一贯运用业务规则处理大量交易 | 系统/程序错误处理数据 |
| 不容易被绕过 | 未经授权访问导致数据毁损/不当修改 |
| 安全控制实现有效职责分离 | IT人员获得超越职责的权限 |
| 提高信息及时性/准确性/可获得性 | 未经授权改变主文档/程序 |
| 提高监督能力 | 数据丢失或无法访问 |

### IT一般控制

**四大领域：**

| 领域 | 目标 | 关键控制要素 |
|------|------|-------------|
| 程序开发 | 确保系统开发/配置/实施能实现管理层信息处理控制目标 | 开发管理方法、测试和质量确保、数据迁移、实施和应急计划、职责分离 |
| 程序变更 | 确保变更经过请求/授权/执行/测试/实施 | 变更请求授权跟踪、测试质量确保、紧急变更管理、职责分离 |
| 程序和数据访问 | 确保访问权限经身份认证和授权 | 用户授权管理、高权限用户管理、职责分工、认证密码控制、用户监控 |
| 计算机运行 | 确保系统完整准确运行，问题被识别和解决 | 系统作业管理、问题故障管理、数据备份恢复、灾难恢复 |

**IT一般控制的作用：**
- 对信息处理控制有效性具有普遍性影响
- 有效的IT一般控制→可信赖信息处理控制→控制风险评估为低于"最高"
- 无效的IT一般控制→即使信息处理控制设计有效，也不能信赖其持续有效运行

### IT应用控制（信息处理控制）

**关注的四个要素：**
1. **完整性**——数据传输/记录/处理的完整性
2. **准确性**——系统运算逻辑的准确性
3. **授权**——业务操作的权限设定和授予
4. **访问限制**——逻辑校验控制（限制检查、合理性检查、存在检查、格式检查）

**常见自动化控制类型：**
- 系统自动生成报告（账龄报告、差异报告等）
- 系统配置和科目映射（数据校验、编辑检查、阈值设定）
- 接口控制（系统间数据传输的完整性和准确性）
- 访问和权限控制（各部门/岗位权限定义和部署）

**三层控制关系：**
- 公司层面IT控制 → 整体控制环境，决定风险基调
- IT一般控制 → 基础，有效性直接关系到信息处理控制能否被信赖
- 信息处理控制 → 直接防止/发现/纠正认定层次错报

### 计算机辅助审计技术（CAATs）

**两大类别：**

| 类别 | 方法 | 说明 |
|------|------|------|
| 面向系统 | 平行模拟法 | CPA用自身软件+被审计单位数据，执行同样操作验证 |
| | 测试数据法 | CPA准备测试数据，用被审计单位系统处理验证控制 |
| | 嵌入审计模块法 | 在被审计单位系统中嵌入模块识别特定交易 |
| | 程序编码审查 | 发现冗余/错误/恶意代码 |
| | 程序代码比较和跟踪 | 不同版本代码比对 |
| 面向数据 | 数据查询、账表分析、审计抽样、统计分析、数值分析 | 分析电子数据 |

**CAATs的应用领域：**
- 最广泛应用于实质性程序（特别是实质性分析程序）
- 可用于控制测试（穿行测试、测试每笔交易）
- 可辅助舞弊检查（审计非正常日记账）

**常用工具：**
- 通用类：Excel、Access
- 数据库类：SQL Server、Oracle
- 专业工具类：ACL、IDEA

**IT审计范围确定的三个维度：**
1. 业务流程复杂程度（涉及人员/部门多少、操作决策数量、数据处理复杂性）
2. 信息系统复杂程度（商业软件vs自研、客制化程度、交易数量）
3. IT环境规模和复杂程度（系统数量、接口、网络规模、外包情况）

**审计策略选择：**
- 不太复杂IT环境→"绕过计算机审计"（了解但不测试IT控制有效性，依赖实质性程序）
- 较为复杂IT环境→"穿过计算机审计"（必须测试IT控制，运用CAATs）

# 第三编 各类交易和账户余额的审计

## 第十章 销售与收款循环的审计

### 业务循环与财务报表项目对照

| 业务循环 | 资产负债表项目 | 利润表项目 |
|---------|-------------|-----------|
| 销售与收款 | 应收票据、应收账款、应收款项融资、合同资产、预收款项、合同负债 | 营业收入、税金及附加、信用减值损失 |
| 采购与付款 | 预付款项、固定资产、在建工程、无形资产、应付票据、应付账款 | 销售费用、管理费用、研发费用 |
| 生产与存货 | 存货 | 营业成本、资产减值损失 |
| 投资与筹资 | 交易性金融资产、长期股权投资、短期借款、长期借款、实收资本等 | 财务费用、投资收益等 |

### 涉及的主要凭证和记录

- **销售类**：客户订购单→销售单→出库单→销售发票→商品价目表→贷项通知单→主营业务收入明细账→转账凭证
- **收款类**：汇款通知书→现金/银行存款日记账→应收账款明细账→坏账核销审批表→客户对账单

### 关键内部控制

| 控制要点 | 具体要求 |
|---------|---------|
| 职责分离 | 销售/发货/收款三部门分设；编制发票与开具发票分离；销售人员不接触现款；记账与出纳分离 |
| 授权审批 | 赊销须信用部门批准；发货须经批准的销售单；价格须按授权价目表；超权限须特别审批 |
| 凭证编号 | 销售单、出库单、发票预先连续编号；定期清点检查缺号重号 |
| 定期对账 | 由非出纳/非记账人员定期向客户发送对账单；不符事项由独立主管处理 |
| 信用控制 | 每个客户设信用额度；新客户须信用调查；赊销金额不超信用限额 |

### 收入确认常见舞弊手段

**虚增收入方式：**
1. 虚构销售交易（无实物流转/有实物流转配合）
2. 显失公允的关联方交易
3. 客户未取得控制权前确认收入
4. 隐瞒退货条款全额确认
5. 隐瞒售后回购/售后租回协议
6. 代理人按主要责任人确认（总额法vs净额法错误）
7. 高估履约进度多确认收入
8. 随意变更会计政策/估计

**虚构收入的资金配合手段：**
- 虚假采购套取资金→货款回笼
- 虚假预付款项套取资金
- 虚增长期资产采购金额
- 通过被投资单位套取资金
- 负债不入账或虚减负债
- 伪造回款单据
- 实际控制人提供体外资金

**舞弊风险迹象：**
- 客户所处行业景气度下降但销售增长
- 与同一客户同时发生销售和采购
- 主要客户规模与交易规模不匹配
- 临近期末大量/大额交易
- 销售价格明显异常
- 期后大量退回
- 应收款项付款方与购买方不一致
- 在货币资金充足时仍大额举债

### 营业收入主要实质性程序

1. **实质性分析程序**：月度/季度收入趋势、毛利率变动、收入与现金/应收/存货变动匹配、财务指标与同行业比较、收入与非财务信息（产能/能耗/运输量）关系
2. **检查收入确认方法**：判断时段/时点履约、评价会计政策合规性
3. **凭证检查**（发生认定）：从明细账→订购单/销售单/出库单/发票
4. **完整性测试**：从出库单（客户签收联）→明细账
5. **截止测试**：
   - 路径一：账簿记录→记账凭证→客户签收出库单（防多记）
   - 路径二：客户签收出库单→账簿记录（防漏记）
6. **函证**：结合应收账款函证验证本期销售额
7. **延伸检查**（识别舞弊风险时）：实地走访客户/供应商、查询最终控制人、检查经销商最终销售、获取关联方银行流水

### 应收账款主要实质性程序

1. **函证程序**：
   - 通常采用积极式函证
   - 选择对象：大额项目 + 风险项目（长账龄、纠纷、关联方、新客户、频繁交易但余额小）
   - 注册会计师对全过程保持控制
   - 未回函替代程序：检查期后收款（核对收款单据）、检查销售合同/出库单、检查往来邮件

2. **坏账准备测试**：
   - 测试账龄分析表准确性（抽样追查至原始凭证）
   - 评价预期信用损失模型和参数
   - 检查坏账核销的授权审批
   - 检查已核销坏账重新收回的处理

## 第十一章 采购与付款循环的审计

### 涉及的主要凭证和记录

- **采购类**：采购计划→供应商清单→请购单→订购单→验收及入库单→卖方发票→付款凭单
- **付款类**：转账/付款凭证→应付账款明细账→现金/银行存款日记账→供应商对账单

### 关键内部控制

| 控制环节 | 控制要点 |
|---------|---------|
| 职责分离 | 请购与审批分离；询价与确定供应商分离；采购与验收分离；付款审批与执行分离 |
| 采购授权 | 仅对经批准的请购单发出订购单；大额采购招标确定供应商 |
| 验收控制 | 验收部门独立；比较收货与订购单；编制预先编号验收单 |
| 三单核对 | 记录前核对订购单+验收单+供应商发票的一致性 |
| 付款控制 | 经授权人员签发支票；支票后附已批准凭单；签发后注销凭证防重复付款 |
| 定期对账 | 与供应商定期核对应付账款/预付款项 |

### 主要重大错报风险

1. **未完整记录负债**（完整性）：遗漏已收货未收票的负债、未记录已购买服务的支出
2. **多计或少计费用**：操纵损益或私人费用入账
3. **费用记录不准确**：复杂交易安排导致分配/计提错误
4. **外币交易记录不正确**：汇率错误、进口费用未正确分摊
5. **未记录的权利和义务**：分类错误或披露不充分

### 应付账款主要实质性程序

1. **函证**：从采购部门获取供应商清单（非仅财务部门），确保完整性后选样函证
2. **截止测试**：
   - 检查期后应付账款贷方发生额凭证（关注验收单/发票日期）
   - 结合存货监盘检查资产负债表日前后入库资料
3. **寻找未入账负债**：
   - 获取期后收取/记录/支付的发票明细
   - 检查支持性文件确定收货/接受服务日期
   - 追踪至应付账款明细账/暂估入账/预提费用
4. **供应商对账**：获取并检查与供应商的对账单及差异调节表

### 一般费用实质性程序要点

- 实质性分析程序：按费用类别细化，与预算/销售量/人数等关键因素建立预期值
- 选取样本检查支持性文件（原始凭证齐全性、记账凭证相符性）
- 期后付款凭证截止测试（关注发票日期和支付日期）

## 第十二章 生产与存货循环的审计

### 涉及的主要凭证和记录

- **生产类**：生产通知单→领料单→产量/工时记录→材料/人工/制造费用分配表→成本计算单→入库单
- **存货管理类**：验收单→入库单→存货台账→盘点表/标签→出库单→存货货龄分析表→可变现净值计算表

### 关键内部控制

| 控制环节 | 控制要点 |
|---------|---------|
| 生产授权 | 根据订单/销售预测签发预先编号的生产通知单 |
| 领料控制 | 领料单须经生产主管批准；一式三联（生产/仓库/财务） |
| 成本核算 | 材料/人工/制造费用分别归集分配；成本计算表经财务经理复核 |
| 产成品入库 | 质检→验收单→仓库清点→入库单→系统更新台账 |
| 存货保管 | 安全环境存放；限制无关人员接触；分类存放并标签 |
| 存货盘点 | 盘点人员含仓储外人员；初盘复盘分人；盘点表预先编号；贴标签防遗漏重复 |
| 跌价准备 | 定期编制货龄分析表；生产/仓储部门上报残冷背次明细；财务分析计提经总经理批准 |

### 主要重大错报风险

1. 存货实物不存在（存在认定）
2. 属于被审计单位的存货未入账（完整性认定）
3. 存货所有权不属于被审计单位（权利和义务认定）
4. 存货单位成本计算错误（准确性、计价和分摊认定）
5. 存货账面价值无法实现/跌价准备不充分（准确性、计价和分摊认定）

### 存货监盘程序

**监盘计划考虑因素：**
- 存货相关重大错报风险（数量/种类/成本归集难度/陈旧速度/失窃风险）
- 内部控制有效性
- 盘点程序是否适当
- 盘点时间安排（是否为资产负债表日）
- 盘存制度（实地盘存制/永续盘存制）
- 存放地点选择（获取完整仓库清单、根据重要性和风险选择）
- 是否需要专家协助

**监盘实施要点：**
1. 评价管理层盘点指令和程序
2. 观察盘点程序执行情况（特别是存货移动控制和截止控制）
3. 检查存货（确定存在、识别过时/毁损/陈旧）
4. 执行抽盘（盘点记录→实物 + 实物→盘点记录双向测试）
5. 获取截止信息（盘点日前后出入库凭证编号/记录）
6. 结束时确认所有存货已盘点、回收并核对盘点表单

**特殊情况处理：**
- 监盘不可行→替代程序（检查期后出售文件记录）
- 不可预见情况→另择日期+间隔期交易审计
- 第三方保管→函证+实施检查/安排监盘

### 存货计价测试

**单位成本测试：**
- 原材料：核对采购凭证（合同/订单/发票）+ 验证计价方法运用
- 产成品/在产品：
  - 直接材料成本：生产指令×定额单耗×实际单价 vs 成本计算单
  - 直接人工成本：工时记录×工资率 vs 成本计算单
  - 制造费用：分配标准合理性 + 分配计算正确性
  - 完工/在产品分配：在产品数量+约当产量+分配标准合理性

**存货跌价准备测试：**
- 识别需计提项目：货龄分析表+监盘中发现的毁损/陈旧存货+询问管理层
- 检查可变现净值：估计售价-至完工成本-销售费用-相关税费，抽样检查依据

### 存货相关舞弊应对

- **虚构存货**：严格监盘程序；关注大额/长库龄/低周转存货；分析存货结构与收入结构匹配性；异地/第三方存货严格函证或异地监盘
- **账外存货**：关注其他资产审计中转移资产情况；关注异常大额盘亏/报废；监盘中关注所有权和完整性；关注多结转成本/多报耗用/少报入库

## 第十三章 货币资金的审计

### 货币资金内部控制要点

| 控制领域 | 关键要求 |
|---------|---------|
| 岗位分离 | 出纳不兼稽核/档案保管/收支费用债权债务记账；不得一人办理全过程 |
| 授权审批 | 支付申请→支付审批→支付复核→办理支付四步流程；重要支付集体决策 |
| 银行账户 | 开立须符合经营需要；定期检查清理；不得出租出借 |
| 银行调节 | 每月至少核对一次；出纳以外人员编制余额调节表；会计主管复核 |
| 印章管理 | 财务专用章与个人名章分人保管；严禁一人保管全部印章 |
| 票据管理 | 专设登记簿；空白票据存保险柜；定期盘点核对 |

### 货币资金舞弊风险迹象

- 现金交易比例高且与行业不符
- 银行账户数量与业务规模不匹配
- 在无经营业务地区开户或异地存放高额资金
- 资金存放于个人账户
- 经营活动现金流与净利润不匹配
- 不能提供银行对账单/余额调节表
- 长期或大量银行未达账项
- 明细账存在非正常转账（相同金额一收一付）
- 存款收益与存款规模不匹配
- 大额货币资金情况下仍高额举债或频繁债务违约
- 与实际控制人/银行签订集团现金管理账户协议

### 银行存款主要实质性程序

1. **核对余额**：明细表加计→与总账/日记账核对→非记账本位币折算检查
2. **银行账户完整性检查**：到人民银行/基本户开户行打印《已开立银行结算账户清单》
3. **实质性分析程序**：计算存款应收利息 vs 实际利息收入，评估合理性
4. **银行存款发生额检查**：
   - 获取全部银行对账单
   - 银行对账单与日记账双向核对（日期/金额/借贷方向/收付款单位/摘要）
   - 关注大额异常发生额（相同金额一收一付等）
5. **银行存款余额调节表检查**：
   - 验证加计正确、调节后余额一致
   - 企付银未付→检查付款原始凭证+期后对账单反映
   - 企收银未收→检查收款入账凭证+期后对账单反映
   - 银收企未收/银付企未付→确定是否为截止错报
   - 关注长期未达账项（可能挪用资金）
6. **银行函证**：
   - 对所有银行存款（含零余额和已注销账户）实施函证
   - 函证内容：存款余额、借款、担保、承兑汇票、贴现、信用证、外汇合约、托管证券、理财产品、资金归集账户等
   - 对全过程保持控制；采用公示地址直接发出/收回
   - 回函与对账单/账面记录核对
7. **截止测试**：选取资产负债表日前后大额凭证，关注业务内容和对应项目
8. **定期存款审计**：
   - 评价商业理由合理性
   - 监盘存款凭据/观察网银查询
   - 检查是否被质押或限制使用
   - 分析利息收入合理性（远大于应得→账外存款；远小于应得→挪用/虚构）
   - 检查期后到期结转/提取的兑付凭证

### 货币资金舞弊应对措施

| 舞弊类型 | 应对程序 |
|---------|---------|
| 虚构货币资金 | 严格函证+保持控制；检查银行账户完整性和对账单真实性；分析利息收入合理性；关注"存贷双高" |
| 大股东侵占 | 识别与实控人/控股股东大额资金往来；识别疑似关联方异常交易；关注期后资金划转和受限情况；了解实控人财务状况和股权质押 |
| 虚构现金交易 | 评价现金交易合理性；计算现金收付比例是否匹配业务性质；检查大额现金收支来源去向；核对至原始单据 |

# 第四编 审计中对特殊事项的考虑

## 第14章 对舞弊和法律法规的考虑

### 舞弊三角理论与风险因素

| 要素 | 含义 | 典型表现 |
|------|------|----------|
| 动机/压力 | 实施舞弊的驱动力 | 财务业绩压力、个人经济困难、激励政策 |
| 机会 | 实施舞弊的可能性 | 内控薄弱、管理层凌驾控制、缺乏监督 |
| 借口/态度 | 合理化舞弊行为 | "只是暂时借用"、行业惯例、管理层诚信缺失 |

**舞弊的两种主要类型：**
1. **编制虚假财务报告**：操纵/伪造/篡改会计记录或凭证、歪曲/故意遗漏重大事项、故意误用会计政策
2. **侵占资产**：挪用收入、盗窃资产、利用关联方以高于公允价值购买资产

### 注册会计师的责任

- 对财务报表整体不存在舞弊或错误导致的重大错报获取**合理保证**（非绝对保证）
- 在整个审计过程中保持**职业怀疑**
- 收入确认推定为舞弊风险（除非有证据推翻）
- 管理层凌驾于内部控制之上的风险始终作为特别风险

### 应对舞弊风险的程序

| 层次 | 应对措施 |
|------|----------|
| 总体应对 | 指派更有经验的人员、增加不可预见性、修改审计程序的性质/时间/范围 |
| 认定层次 | 改变实质性程序的性质（更可靠的证据）、扩大样本量、接近期末实施 |
| 管理层凌驾 | ①测试日常会计分录和调整分录 ②复核会计估计是否存在偏向 ③评价异常重大交易的商业理由 |

### 评价审计证据

- 临近审计结束时的分析程序是否表明存在此前未识别的舞弊风险
- 识别出错报时→评价是否表明存在舞弊→评价对审计其他方面的影响
- 涉及管理层的错报（无论是否重大）→重新评价舞弊风险评估结果

### 与管理层/治理层的沟通

| 沟通对象 | 沟通事项 |
|----------|----------|
| 管理层 | 存在或可能存在舞弊时及时提请关注（至少高出涉嫌人员一个级别） |
| 治理层 | 涉及管理层/内控重要人员的舞弊或舞弊嫌疑 |
| 外部机构 | 法律法规要求时向监管机构报告 |

### 书面声明（舞弊相关）

管理层应确认：①设计/执行/维护防止和发现舞弊的内控的责任 ②已披露舞弊风险评估结果 ③已披露已知舞弊/舞弊嫌疑 ④已披露舞弊指控

### 对法律法规的考虑

**两类法律法规及注册会计师责任：**

| 类别 | 特征 | CPA责任 |
|------|------|---------|
| 第一类 | 对财务报表金额和披露有直接影响（如税法） | 获取遵守规定的充分、适当审计证据 |
| 第二类 | 对经营活动至关重要但不直接影响报表（如经营许可） | 仅实施特定程序以识别可能的重大影响 |

**识别违法行为的迹象：** 受到监管调查/罚款、异常付款、采购价格异常、避税天堂交易、未授权交易等

**违法行为对审计报告的影响：**
- 重大影响未恰当反映→保留/否定意见
- 管理层阻挠无法获取证据→保留/无法表示意见

## 第15章 审计沟通

### 与治理层沟通的事项

| 沟通事项 | 要点 |
|----------|------|
| CPA责任 | 对财务报表形成和发表意见；审计不减轻管理层/治理层责任 |
| 计划的审计范围和时间 | 特别风险应对、内控方案、重要性运用、专家利用、关键审计事项初步判断 |
| 审计中的重大发现 | 会计实务质量看法、重大困难、与管理层讨论的重大事项、影响审计报告形式的情形 |
| 独立性（上市实体） | 独立性声明、可能影响独立性的关系和事项、防范措施 |
| 值得关注的内控缺陷 | 以书面形式及时通报 |

### 沟通形式与时间

| 要求 | 说明 |
|------|------|
| 书面沟通 | 审计中发现的重大问题（口头不适当时）、独立性、值得关注的内控缺陷 |
| 及时性 | 计划事项→审计早期；重大困难→尽快；关键审计事项→可在不同时点 |
| 充分性评价 | 治理层措施的适当性和及时性、坦率程度、理解能力 |

### 前后任注册会计师的沟通

**核心规则：** 后任主动发起 + 征得被审计单位同意 + 记录于工作底稿

**接受委托前（必要程序）：**

| 询问事项 | 目的 |
|----------|------|
| 管理层诚信问题 | 评估客户风险 |
| 重大会计/审计意见分歧 | 识别"购买审计意见"风险 |
| 舞弊/违法行为/内控缺陷 | 评估审计风险 |
| 变更事务所的原因 | 判断是否接受委托 |

- 被审计单位不同意前任答复或限制答复范围→一般应拒绝接受委托
- 前任答复有限→应说明原因

**接受委托后（非必要程序）：**
- 查阅前任工作底稿需征得被审计单位同意
- 前任自主决定可供查阅的内容
- 双方就使用目的、范围和责任达成一致

## 第16章 利用他人的工作

### 利用内部审计工作

**能否利用的评价标准（三项均须满足）：**
1. 内部审计在被审计单位中的地位 + 相关政策程序支持客观性的程度
2. 内部审计人员的胜任能力
3. 是否采用系统、规范化的方法（含质量管理）

**不得利用的情形：** 上述三项任一不满足

**应当较少利用的情形：**
- 涉及较多判断
- 评估的重大错报风险较高/特别风险
- 客观性支持程度较弱
- 胜任能力较低

**利用内部审计人员提供直接协助的限制：**
- 不得涉及作出重大判断的程序
- 不得涉及较高重大错报风险且需较多判断的程序
- 不得涉及内审已参与并将向管理层报告的工作

### 利用专家的工作

**专家类型：** 内部专家（事务所合伙人/员工）、外部专家

**评价专家工作的要点：**

| 评价内容 | 关注事项 |
|----------|----------|
| 胜任能力/专业素质/客观性 | 专长性质和水平、利益冲突 |
| 方法和假设 | 是否得到普遍认可、是否符合财务报告编制基础 |
| 原始数据 | 相关性、完整性、准确性 |
| 工作结果 | 与其他审计证据的一致性 |

**审计报告中提及专家：**
- 无保留意见→不应提及（除非法律法规要求）
- 非无保留意见→可提及但须指明不减轻CPA责任

## 第17章 集团财务报表审计

### 核心概念

| 概念 | 定义 |
|------|------|
| 重要组成部分 | ①单个组成部分对集团具有财务重大性 ②可能存在导致集团报表重大错报的特别风险 |
| 组成部分重要性 | 低于集团财务报表整体重要性；不同组成部分可不同；无需按比例分配 |
| 集团项目组责任 | 对集团审计意见负全部责任，不因利用组成部分CPA工作而减轻 |

### 识别重要组成部分的基准

- 选定基准（资产/负债/现金流/利润/收入）× 百分比（通常15%左右）
- 特定性质或情况导致特别风险的组成部分也可识别为重要组成部分

### 对组成部分执行工作的类型

| 组成部分类型 | 执行的工作 |
|-------------|-----------|
| 重要组成部分（财务重大性） | 使用组成部分重要性实施审计 |
| 重要组成部分（特别风险） | ①审计 ②针对特定账户/交易/披露审计 ③针对特别风险实施特定程序 |
| 不重要组成部分 | 集团层面分析程序 |
| 补充选择的不重要组成部分 | 审计/审阅/特定程序（定期轮换） |

### 集团项目组参与组成部分CPA工作

对重要组成部分至少应包括：
1. 讨论重要的组成部分业务活动
2. 讨论舞弊或错误导致重大错报的可能性
3. 复核识别出的特别风险相关工作底稿

### 与组成部分CPA的沟通

**集团项目组→组成部分CPA：** 职业道德要求、组成部分重要性和临界值、特别风险、关联方清单

**组成部分CPA→集团项目组：** 独立性确认、未更正错报清单、管理层偏向迹象、值得关注的内控缺陷、舞弊/违法信息

## 第18章 其他特殊项目的审计

### 会计估计审计

**风险评估关注点：**
- 估计不确定性程度
- 复杂性、主观性及其他固有风险因素
- 管理层偏向的可能性

**应对重大错报风险的三种方法（选一或多项）：**
1. 从截至审计报告日发生的事项获取审计证据
2. 测试管理层如何作出会计估计（方法→假设→数据→点估计选择）
3. 作出注册会计师的点估计或区间估计

**特别风险的应对：**
- 拟信赖控制→本期测试运行有效性
- 仅实施实质性程序→应包括细节测试

**管理层偏向迹象：**
- 主观认为环境变化并改变估计方法
- 选择有利于管理层目标的假设/数据
- 选择带有乐观/悲观倾向的点估计
- 点估计始终倾向区间估计的一端

### 关联方审计

**关联方交易的特殊风险：**
- 复杂的关系和组织结构
- 信息系统可能无法有效识别
- 可能未按正常市场条件进行
- 为管理层串通/隐瞒/操纵提供更多机会

**超出正常经营过程的重大关联方交易→确定为特别风险**

**审计程序要点：**
1. 检查合同/协议→评价商业理由、条款一致性、会计处理和披露
2. 获取已恰当授权和批准的证据
3. 对"按公平交易条款执行"的认定获取充分、适当审计证据

**识别出以前未识别的关联方时：**
- 立即通报项目组其他成员
- 要求管理层识别与新关联方的所有交易
- 实施恰当的实质性程序
- 评价管理层不披露是否有意（可能表明舞弊）

### 期初余额审计（首次接受委托）

**审计目标：** 确定期初余额是否存在重大错报、上期会计政策是否一贯运用

**审计程序：**
- 查阅前任CPA工作底稿
- 评价本期审计程序是否提供期初余额相关证据
- 实施专门的审计程序

**对审计意见的影响：**
- 无法获取充分证据→保留意见或无法表示意见
- 期初余额存在重大错报且未恰当反映→保留意见或否定意见

# 第五编 完成审计工作与出具审计报告

## 第19章 完成审计工作

### 期后事项

**三个时段及CPA责任：**

| 时段 | 期间 | CPA责任 |
|------|------|---------|
| 第一时段 | 财务报表日→审计报告日 | **主动识别**，设计专门审计程序 |
| 第二时段 | 审计报告日→财务报表报出日 | **被动识别**，无义务实施程序但知悉后须处理 |
| 第三时段 | 财务报表报出日后 | **无义务**，但知悉后须处理（限审计报告日已存在的事实） |

**期后事项分类：**
- **调整事项**：对财务报表日已存在情况提供进一步证据→调整报表金额
- **非调整事项**：财务报表日后发生的情况→附注披露

**第一时段审计程序：**
1. 了解管理层识别期后事项的程序
2. 询问是否发生可能影响报表的事项
3. 查阅报表日后的会议纪要
4. 查阅最近的中期报表（如有）

**第二时段处理：**

| 情形 | 处理方式 |
|------|----------|
| 管理层修改报表 | 实施必要审计程序 + 出具新审计报告（日期不早于修改后报表批准日） |
| 管理层不修改且报告未提交 | 发表非无保留意见后再提交 |
| 管理层不修改且报告已提交 | 通知不要报出；若仍报出→采取措施防止使用者信赖 |

**第三时段处理：**
- 管理层修改→实施必要程序 + 复核通知措施 + 出具新报告 + 增加强调/其他事项段
- 管理层不修改→通知后采取措施防止使用者信赖审计报告

### 持续经营

**可能导致重大疑虑的事项：**

| 方面 | 典型情形 |
|------|----------|
| 财务 | 净资产为负、营运资金负数、借款到期无法偿还、经营现金流为负、关键财务比率不佳 |
| 经营 | 计划清算/终止、关键管理人员离职无替代、失去主要市场/客户/供应商 |
| 其他 | 违反资本/法定要求、未决诉讼可能无法支付、法规政策不利变化 |

**CPA责任：** 就管理层运用持续经营假设的适当性获取充分、适当审计证据，并就是否存在重大不确定性得出结论

### 书面声明

**必须获取的书面声明：**
1. 管理层确认已履行编制报表的责任
2. 已提供所有相关信息并允许不受限制地接触
3. 所有交易均已记录并反映在报表中

**书面声明的日期：** 尽量接近审计报告日，不得在其之后

**管理层不提供书面声明的后果：**
- 不提供针对管理层责任的声明→无法表示意见
- 对管理层诚信产生重大疑虑致声明不可靠→无法表示意见

## 第20章 审计报告

### 审计意见类型

| 意见类型 | 适用情形 | 影响程度 |
|----------|----------|----------|
| 无保留意见 | 财务报表在所有重大方面按照编制基础编制并公允反映 | — |
| 保留意见 | 存在重大错报但不广泛，或无法获取证据但可能影响不广泛 | 重大但不广泛 |
| 否定意见 | 存在重大错报且影响广泛 | 重大且广泛 |
| 无法表示意见 | 无法获取充分、适当审计证据且可能影响广泛 | 可能重大且广泛 |

**决策逻辑：**

```
存在重大错报？
├─ 是 → 影响是否广泛？
│       ├─ 不广泛 → 保留意见
│       └─ 广泛 → 否定意见
└─ 无法获取充分证据 → 可能影响是否广泛？
        ├─ 不广泛 → 保留意见
        └─ 广泛 → 无法表示意见
```

**"广泛"的判断标准：**
- 不限于财务报表的特定要素、账户或项目
- 虽限于特定要素但占报表重大比例
- 就披露而言对使用者理解报表至关重要

### 非无保留意见审计报告的修改

| 修改内容 | 保留意见 | 否定意见 | 无法表示意见 |
|----------|----------|----------|-------------|
| 意见段标题 | "保留意见" | "否定意见" | "无法表示意见" |
| 意见段措辞 | "除…影响外，公允反映" | "由于…重大影响，未能公允反映" | "未能获取…证据，无法发表意见" |
| 基础段标题 | "形成保留意见的基础" | "形成否定意见的基础" | "形成无法表示意见的基础" |
| CPA责任段 | 正常描述 | 正常描述 | 修改为"责任是…实施审计以出具报告"，删除"合理保证"等表述 |

### 关键审计事项（KAM）

**适用范围：** 上市实体财务报表审计（强制）；其他实体（自愿或法规要求）

**确定过程：**
1. 确定与治理层沟通的事项中最为重要的事项
2. 考虑因素：评估的重大错报风险较高的领域、涉及重大判断的领域、本期重大交易或事项的影响

**在审计报告中的描述：**
- 为何被认定为关键审计事项
- 如何在审计中应对该事项
- 提及相关披露（如适用）

**不沟通KAM的情形：**
- 法律法规禁止公开披露
- 极端罕见情况下，合理预期沟通造成的负面后果超过公众利益

### 强调事项段与其他事项段

| 类型 | 目的 | 位置 |
|------|------|------|
| 强调事项段 | 提请使用者关注报表中已恰当列报/披露的事项（对理解报表至关重要） | 紧接"形成意见的基础"之后（或KAM之后） |
| 其他事项段 | 提请使用者关注未在报表中列报/披露但与理解审计相关的事项 | 紧接"审计意见"和"形成意见的基础"之后（或KAM/强调事项段之后） |

**强调事项段典型情形：** 持续经营重大不确定性、重大期后事项、重大诉讼不确定性、提前运用新准则

### 比较信息

| 方法 | 对应数据 | 比较财务报表 |
|------|----------|-------------|
| 审计意见提及 | 仅提及本期 | 提及每一期间 |
| 上期由前任审计 | 其他事项段说明 | 可重新审计或说明 |
| 上期未经审计 | 其他事项段说明 | — |

## 第21章 审计工作底稿

### 审计工作底稿的性质和目的

**定义：** 注册会计师对制定的审计计划、实施的审计程序、获取的相关审计证据以及得出的审计结论作出的记录

**目的：**
1. 提供充分、适当的记录作为审计报告的基础
2. 提供证据证明审计工作按照审计准则和法律法规执行

**功能：** 协助计划和执行审计、协助指导/监督/复核、使项目组对工作负责、保留对未来审计有持续重大意义事项的记录、接受质量管理复核和检查

### 工作底稿的编制要求

**总体要求：** 使未曾接触该项审计工作的有经验的专业人士清楚了解：
- 实施的审计程序的性质、时间安排和范围
- 审计程序的结果和获取的审计证据
- 就重大事项得出的结论

**记录的内容：**
- 识别被审计项目或事项的特征
- 执行人员和复核人员的姓名及日期
- 与项目组内部/外部讨论重大事项的记录
- 识别出的信息与得出结论不一致时如何处理

### 归档要求

| 要求 | 内容 |
|------|------|
| 归档期限 | 审计报告日后**60天**内完成归整 |
| 保存期限 | 自审计报告日起不少于**10年** |
| 归档后的修改 | 不得删除或废弃；只能增加说明修改原因、时间和人员的补充文件 |
| 行政性变动 | 归档后可进行（如分类/排序/交叉索引），无需记录修改信息 |

### 审计工作底稿的所有权和保密

- **所有权**：属于会计师事务所
- **保密义务**：未经被审计单位同意不得向第三方提供（法律法规另有规定除外）
- **接触权限**：被审计单位不能以底稿替代自身会计记录

# 第六编 企业内部控制审计

## 第二十二章 企业内部控制审计

### 内部控制审计的概念与目标

- **审计对象**：企业财务报告内部控制的有效性
- **基准日**：通常为财务报表日（12月31日）
- **审计目标**：对财务报告内部控制的有效性发表审计意见，并对注意到的非财务报告内部控制重大缺陷进行披露
- **整合审计**：内部控制审计应与财务报表审计整合进行，两份报告签署相同日期

### 自上而下的方法

选择测试控制的逻辑路径：

1. **企业层面控制** → 2. **重要账户/列报及相关认定** → 3. **重要业务流程** → 4. **业务流程层面控制**

#### 企业层面控制测试要点

| 控制类别 | 关注内容 |
|---------|---------|
| 控制环境 | 管理层理念、诚信文化、治理层监督 |
| 凌驾风险控制 | 异常交易、关联方交易、重大估计、动机压力、举报制度 |
| 风险评估过程 | 是否系统识别和应对风险 |
| 信息传递与期末报告 | 总账登记、会计政策、调整分录、报表编制 |
| 内部监督 | 定期盘点核对、内审有效性、自我评价 |
| 监督经营成果 | 预算对比、经营指标分析 |

#### 业务流程层面控制

- **控制目标**：完整性、发生、准确性、截止、分类、正确汇总和过账
- **控制类型**：
  - 预防性控制：防止错报发生（如三单核对、审批程序）
  - 检查性控制：发现已发生的错报（如银行余额调节表、定期核对）
- **关键控制判断标准**：与认定直接相关、执行精确、能有效防止/发现重大错报

#### 控制测试样本规模

| 控制运行频率 | 最小样本规模 |
|------------|------------|
| 每天多次 | 25~60 |
| 每天1次 | 20~40 |
| 每周1次 | 5~15 |
| 每月1次 | 2~4 |
| 每季1次 | 2 |
| 每年1次 | 1 |

- 自动化控制：IT一般控制有效前提下，通常只需1次系统查询
- 多人执行同一控制：分别确定总体，针对每人确定样本

### 信息技术控制

| IT一般控制领域 | 目标 |
|--------------|------|
| 程序开发 | 系统开发实施能实现管理层信息处理控制目标 |
| 程序变更 | 变更经申请、授权、测试和实施 |
| 程序和数据访问 | 访问权限经身份认证和授权 |
| 计算机运行 | 系统正常运行，异常得到处理 |

### 内部控制缺陷分类

| 分类维度 | 类别 | 定义 |
|---------|------|------|
| 按性质 | 设计缺陷 | 缺少必需控制或设计不当，正常运行也难以实现目标 |
| 按性质 | 运行缺陷 | 设计适当但未按设计运行，或执行人员缺乏授权/胜任能力 |
| 按严重程度 | 重大缺陷 | 可能导致不能及时防止或发现并纠正财务报表重大错报 |
| 按严重程度 | 重要缺陷 | 严重程度不如重大缺陷，但足以引起监督人员关注 |
| 按严重程度 | 一般缺陷 | 除重大和重要缺陷之外的控制缺陷 |

#### 缺陷严重程度评价要素

- 控制不能防止/发现错报的**可能性**
- 潜在错报的**金额大小**
- 是否存在有效的**补偿性控制**
- 多项缺陷影响同一账户时需评估**组合效应**

#### 表明可能存在重大缺陷的迹象

1. 董事、监事和高管的任何舞弊
2. 重述以前公布的财务报表
3. 当期存在重大错报而内控未能发现
4. 审计委员会和内审对内控监督无效

#### 整改后控制的最短运行期间

| 控制频率 | 最短运行期间 | 最少测试数量 |
|---------|------------|------------|
| 每季1次 | 2个季度 | 2 |
| 每月1次 | 2个月 | 2 |
| 每周1次 | 5周 | 5 |
| 每天1次 | 20天 | 20 |
| 每天多次 | 25次（不少于15天） | 25 |

### 内部控制审计报告类型

| 意见类型 | 适用情形 |
|---------|---------|
| 无保留意见 | 所有重大方面保持有效内控，审计未受限制 |
| 带强调事项段的无保留意见 | 不存在重大缺陷但有重大事项需提请注意 |
| 否定意见 | 存在一项或多项重大缺陷 |
| 无法表示意见 | 审计范围受到限制 |

#### 非财务报告内控缺陷的处理

| 缺陷级别 | 处理方式 |
|---------|---------|
| 一般缺陷 | 与企业沟通，无需在报告中说明 |
| 重要缺陷 | 书面与董事会和管理层沟通，无需在报告中说明 |
| 重大缺陷 | 书面沟通 + 在报告中增加描述段披露 |

## 第二十三章 企业可持续信息鉴证

### 基本概念

- **定义**：鉴证者对被鉴证单位依据可持续信息披露准则编制的可持续信息执行鉴证工作，出具鉴证报告
- **鉴证范围**：可以是全部可持续信息，也可以是部分
- **准则依据**：《可持续信息鉴证业务准则第6101号——基本准则（试行）》（2026年1月）

### 合理保证与有限保证

| 对比维度 | 合理保证 | 有限保证 |
|---------|---------|---------|
| 保证程度 | 较高 | 较低（但应有意义） |
| 业务风险 | 可接受的低水平 | 可接受的水平 |
| 程序范围 | 较为充分 | 较为有限 |
| 证据要求 | 充分、适当 | 较为有限 |
| 意见方式 | 积极方式："我们认为……已按照……编制" | 消极方式："我们没有注意到……使我们认为……没有按照……编制" |
| 风险评估层次 | 披露认定层次 | 披露层次 |

- 可对部分信息获取合理保证、部分获取有限保证

### 鉴证业务承接条件

同时满足：
1. 职业道德要求（含独立性）能够得到遵守
2. 人员具备必要胜任能力和专业素质
3. 满足鉴证业务前提条件
4. 已就业务约定条款达成一致

### 主要鉴证程序

#### 风险评估程序

- 了解可持续事项和信息
- 确定披露标准是否适当
- 了解报告政策
- 了解被鉴证单位及其环境
- 了解法律和监管环境
- 了解内部控制体系（有限保证：询问为主；合理保证：询问+其他程序）

#### 进一步鉴证程序

| 程序类型 | 有限保证 | 合理保证 |
|---------|---------|---------|
| 控制测试 | 可选 | 必要时实施 |
| 实质性程序 | 根据职业判断确定 | 重大披露和高风险均需实施 |
| 分析程序 | 设定预期，询问差异原因 | 设定精确预期，调查差异并获取证据 |
| 管理层凌驾风险 | — | 评估为最高风险等级，专门应对 |

#### 完成鉴证工作

- 累积和评价错报（单独或汇总是否重大）
- 评价可持续信息对标准的描述
- 考虑期后事项
- 获取书面声明
- 阅读和考虑其他信息
- 与管理层和治理层沟通重大事项

### 鉴证报告

#### 报告要素

标题、收件人、鉴证意见、形成意见的依据、其他信息、管理层责任、固有限制、鉴证者责任、已实施工作概述（仅有限保证）、签名盖章、机构信息、报告日期

#### 报告标题

- "有限保证鉴证报告"
- "合理保证鉴证报告"
- "有限保证和合理保证鉴证报告"

#### 意见类型

| 类型 | 适用情形 |
|------|---------|
| 无保留意见 | 未注意到/确认不存在重大错报 |
| 保留意见 | 范围受限或存在错报，影响重大但非广泛 |
| 否定意见 | 存在重大错报 |
| 无法表示意见 | 范围受限严重，无法获取保证 |

### 工作底稿要求

- 归档期限：鉴证报告日后60天内
- 保存期限：自鉴证报告日起至少10年
- 所有权属于鉴证机构

## 第二十四章 会计师事务所业务质量管理

### 质量管理体系的目标

为以下两方面提供合理保证：
1. 事务所及其人员按照法律法规和职业准则履行职责并执行业务
2. 事务所和项目合伙人出具适合具体情况的业务报告

### 质量管理体系八大要素

| 序号 | 要素 | 核心内容 |
|------|------|---------|
| 1 | 风险评估程序 | 设定质量目标→识别评估质量风险→设计应对措施 |
| 2 | 治理和领导层 | 营造"质量至上"文化，设定高层基调 |
| 3 | 相关职业道德要求 | 独立性、关键审计合伙人轮换 |
| 4 | 客户关系和业务接受与保持 | 知己知彼，质量优先于商业利益 |
| 5 | 业务执行 | 项目合伙人总体责任、指导监督复核、咨询、意见分歧 |
| 6 | 资源 | 人力、技术、知识、财务资源 |
| 7 | 信息与沟通 | 内外部双向沟通，信息系统支持 |
| 8 | 监控和整改程序 | 持续监控、识别缺陷、及时整改 |

### 总体要求

1. 全所范围统一设计、实施和运行（含分所）
2. 风险导向思路：质量目标→质量风险→应对措施
3. 根据本所实际"量身定制"
4. 动态调整，不断优化

### 质量管理领导层三种角色

| 角色 | 责任 |
|------|------|
| 主要负责人（首席合伙人） | 对质量管理体系承担**最终责任** |
| 质量管理体系运行负责人 | 对体系**运行**承担责任 |
| 特定方面运行负责人 | 对体系**特定方面**运行承担责任 |

三类人员须具备：适当知识经验资质、权威性和影响力、充足时间和资源、理解并接受问责

### 项目质量复核

#### 适用范围

1. 上市实体财务报表审计业务
2. 法律法规要求实施的业务
3. 事务所认为有必要的业务

#### 复核人员要求

- 独立于项目组（项目组成员不得担任）
- 胜任能力至少与项目合伙人相当
- 遵守职业道德，保持独立客观公正
- 冷却期：前任项目合伙人至少2年后方可担任
- 避免同年度交叉复核

#### 复核程序

1. 阅读了解项目和客户信息
2. 与项目合伙人讨论重大事项和重大判断
3. 选取工作底稿复核：判断依据、底稿支持性、结论恰当性
4. 评价独立性遵守依据
5. 评价咨询和意见分歧处理
6. 复核财务报表和审计报告

#### 与项目组内部复核的区别

| 对比维度 | 项目质量复核 | 项目组内部复核 |
|---------|------------|------------|
| 主体 | 独立于项目组的复核人员 | 项目组内部人员 |
| 适用范围 | 特定业务 | 所有业务 |
| 内容 | 重大判断和结论 | 项目各方面 |

### 项目合伙人的质量管理责任

#### 总体责任

- 对管理和实现审计项目高质量承担**总体责任**
- 充分、适当地参与整个审计过程
- 确定重大判断和结论是否适当

#### 具体责任领域

| 领域 | 关键要求 |
|------|---------|
| 职业道德 | 确保项目组了解并遵守，签署报告前确认独立性 |
| 业务接受与保持 | 确认政策已遵守、结论适当 |
| 资源 | 确保充分适当资源已分配，项目组具备胜任能力和时间 |
| 指导监督复核 | 追踪进程、解决问题、识别咨询事项、复核工作底稿 |
| 咨询 | 困难/争议事项进行咨询，确认结论达成一致并执行 |
| 项目质量复核 | 配合复核人员，复核完成后方可签署报告 |
| 意见分歧 | 按政策处理，分歧解决前不得签署报告 |
| 监控与整改 | 了解相关信息，评估对项目影响，保持警觉 |

#### 签署审计报告前的确认事项

1. 已充分适当参与全过程
2. 重大判断和结论适当
3. 职业道德和独立性已遵守
4. 已获取充分适当审计证据
5. 项目质量复核已完成（如适用）

# 审计方法论（实务心智模型）

> 核心框架：业务优先于财务，风险驱动审计程序，底稿逻辑贯穿始终。

## 八大心智模型

### 模型1: 业务优先于财务

先有业务，后有财务，最后才是审计。在财务报表中审计，无异于在圆圈中画圆。

分析任何审计问题时，先问"这个公司的业务是什么"，而不是"这个科目的会计处理对不对"。

经典案例：瑞幸做空报告——浑水从门店销售数据切入而非财务报表。

### 模型2: 底稿逻辑论（横向+纵向+洋葱+收口）

底稿不是填表格，是要有连贯的执业思路。

| 维度 | 含义 |
|------|------|
| 横向逻辑 | 各工作表之间串联，分析程序为后续底稿提供方向和指引 |
| 纵向逻辑 | "我要干什么"→"我怎么干的"→"我干的结果是什么" |
| 洋葱法 | 从面到点层层展开，不要上来一下钻到心里去 |
| 收口思维 | 样本随程序深入逐渐收口，越往后做样本量越少 |
| 扣题原则 | 审计说明必须呼应审计目标，总分总结构 |

### 模型3: 接受结果论

所有的结果都是最好的结果，因为无论什么事情只要发生了，就是不可逆的。迅速切换到"从中能学到什么"的模式。

### 模型4: 行动优先

不干永远0分，只要干就有得分的可能。先做起来再说，在实践中解决问题。

### 模型5: 60分哲学

十个任务每个60分，比六个100分加四个0分强。多任务环境下优先保证每个任务的基本完成度。

### 模型6: AI放大器论

AI是放大器，1会变成100，但-1同样会变成-100。核心能力在人的底层逻辑，不在工具本身。底层逻辑错的话，工具只会放大错误。

### 模型7: 反模板手搓党

模板的功能是参考和指引，并不是标准。模板是死的，但业务是活的。先理解背后逻辑，再根据实际情况调整。

### 模型8: 三大表联动思维

不孤立看单个科目，通过资产负债表、利润表、现金流表之间的业务逻辑联动来识别风险。

经典案例：在建工程不转固→利息资本化不配比→付息债务增加→持续经营风险（全程三表联动）。

## 十一条决策启发式

1. **先看业务，再看财务**：不了解业务的审计四舍五入约等于抄数。
2. **风险评估不能架空**：先评估风险，再针对性地设计程序。不能一竿子怼到细节测试。
3. **样本选择要有逻辑**：不是抽得越多越好，重要的是为什么抽以及抽什么。
4. **遇到问题早汇报**：早汇报的"挨卷"和拖到最后导致项目滞后的"挨卷"不是一个程度的。
5. **不要和一个事死磕**：工作是多线程的，后面可能还有更恶心的事等着你。
6. **工具为人服务，不是人为工具服务**：选工具看应用场景，不看流行度。
7. **职业怀疑不是杯弓蛇影**：保持怀疑，但不是所有异常都等于风险。
8. **先判断值不值得做**：在做任何程序之前，先想想要不要做。
9. **反馈要及时，闭环要完整**：底稿最怕"有头没尾"——做了风险评估没做风险应对。
10. **被骂两次才是真正的失败**：第一次被骂是学习机会，同样的事被骂两次是浪费。
11. **存货跟未来相关，不跟过去相关**：存货结存数量跟在手订单和销售预期相关，不是跟历史收入相关。

## 审计分析框架（四维度）

### 维度1：看业务本质
- 主营业务是什么？商业模式是什么？
- 行业的核心指标是什么？（制造业：产能利用率；快消品：单位门店销售）
- 同行业可比公司情况？行业整体趋势？
- 上下游产业链结构？谁有定价权？

### 维度2：看风险信号（逆向思考）
- 有没有"八字"风险？（农林牧渔、衣食住行）
- 收入变动的主要驱动因素？价格还是数量？
- 有没有违反常识的异常？（新增小客户追着付钱、长账龄突然集体还款）
- 管理层有没有质押股份？关联方关系异常？

### 维度3：看审计逻辑
- 风险评估是否充分？还是直接跳到细节测试？
- 审计程序是否和风险识别结果挂钩？有没有"有头没尾"？
- 样本选择是否合理？
- 底稿有没有形成横向逻辑和纵向逻辑？

### 维度4：看工具效率
- 有没有自动化工具可以提升效率？
- 这个问题值不值得花这么多时间？
- 数据源的质量如何？处理数据之前有没有先做数据清理？

## 通用科目核对方法论（GL Reconciliation）

### 概述

科目核对是审计和财务的基础工作：将总账余额与明细账/子系统/外部数据源逐笔比对，找出差异（break），追溯原因，出具核对报告。

银行余额调节表是最常见的一种，但同样的方法论适用于所有科目核对。

### 核对类型

| 核对场景 | 总账侧 | 对方侧 | 常见差异原因 |
|----------|--------|--------|-------------|
| 银行存款 | 银行日记账 | 银行对账单 | 未达账项（时间性差异） |
| 应收账款 | 应收明细账 | 客户对账单/回函 | 发货未开票、退货未入账 |
| 应付账款 | 应付明细账 | 供应商对账单 | 到货未入账、付款未核销 |
| 存货 | 存货明细账 | 仓库实盘表 | 盘盈盘亏、系统未及时入库 |
| 固定资产 | 固定资产台账 | 实物盘点表 | 已报废未销账、已购入未登记 |
| 往来内部 | A公司应收B | B公司应付A | 内部交易未同步确认 |

### 核对五步法

**第一步：标准化**

将两侧数据对齐到统一格式：
- 统一日期格式（YYYY-MM-DD）
- 统一金额精度（保留两位小数）
- 统一编号格式（去空格、统一大小写）
- 确定匹配键（交易日期 + 金额 + 摘要/单号）

**第二步：逐笔匹配**

按匹配键做全外连接（full outer join），每笔交易归入以下类别：

| 类别 | 条件 | 含义 |
|------|------|------|
| 已匹配 | 两侧都有，金额一致 | 无差异 |
| 金额差异 | 两侧都有，金额不一致 | 可能是部分付款、手续费 |
| 时间差异 | 两侧都有，金额一致但日期不同 | 未达账项 |
| 仅总账侧 | 总账有，对方没有 | 企业已记、对方未记 |
| 仅对方侧 | 对方有，总账没有 | 对方已记、企业未记 |

**第三步：差异分类**

对每笔差异标注可能原因：
- **时间性差异**：截止日不同、跨月入账
- **遗漏**：一方漏记
- **重复**：一方重复入账
- **金额错误**：录入错误、汇率差异
- **科目错误**：记错科目导致某科目多/少

**第四步：编制核对表**

```
总账余额                              X
加：对方已记、企业未记                 A
减：企业已记、对方未记                 B
调整后总账余额                        X + A - B

对方余额                              Y
加：企业已记、对方未记                 B
减：对方已记、企业未记                 A
调整后对方余额                        Y + B - A

校验：调整后总账余额 = 调整后对方余额
```

**第五步：出具报告**

核对报告包含：
- 核对范围（科目、期间、数据来源）
- 匹配率（已匹配笔数 / 总笔数）
- 差异清单（按金额从大到小排列）
- 每笔差异的原因分类和处理建议
- 结论（是否需要调整分录）

### 匹配容差规则

| 场景 | 金额容差 | 日期容差 |
|------|----------|----------|
| 银行对账 | 0（精确匹配） | 3 个工作日 |
| 往来对账 | 0.01（分级差异） | 5 个工作日 |
| 存货盘点 | 按盘点误差率（通常 ≤ 0.5%） | 不适用 |
| 内部往来 | 0（必须精确） | 0（同一截止日） |

## 关联方识别方法论（12 维度交叉比对）

### 概述

关联方识别是审计中最容易出事的环节。隐性关联方不会主动告诉你它和被审计单位有关系——你需要从工商数据中主动发现线索。

核心思路：把被审计单位及其子公司的工商信息，与客户/供应商的工商信息做交叉比对，找出异常重叠。

### 数据来源

- **中注协行业知识库**（https://cmis.cicpa.org.cn）：CPA 证登录，"执业信息知识库"→"企业云"→"查企业"，可批量查询工商信息（一次最多 1 万家，每家 50+ 维度）
- **天眼查/企查查**：付费，数据更全
- **国家企业信用信息公示系统**：免费，但不支持批量

### 12 个识别维度

| 维度 | 检测逻辑 | 典型案例 |
|------|----------|----------|
| **D01 股权穿透** | 审计方（含子公司）有没有直接/间接投资客户或供应商 | A 公司通过 3 层子公司持有了客户 B |
| **D02 共同股东** | 多个客户/供应商背后是不是同一批股东 | 客户 A 和客户 B 的第一大股东都是张三 |
| **D03 实际控制人** | 穿透到最终受益人是不是同一个人 | 穿透后发现 5 家公司的实控人都是王五 |
| **D04 关键管理人员重叠** | 同一个人在被审计单位和客户/供应商都当高管 | 被审计单位的副总经理，同时在客户那当董事 |
| **D05 法定代表人交叉** | 同一个人给多家客户/供应商当法人 | 一人担任 4 家供应商的法定代表人 |
| **D06 法人变更轨迹** | 客户的"前任"法人是不是审计方的人 | 客户半年前把法人从"审计方员工"换成了别人 |
| **D07 同址经营** | 注册地址相同或高度相似 | 多家客户注册在同一个门牌号 |
| **D08 联系方式共用** | 电话号码或邮箱域名相同 | 3 家客户留了同一个手机号 |
| **D09 变更时间窗口** | 交易前后有没有"巧合"的工商变更 | 签合同前 1 个月客户突然换了法人 |
| **D10 企业名称相似** | 名字长得像（字号相同/相近） | 被审计单位叫"XX科技"，客户叫"XX科技发展" |
| **D11 参保人数异常** | 没人参保却做大额交易 | 参保人数 0，年交易额几千万 |
| **D12 补充人员关联** | 年报/公告里的完整董监高名单（含离任） | 工商登记只有 3 人，年报里其实有 12 人 |

### 操作步骤

**第一步：确定核查范围**
- 被审计单位本身
- 被审计单位的子公司（持股 50% 以上，逐层穿透）
- 全部客户清单
- 全部供应商清单
- 其他重要往来方

**第二步：获取工商数据**
- 从中注协企业云批量导出所有企业的工商信息
- 关键字段：股东信息、法定代表人（含历史变更）、主要人员、注册地址、联系电话、对外投资、参保人数

**第三步：补充人员名单**
- 从年报/公告中提取完整的董监高名单（包括离任人员）
- 这些人员参与所有维度的匹配（工商登记的"主要人员"通常只有部分高管）

**第四步：12 维度交叉比对**
- 每个维度本质上是集合求交集：
  - A 的股东 ∩ B 的股东 ≠ ∅ → 共同股东
  - A 的高管 ∩ B 的高管 ≠ ∅ → 人员关联
  - A 的电话 ∩ B 的电话 ≠ ∅ → 联系方式共用
- 对所有企业两两比对

**第五步：输出风险清单**
- 按命中维度数量排序（命中越多风险越高）
- 每条命中标注：维度名称 + 具体证据（如"共同电话：138XXXX"）
- 未命中任何维度的企业标记为"未发现异常"

### 关键提示

- **联系方式共用**是最强信号——不相关的企业用同一个手机号注册工商信息的概率极低
- **法人变更轨迹**容易被忽略——只看当前法人看不出来，要看历史变更记录
- **参保人数为 0**不一定是关联方，但结合大额交易就是强烈的空壳公司信号
- **补充人员名单**很关键——副总经理、财务总监等不在工商登记里，但可能是关联方的法人
- 12 个维度不需要全部命中才算关联方——命中 1-2 个就值得深入调查

# 各科目审计陷阱（实务经验精华）

> 来源：天职国际《实质性程序审计工作底稿编制指引》(2008) 经验性提示，经 2026 准则校验后保留。
> 仅保留"不随准则版本变化的实务智慧"，过时的会计/税务条款已剔除。
> 古立特在执行各科目审计程序时，应结合以下陷阱清单提醒用户注意。

## 货币资金

1. **定期存款查原件**：单位定期存款通常只有开户证实书而非存单。如果企业提供的是存单，要追问来源——可能存在未披露的质押事项。必须查看原件并亲自复印，不能只看复印件（可能已提前支取/已质押/已背书转让）。
2. **现金盘点四原则**：突击盘点 / 盘存保险柜内所有物件 / 出纳自行盘点审计员监盘 / 多地点同时盘点防止挪用。
3. **白条抵库必查**：盘点中发现的借条、未提现支票、未报销原始凭证，须在盘点表中注明。数家公司混用保险箱的情况要特别关注。
4. **多出来的东西比少的更危险**：盘点中多出来的款项、存单、有价证券，不能听任出纳随便解释——要追查原因。
5. **信用卡保证金**：余额较小时可用网上银行打印对账单替代函证。但如果持卡人是高管/出纳/个人，必须取得书面声明并检查资金流水。
6. **函证范围**：本期存过款的所有银行都要发函，包括零账户和已结清的账户。
7. **私设小金库手法**：截留收入 / 虚列支出 / 隐匿佣金回扣 / 截留罚没收入。检查方法：查"固定资产清理""包装物""低值易耗品"明细账的残值收入记录。

## 应收票据

1. **关联方票据**：重点关注关联方票据往来是否有违法情况（如融资性票据）。
2. **商业承兑汇票连带责任**：已贴现/已转让但未到期的商业承兑汇票，企业仍有连带付款责任，必须编制清单并检查是否存在贴现保证金。
3. **贴现息标记**：复核贴现息后用自定义标识标记，并在底稿说明含义。

## 应收账款

1. **借方发生额 vs 营业收入**：两者应配比。差异原因通常是预收账款结转收入、或货币资金收付的销售款未通过应收账款核算。
2. **贷方余额必查**：大额贷方余额必须查验合同和凭证，确认是预收账款才能重分类。须排除"货已发出+已收款但未开票"的隐瞒销售行为。
3. **同一客户多处挂账**：检查有无同一客户在应收/预收/其他应收等多个科目同时挂账。如有，函证时要保证数据完整性。
4. **账龄先进先出**：多笔应收且账龄不同时，收到还款按"先发生先收回"原则确定剩余账龄。
5. **函证覆盖率**：选取所有超过实际执行重要性 50% 的明细 + 金额覆盖率 ≥ 50% + 关联方/异常款项必抽。
6. **函证控制**：审计人员亲自收发，回函直接寄事务所。邮寄凭据保存在底稿中。传真件如果是客户传真号收到的，需注明"本人直接接收"。
7. **替代测试是逐户的**：不能把一个科目编一张替代测试表，必须每个明细客户单独一张。
8. **关联方销售真实性**：特别关注货物是否真实发出、是否存在季末/年末突击销售或超常大额销售。
9. **应收账款融资**：检查银行回函、会议纪要、借款协议、贷款卡信息，确认应收账款是否已被质押或出售。

## 其他应收款

1. **长期挂账大额款项**：即使回函相符，仍应进一步取证核实债务人偿还能力。累计大额亏损或资产不实的，应考虑补提坏账准备。
2. **长期无发生额的大额挂账**：重点检查初始发生时的原始依据、董事会决议、支付授权，查证是否为关联方/股权投资/非法谋利/未经授权的非正常支付。
3. **账龄操纵**：特别关注账龄较长的余额发生大幅变化，或余额不变但发生额较大——可能是为重新计算账龄而进行的调节。
4. **"其他应收款-其他"**：期末余额不大但发生频繁的混合项目，必须按最明细项目分解，逐项查验发生额。
5. **外部证据优先**：相关交易的真实合法性需要从独立第三方角度判断，不能仅依赖函证。

## 金融资产（交易性/可供出售/持有至到期 → 2026 准则下按 SPPI 三分类）

1. **股东卡查原件**：要求公司提供所有证券账户的资金卡和股东卡，包括余额为零但未销户的。股东卡必须查看原件以防伪造。
2. **以他人名义的账户**：需进一步确认账户内资金及证券的所有权，并及时上报外勤主管。
3. **未经授权开户**：没有董事会批准就开立了资金/证券账户的，需追查交易行为合法性。
4. **在外保管的有价证券**：需取得保管协议/托管凭证，向保管人函证，函证中注明是否存在抵押/出借。
5. **持有意图和能力**：每个资产负债表日都要评价。发生变化的应重分类。
6. **国债不减值，公司债可能减值**：国债是财政部发行不会无法收回；公司债券发行方财务恶化时应考虑个别认定法计提减值。

## 固定资产

1. **台账 vs 财务账核对**：初次审计时将设备管理部门台账与固定资产账核对十分必要。名称/分类差异较小的找原因核对，差异大的以财务记录为准与实物核对。
2. **盘点重点是新增**：常年审计客户，新增固定资产盘点是重点。
3. **有证无账 / 有账无证**：
   - 有权证但账面无记录 → 查明原因，判断是否存在账外资产
   - 账面有记录但无权证 → 查明未能取得权证的原因，分析对经营的影响
4. **权证名称不一致**：权证上权利人与客户名称不一致的，要求书面解释 + 权利人书面声明。金额大的要求变更权属关系。
5. **隐性质押**：注意房产已抵押但"他项权利记录"未登记入产权证的情况。结合贷款卡信息和银行询证了解。
6. **闲置固定资产**：获取闲置证明文件，观察实际状况，检查是否仍在计提折旧。

## 在建工程

1. **前 10 大合同必查**：筛选借方发生额最大的前 10 笔，检查合同、发票、进度确认单。
2. **资本化利息**：检查借款合同、利率、资本化起止时间，复核计算是否正确。
3. **转固时点**：关注在建工程是否及时转入固定资产（延迟转固 = 少提折旧 = 虚增利润）。

## 无形资产

1. **使用寿命不确定的不摊销**：但每个资产负债表日必须复核使用寿命。
2. **摊销基数含减值**：应摊销金额 = 成本 - 预计残值 - 减值准备。计提减值后当月起按新基数摊销。
3. **研发支出资本化**：结合"开发支出"审计，检查是否满足 5 个资本化条件。

## 长期股权投资

1. **被投资单位报表未审计**：根据重要性考虑是否对被投资单位实施审计/审阅程序。
2. **利用其他注册会计师工作**：被投资单位由其他所审计的，需复核其审计报告，必要时重新审计。
3. **权益法调整三项**：
   - 会计政策/会计期间不一致 → 调整
   - 以取得投资时公允价值为基础调整净损益
   - 内部交易损益按持股比例抵销
4. **购并日判断 5 条件**：股东大会通过 / 主管部门批准 / 财产权交接 / 已付大部分价款(>50%) / 实际控制财务和经营。

## 往来款项通用

1. **期初数必须与上年审定数核对**：如果客户未对上年调整数做账务处理，审计员要将期初数调整至与上年审定数一致，并在审计说明中描述。
2. **追溯调整**：本期如存在追溯调整事项，期初数应该是追溯调整后的数字。
3. **方向异常**：资产贷余 / 负债借余 → 查明原因，必要时重分类调整。

## 应付账款

1. **长期挂账（≥3 年）三种原因**：质量纠纷挂账 / 虚假业务形成的负债 / 债权人破产或豁免。审计人员可向法务部调查是否存在未决诉讼。
2. **完整性检查**：复核截至审计现场日的全部未处理供应商发票 + 检查期后付款追查至银行对账单 + 结合存货监盘检查"货到单未到"暂估入库。
3. **虚假银行承兑汇票融资**：部分企业以虚假交易合同开具银行承兑汇票作为融资手段。关注是否存在因财务指标恶化无法正常贷款的情况。

## 应付票据

1. **保证金核对完整性**：银行承兑汇票需按比率存入保证金（10%-100%），结合其他货币资金检查判断票据记录是否完整。
2. **空白票据管理**：未签发的空白票据是否妥善保管，已作废的是否加盖作废印章、三联是否齐全。
3. **逾期票据**：逾期银行承兑汇票检查是否转入短期借款，逾期商业承兑汇票检查是否转入应付账款，带息票据是否已停止计息。

## 长期借款

1. **函证与银行存款合并**：借款函证应结合银行存款函证一并进行。如果由不同人员完成，需统一汇总防止遗漏。
2. **外币借款填原币**：询证函上的金额应填写原币而非折算后的记账本位币，并注明币种。
3. **利息预提判断**：如年末借款余额均衡、利率变动不大、核算归属期为上年末至本年末结息日，可不预提最后 11 天的应计利息。

## 预计负债

1. **需要大量职业判断**：被审计单位各业务环节均可能涉及或有事项。负责该科目的审计人员必须先充分了解被审计单位及其环境，且与项目组其他成员随时沟通。
2. **不提供资料 = 审计范围受限**：如果客户不愿提供与或有事项有关的全部文件和凭证，视为审计范围受限。
3. **明细项目**：通常包括未决诉讼/仲裁、对外担保、票据贴现、亏损合同、环境整治义务、重组义务、产品质量保证、对被投资单位超额亏损承担额外损失义务等。

## 营业收入

1. **定价政策三关注**：是否违反国家规定 / 是否向关联方不按市场价转移利润 / 售价是否背离价格目录。
2. **抽查发货凭证关注完整性**：发现已出库但未开票/未入账的情况。
3. **收入真实性查验要点**：结合应收账款借方发生额 / 重点关注前 10 大客户 / 询证发生额 / 检查销售合同与全年销量是否相符。
4. **重大调整事项 100% 测试**：虚假销售退回、非常规项目多计/少计成本，是调节利润的常见手段，必须逐笔细节测试。

## 营业成本

1. **存货发出计价操纵**：企业可能人为调小发出单位成本（虚增期末存货 = 虚增利润）。需结合存货发出计价测试检查。
2. **成本与费用划分**：关注是否将应计入生产成本的支出计入期间费用（或反之）来调节利润。
3. **计价方法变更**：同一会计期内不得变动。如变更需检查理由、性质、对当期损益的影响。

## 费用类通用

1. **截止测试样本量**：资产负债表日前后 5-20 天，根据发生频率确定样本量。频率低的可全抽。
2. **返利/佣金/业务提成**：取得相关费用支出规定，检查是否符合规定、审批手续是否健全、是否取得有效原始凭证。根据销售收入测算是否充分完整。

# 底稿编制全流程操作手册（2026 准则版）

> 本章节是古立特做底稿的"操作步骤总纲"。
> 详细的六阶段程序逻辑、数据流转关系、各阶段底稿清单见 `audit_procedures.md`。
> 古立特的定位是"半自动"——搭起骨架让人填，不替代人做实地盘点/管理层访谈/形成审计意见。

## 一、AI 与人工的分工边界

| AI 能做（古立特工具完成） | 人工必做（古立特只能辅助） |
|---------------------------|---------------------------|
| 数据计算（调节、账龄、折旧、抽样、截止） | 实地盘点 / 现场观察 |
| 模板填充（文件夹+空白模板） | 询问管理层 / 治理层沟通 |
| 异常项标记（波动率、本福特、整数、月末） | 评价管理层诚信 |
| 报表勾稽校验 | 检查实物凭证原件 |
| 重要性计算 + 职业判断说明 | 形成审计意见 |
| 函证文件生成（Word + Excel） | 签字盖章 / 装订 |

## 三、关键阈值（古立特默认建议值）

依据 CSA 1320 + 实务通行标准。所有阈值在 `materiality_calculator` 工具中可自定义：

| 项目 | 古立特默认 | 准则范围 | 调整方向 |
|------|----------|---------|---------|
| 重要性（税前利润法） | 5% | 5%-10% | 风险高时取上限 |
| 重要性（总资产法） | 0.5% | 0.5%-2% | 用于亏损或微利企业 |
| 重要性（营业收入法） | 0.5% | 0.5%-2% | 用于初创/研发驱动 |
| 实际执行重要性 | 60% | 50%-75% | 风险高时取下限（更严） |
| 明显微小错报 | 5% | 3%-10% | 实务通行，准则未规定具体比例 |
| 集团组件重要性 | 80% | 50%-90% | CSA 1411 双限值 |
| 截止测试窗口 | 前后各 7 天 | 5-10 天 | 风险高时缩到 3 天 |
| 大额检查阈值 | 实际执行重要性 | 同左 | — |
| 控制测试样本量 | 25 | 25-60 | 控制频率高时取上限 |
| 账龄长账龄阈值 | ≥ 1 年 | 1-2 年 | — |
| 异常波动阈值 | ≥ 10% | 5%-15% | 行业不同适当调整 |

## 四、底稿编制顺序（按 W13 工作流）

```
1. workpaper_init        → 建目录 + 生成空白模板
2. 业务承接评价          → 决定是否承接（人工）
3. materiality_calculator → 计算重要性
4. 数据准备：data_file_overview 看科目余额表/序时账
5. analytical_procedures → 本期 vs 上期波动分析
6. 风险评估              → 识别特别风险（人工 + 古立特辅助）
7. 制定具体审计计划      → 各科目程序设计

8. 各科目实质性程序：
   • 货币资金   → bank_reconciliation + cutoff_test
   • 应收账款   → aging_analysis + audit_sampling + confirmation_letter + cutoff_test
   • 存货       → cutoff_test + 实地盘点
   • 固定资产   → depreciation_check + audit_sampling
   • 收入       → cutoff_test + benford_law_check + 分析性程序
   • 费用       → cutoff_test + audit_sampling
   • 应付账款   → aging_analysis + audit_sampling + confirmation_letter
   • 在建工程   → data_filter + 资本化利息测试

9. 边做边记录：
   • 自动化工具使用记录（每用一次工具就更新）
   • 重大职业判断记录单（关键判断必填）

10. audit_adjustment    → 三栏 TB（未审 + 调整 = 审定）
11. 完成阶段：
    • 未更正错报汇总
    • 期后事项复核
    • 项目组复核
    • 审计总结
    • 审计报告

12. 归档：所有产出按 1-7 文件夹分类存放
```

## 五、ISA 500 自动化工具使用记录要求

2026 年准则强化要求：使用 AI / RPA / 数据分析工具时，必须在底稿中记录：

| 字段 | 内容 | 示例 |
|------|------|------|
| 工具名称 | 完整工具名 + 版本 | 古立特 Gridman v1.7.0 / pandas 2.3 |
| 用途 | 执行的审计程序 | 银行余额调节表自动编制 |
| 数据来源 | 输入文件路径 | 客户提供的银行对账单 + 银行日记账 |
| 数据验证步骤 | 古立特/审计员做了哪些校验 | 金额格式 / 日期格式 / 重复值检查 |
| 人工复核逻辑 | 审计员复核了什么 | 未达账项原因人工核实 |
| 执行人 + 日期 | 谁、何时执行 | — |

`workpaper_init` 生成的【自动化工具使用记录.xlsx】已预填古立特工具清单，用户填执行人和日期即可。

## 六、重大职业判断记录要求

2026 年准则要求记录的关键判断：

1. **重要性水平的确定** → `materiality_calculator` 输出的"职业判断说明"可直接复制
2. **特别风险的识别依据**
3. **会计估计的合理性评价**（如坏账准备计提比例）
4. **以前年度损益调整**等非常规事项
5. **持续经营能力评估**

每项判断需记录：判断内容 / 判断依据 / 支持证据 / 判断结论 / 判断人 / 判断日期。

## 八、对接事务所内部模板

古立特生成的是"通用骨架"，不是事务所专属模板。需要做的事：

1. **替换封面**：用所内标准封面
2. **替换抬头**：用所内格式（报告字号、客户全称等）
3. **加签字栏**：审计员/项目经理/合伙人签字位置
4. **统一配色**：如有所内 VI 规范
5. **装订规范**：纸质/电子归档要求

古立特负责"内容正确"，事务所负责"形式合规"。
