# 上市公司财报研究工作流

> 定位：从外部研究一家上市公司单季/年度财报的端到端方法论，证据链驱动、防编造
> 来源：杜雨「AI 财报分析工作流」五阶段模板（GitHub 开源）+ 审计证据可靠性思维蒸馏
> 覆盖：公司画像、材料搜集分级、指标抽取、数字稽核、报告撰写、行业特有指标、三市场来源
> 性质：实操+判断

本文件是"外部研究上市公司财报"的主存放点，与以下文件分工不同，按需联动：
- 报表勾稽、税务风险、合规审查（拿到报表后逐表核） → 详见：statement.md
- 财务造假/洗澡识别（研究中发现异常时） → 详见：financial_fraud.md
- 一级市场尽调、DCF/可比公司估值 → 详见：investment.md
- 企业内部经营分析（BP 视角，非外部研究） → 详见：financial_bp.md
- 下年报/识别扫描件/算财务比率（MCP 工具） → report_download、document_ocr、financial_ratios、stock_financial

核心区别：investment.md 偏一级市场尽调与估值建模；本文件偏**二级市场已上市公司的季度/年度财报研究**，目的是搞清"这家公司本季最值得关注、最可能影响估值的变化是什么"，不给买卖建议。

## 一、五阶段工作流总览

> 类型：流程
> 来源：杜雨财报分析工作流
> 置信度：实务经验

财报研究分五个阶段，**每阶段一个独立会话，严格隔离，前一阶段产出是后一阶段唯一输入**。这种隔离是防 AI 编数据的核心机制——把"抽取"和"写报告"混在一起，AI 容易凭记忆补全不存在的数字。

```
00 公司画像 → 01 材料搜集 → 02 指标抽取 → 03 数字稽核 → 04 报告撰写
   建框架       搜集分级       只抽取        只核验        只写已核验
```

| 阶段 | 干什么 | 绝对不能干 | 产出 |
|---|---|---|---|
| 00 公司画像 | 建立"这家公司怎么看"的研究框架 | 不写分析报告、不下投资结论 | 研究框架 + 后续搜集重点 |
| 01 材料搜集 | 搜集、下载、登记、分级一手材料 | 不写分析、不抽数字 | 来源清单（含可信度分级） |
| 02 指标抽取 | 从材料抽取数字与观点 | 不成稿、不下结论 | 指标表 + 管理层观点 + 待稽核清单 |
| 03 数字稽核 | 回原始材料逐项核验 | 不写报告、不相信 02 的结论 | 已核验事实清单 + 不可进报告清单 |
| 04 报告撰写 | 只用核验通过的事实写报告 | 不用未核验内容、不给买卖建议 | 财报研究报告 + 入库计划 |

⚠️ 阶段隔离铁律（防编造的关键）：
- 新阶段只读上一阶段的产出 + handoff + 必要原始材料，**不读完整旧对话**。
- 不把整份 PDF / 长电话会文字稿塞进上下文——保存到本地，按需读片段。
- 03 稽核阶段**不准相信 02 的抽取结果**，必须回原始材料重核。

### 工作流定位判断

> 类型：判断表
> 来源：实务经验
> 置信度：实务经验

[场景：用户拿来一个研究需求，先判断走不走这套完整工作流]

| 判断顺序 | 情形 | 处理 |
|---|---|---|
| 1 | 研究一家**已上市公司**的某季/某年财报、首次覆盖、重仓标的跟踪 | 走本工作流完整五阶段 |
| 2 | 只是快速看一个指标、算个比率 | 不必走全流程，直接 financial_ratios / statement.md |
| 3 | 一级市场拟 IPO 企业尽调、估值建模 | 转 investment.md |
| 4 | 企业内部经营分析、给老板写经营月报 | 转 financial_bp.md |
| 5 | 已怀疑财务造假，要查手法/红旗 | 转 financial_fraud.md（本工作流的 03 稽核会用到它） |

## 二、阶段 00：公司画像与研究框架

> 类型：流程
> 来源：杜雨工作流 00 阶段
> 置信度：实务经验

[场景：财报研究的第一步——先回答"这家公司应该怎么看"，再回答"本季表现如何"。不套通用模板。]

**本阶段只建框架，不写分析、不下结论、不入库。**

核心原则：
- 先识别公司所属行业、商业模式、产业链位置、估值敏感变量，再谈本季数据。
- 优先用官方材料、监管文件、交易所公告、业绩会文字稿、investor presentation。投行/咨询/媒体只能作"市场关注点"参考，不能替代官方披露。
- 不确定信息标注"待核验"，不编造。

**13 节研究框架产出清单：**

| # | 框架要素 | 关键产出 |
|---|---|---|
| 1 | 公司一句话定位 | 是谁、什么行业、靠什么赚钱、产业链什么角色、市场当成什么资产定价 |
| 2 | 主营业务与收入/利润结构 | 业务板块表：产品/收入贡献/利润贡献/增长状态/后续重点 |
| 3 | 商业模式与价值驱动因子 | 增长靠什么、利润率靠什么、现金流受什么影响、估值最敏感变量 |
| 4 | 产业链位置 | 上游依赖、下游客户、关键供应商、议价能力、卡脖子环节 |
| 5 | 核心竞争力与护城河 | 区分"已被事实验证的优势"vs"市场叙事中的优势" |
| 6 | 市场和分析师真正关注什么 | 从近 2-4 季电话会分析师反复追问的问题反推 |
| 7 | 行业特有指标自动识别 | 见第五章——按商业模式反推该看什么指标 |
| 8 | 本季应重点抽取的指标 | 通用财务指标 + 行业特有指标两类 |
| 9 | 管理层观点应重点听什么 | 表现解释/指引/订单/价格/产能/CapEx/新品/竞争/风险 |
| 10 | 关键风险与反证点 | 最容易误判的地方：一次性收益、周期高点、库存、客户集中度等 |
| 11 | 对标公司 | 选谁对比 + 为什么 + 比哪些指标 |
| 12 | 后续材料搜集清单 | 按优先级列 01 阶段该搜什么 |
| 13 | 公司专属搜集提示 | 为 01 阶段生成本公司专属的搜集重点输入 |

⚠️ 第 5 节"护城河"必须区分两类，否则 AI 会把市场叙事当事实：
- **事实验证的优势**：有财报数字/披露支撑（如毛利率持续高于同行 5 年）。
- **市场叙事中的优势**：只是研报/媒体在说（如"AI 龙头""国产替代核心标的"）。

## 三、阶段 01：材料搜集与来源分级

> 类型：流程
> 来源：杜雨工作流 01 阶段
> 置信度：实务经验

[场景：基于 00 的框架去搜集一手材料。本阶段只搜集/下载/登记/分级，不抽数字、不写分析。]

核心原则：一手材料优先于媒体报道；官方/监管/交易所/IR/电话会文字稿优先；不把网页搜索结果直接当事实，必须记来源链接、发布时间、材料类型、可信度。

### 财报事项时间线（五类材料）

> 类型：判断表
> 来源：杜雨工作流 01 阶段
> 置信度：实务经验

不要只找财报 PDF——一次完整研究要覆盖财报发布的全时间线：

| 时间线 | 材料 | 发布主体 | 用途 |
|---|---|---|---|
| 财报前 | 盈利预期 / Consensus Estimates | 分析师、投行、数据机构 | 判断业绩 beat/miss/in-line 的基准 |
| 财报前/发布时 | 业绩指引 / Guidance | 公司管理层 | 判断未来预期变化 |
| 发布当天 | 业绩公告 / Earnings Release | 公司 | 快速获取核心结果 |
| 发布后 | 电话会 / 业绩说明会文字稿 | 管理层 + 分析师 | 抽经营解释、展望、Q&A、语气变化 |
| 发布后 | 正式季报/年报（10-Q/10-K 等） | 公司向监管提交 | 最终事实核验、三表、MD&A、风险因素 |

### 三市场来源优先级

> 类型：判断表
> 来源：杜雨工作流 01 阶段
> 置信度：实务经验

按公司上市地选择来源，由高到低：

| 市场 | 一手来源优先级 | 重点文件 |
|---|---|---|
| 美股 | 公司 IR 官网 → SEC EDGAR → Earnings Release → 电话会文字稿 → 10-Q/10-K/20-F/6-K → Investor Presentation → Consensus | 10-Q（季报）、10-K（年报）、8-K（重大事项/业绩附件）、20-F/6-K（外国发行人） |
| 港股 | HKEXnews → 公司公告 → 年度/中期业绩公告 → 年报/中报 → 业绩发布会材料 → IR 官网 | 港股无类似 10-Q 的季度正式文件，年度/中期业绩公告比新闻稿重要 |
| A股 | 巨潮资讯 → 沪深交易所公告 → 定期报告 → 业绩预告/快报 → 业绩说明会 → 投资者关系活动记录表 | 一季报/三季报较简略，需结合投资者关系活动记录表；业绩预告/快报与正式报告口径可能不同，两者都留 |

⚠️ A 股特别提醒：投资者关系活动记录表常含订单、产能、海外、技术路线等"财报外信息"，别漏。古立特可用 `report_download` 从巨潮下 A 股年报/公告。

### 来源可信度分级（ABCDE）

> 类型：判断表
> 来源：杜雨工作流 01 阶段
> 置信度：实务经验

[场景：每份材料登记时打一个可信度等级，决定它在后续能怎么用——这是防止把传闻当事实的闸门]

| 等级 | 来源类型 | 使用方式 |
|---|---|---|
| A | 公司官方、监管文件、交易所公告、财报、电话会文字稿 | 可作核心事实依据 |
| B | 公司 IR 活动、investor presentation、管理层公开访谈 | 可作管理层观点依据 |
| C | 行业协会、政府机构、咨询公司、权威数据平台 | 可作行业背景和交叉验证 |
| D | 投行/卖方报告、权威财经媒体、市场预期数据 | 可作市场关注点、consensus、观点参考 |
| E | 普通媒体、博客、论坛、社交平台 | 只作线索，不作最终事实 |

### 搜集优先级（P0/P1/P2）

> 类型：判断表
> 来源：杜雨工作流 01 阶段
> 置信度：实务经验

| 优先级 | 材料 |
|---|---|
| P0 必须 | 官方季报/年报/10-Q/10-K/交易所公告、Earnings Release、电话会文字稿、Investor Presentation、本季/下季 Guidance |
| P1 强烈建议 | Consensus Estimates、CEO/CFO 访谈、analyst day 材料、IR 活动记录、产销/运营月度数据、对标公司同季财报 |
| P2 辅助 | 行业协会/咨询报告、卖方关注点摘要、权威财经媒体、供应链/客户/订单/产能公开资料、公司新闻稿/技术白皮书 |

⚠️ 材料缺失时**不编造**，写入"缺失材料与替代方案"——记录缺什么、为什么需要、替代来源、后续动作。

⚠️ 必须区分四类材料，登记时标清楚：①财报前市场预期 ②公司实际披露 ③管理层解释 ④监管正式文件。这四类在后续 02/03 阶段不能混用。

## 四、阶段 02：指标与管理层观点抽取

> 类型：流程
> 来源：杜雨工作流 02 阶段
> 置信度：实务经验

[场景：从 01 搜集的材料里抽取可被后续稽核和写作使用的事实、数字、口径、观点。只抽取，不成稿，不下结论。]

核心原则——**抽取阶段最大的纪律是"区分来源类型，不合并口径"：**

⚠️ 每个数字、每个观点必须显式区分以下来源类型，混用是 02 阶段最致命的错误：
- 实际披露结果
- 市场预期 / consensus
- 管理层 guidance
- 管理层解释
- 分析师 / 媒体观点
- 监管正式文件

**抽取顺序：** 财务总览 → 业务变化 → 财务三表 → 行业特有指标 → 管理层观点 → Q&A → 风险因素 → Guidance → 现金流质量 → 待稽核清单。

**抽取规则（每条都是防错）：**

1. 每个关键数字必须记录**来源 + 位置（页码/章节/表格）+ 口径 + 单位 + 期间**。
2. 公司披露同比但没披露环比 → 环比填"未披露"，不自己算了硬填。
3. 需要自行计算的 → 标注"计算值"并写出公式。
4. GAAP / Non-GAAP 同时存在 → **分开列，不合并**。
5. 币种不同 → 保留原币种，**不擅自换算**。
6. 单位必须标注（千/百万/十亿/亿元/亿美元）。
7. 口径不一致 → **不自行合并，标注"待稽核"**。
8. 管理层观点 → 记录说话人、来源材料、上下文、主题；可短引原文，不大段复制。
9. 模糊表述 → 标注"叙事性表述，待后续验证"。

**核心抽取模块：**

| 模块 | 抽取要点 |
|---|---|
| 核心财务指标 | 营收/毛利率/营业利润/净利润/EPS/Non-GAAP/经营现金流/自由现金流/CapEx/库存/应收/合同负债/backlog/guidance |
| 分部/业务板块 | 各板块收入/同比环比/利润率/运营指标；增长来自哪、谁拖累 |
| 行业特有指标 | 见第五章，按 00 框架识别的指标抽取 |
| 市场预期对比 | consensus vs 实际，beat/miss；无可靠 consensus 则写"未找到可靠市场预期来源" |
| Guidance | 指引区间/覆盖期间/口径/相比上期变化（上调/下调/维持） |
| 现金流质量 | 净利润与经营现金流是否匹配、FCF 是否改善、CapEx 性质、应收存货是否占用现金、回购分红偿债是否靠外部融资 |
| 管理层观点 | 按主题抽取 + 语气（更乐观/更谨慎/无变化） |
| 分析师 Q&A | 反复追问什么、管理层是否正面回答、哪些是市场担忧、哪些对应估值敏感变量 |
| 风险与反证 | 像做空调查员一样审视，但只抽证据不定性；重点看 Risk Factors 中新增/强化的风险 |

最后产出**待稽核清单**：把所有需要进入 03 阶段核验的项目列出（数字口径、同比环比计算、单位币种、GAAP/Non-GAAP、指引变化、市场预期差异、分部口径、管理层叙事、行业特有指标、对标比较）。

## 五、行业特有指标库（自动识别）

> 类型：判断表
> 来源：杜雨工作流 00/02 阶段
> 置信度：实务经验

[场景：财报研究最核心的能力——不套通用财务指标，按公司商业模式反推"这家公司必须看什么"]

反推四问：①公司最大约束是什么（需求/产能/技术/原材料/监管/资金/渠道/客户集中度）？②市场最担心什么（增长放缓/毛利下滑/库存/订单取消/CapEx 过重/技术替代/价格战/监管）？③管理层最需要证明什么？④哪些指标能验证这些问题？

| 行业 | 行业特有指标 |
|---|---|
| AI 半导体 | GPU/ASIC 出货、HBM 供应、先进封装产能、云厂商 CapEx、backlog、长期供货协议、产品迭代节奏、客户集中度 |
| 晶圆代工 | 先进制程收入占比、产能利用率、CapEx、先进封装、主要客户订单、制程节点迁移 |
| SaaS/订阅软件 | ARR、NRR、RPO、客户留存、ARPU、CAC、payback period、AI 功能变现 |
| 新能源车 | 销量、ASP、车型结构、毛利率、库存、海外销量、电池成本、智能驾驶渗透率 |
| 电池/储能 | 出货 GWh、单位成本、产能利用率、长单、储能订单、材料成本、技术路线 |
| 资源企业 | 产量、售价、单位现金成本、储量、品位、剥采比、长协价格、CapEx |
| 银行 | 净息差、不良率、拨备覆盖率、贷款结构、存款成本、资本充足率 |
| 医药研发 | 管线进度、临床阶段、适应症、专利期、销售峰值、医保/支付方覆盖 |
| 云计算 | 收入增速、营业利润率、backlog、CapEx、AI revenue run-rate、GPU/数据中心容量 |
| 消费品牌 | 同店销售、渠道库存、毛利率、客单价、复购率、营销费用率 |

⚠️ 上表是提示不是清单——必须以本公司 00 阶段研究框架反推的结果为准。一家公司常跨多个商业模式（如车企同时是制造+电池+智驾软件），指标要叠加。

## 六、阶段 03：数字与叙事稽核

> 类型：流程
> 来源：杜雨工作流 03 阶段
> 置信度：实务经验

[场景：质量闸门。不写报告、不入库。任务是验证 02 抽取结果是否可靠，找出口径差异、数字错误、叙事冲突、待人工复核项。这一步是整个工作流的"职业怀疑"所在。]

⚠️ 03 阶段第一铁律：**不准相信 02 的抽取结果，必须回原始材料逐项重核。** 这等同于审计的"不依赖管理层声明，独立获取证据"。

**稽核优先级（12 项）：** 核心财务数字 → 同比环比计算 → 单位币种期间 → GAAP/Non-GAAP 口径 → 分部数据 → 市场预期 vs 实际 → Guidance 变化 → 现金流与利润质量 → 行业特有指标 → 管理层观点与 Q&A → 风险因素 → 对标公司口径。

**口径稽核重点检查项**（最易出错的地方）：

| 口径维度 | 易错点 |
|---|---|
| 来源口径 | Earnings Release vs 10-Q/10-K |
| GAAP/Non-GAAP | 两者数字不同，混用导致 beat/miss 误判 |
| 汇率口径 | Reported vs constant currency（固定汇率） |
| 合并范围 | 合并报表 vs 分部、持续经营 vs 总经营 |
| 期间口径 | 财季 vs 自然季（财年错位的公司，如英伟达 FY 与自然年差近一年） |
| 币种 | RMB/USD/HKD/KRW/TWD 不擅自换算 |
| 单位 | million/billion/亿元/万元 |

**稽核状态四档：** 通过 / 部分通过 / 未通过 / 待人工复核。

**两个关键产出：**
1. **不可进入最终报告的内容清单**——无来源、可信度低、口径不清、无法复算、与官方冲突、只是传闻、管理层表述过于模糊的，全部排除。
2. **已核验事实清单**——标注置信度（高=官方披露口径清楚 / 中=有来源但需说明限制 / 低=只作背景或待观察）。只有"通过/部分通过且说明限制"的内容才能进 04 报告。

⚠️ 现金流质量稽核——CapEx 是 growth（增长性）还是 maintenance（维持性），公司未披露时标注"无法确认"，不臆断。这直接影响自由现金流质量判断。

> 研究中若稽核出造假/洗澡红旗（如收入与现金流持续背离、毛利率显著偏离同行、应收增速远超收入），转 financial_fraud.md 做深度判断；涉及处罚先例查 penalty_cases.md。

## 七、阶段 04：报告撰写与知识入库

> 类型：流程
> 来源：杜雨工作流 04 阶段
> 置信度：实务经验

[场景：最终成稿阶段。只用 03 稽核标注为"通过/部分通过且说明限制"的内容写报告。可生成入库计划，但不直接写入知识库，须等用户确认。]

**报告十条原则：**
- MECE（结构互斥且完整，不重复堆叠）
- 客观（事实/管理层观点/推断/风险必须区分）
- 简洁（先写关键结论，不写流水账）
- 主次分明（先写最影响公司价值的变量，不平均用力）
- 喜忧都报（既写机遇也写挑战和风险，不只写增长亮点）
- 公司独特性（不写成通用财报模板）
- 行业 know-how（体现行业特有指标和产业链位置）
- 可追溯（核心数字和判断能回到来源）
- **不给买卖建议**（输出研究判断，不输出交易指令）
- 不把短期季度波动写成长期确定趋势

每份报告必须回答的核心问题：**这家公司本季度最独特、最值得关注、最可能影响未来估值的变化是什么？**

**报告标准结构：** 一句话结论 → 核心结论摘要（5-8 条，按重要性排，每条含 结论/依据/影响/风险限制）→ 核心财务指标表 → 本季表现拆解（MECE）→ 行业特有指标与公司独特性 → 管理层观点与语气变化 → 市场预期差异 → 现金流与利润质量 → 机遇挑战与风险 → 对标公司视角 → 后续跟踪清单。

**写作风格：** 中文、结论先行、不写废话、不堆材料、不用营销语言、不写"公司未来可期"这种空话、重点数字带口径、重要不确定性明示。

⚠️ 知识入库（如写入个人笔记库）规则：
- 不直接写入，只生成入库计划，**默认所有写入都需用户确认**。
- 笔记已存在同名公司 → 不简单覆盖，先比对差异，做增量摄入；旧结论被本次证据推翻时明确标注"原判断需修正"+依据；保留旧笔记中仍有效的长期公司画像/商业模式/产业链判断。
- 未通过稽核的内容不得建议写入。
- 项目过程文件（00-03）不入库，只入 04 的最终报告。

## 九、重要提示

1. **阶段隔离是防编造的核心**：不把"抽取"和"写报告"混在一起；新阶段只读上阶段产出，不读完整旧对话；不把整份 PDF 塞进上下文。
2. **来源决定一切**：每个数字带来源口径，区分实际/预期/guidance/解释/分析师观点/监管文件，混用即错。
3. **不给买卖建议**：古立特做财报研究判断，不做交易指令——这是边界。
4. **不把季度波动写成长期趋势**：单季 beat/miss 不等于长期拐点。
5. **缺材料不编造**：缺什么写清楚 + 替代方案，宁可承认置信度受限。
